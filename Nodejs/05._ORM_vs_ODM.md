# ORM vs ODM

Both **ORM** and **ODM** are tools that let you interact with databases using **JavaScript objects** instead of writing raw database queries.

- **ORM** (Object Relational Mapper) = for **relational databases** (MySQL, PostgreSQL, SQLite)
- **ODM** (Object Document Mapper) = for **document-based databases** (MongoDB)

---

## Real-World Example

### Without ORM (raw SQL)
```js
// You must write SQL manually
const result = await db.query(
  "SELECT * FROM users WHERE email = $1",
  ["nishant@example.com"]
);
```

### With ORM (Sequelize)
```js
// Clean JavaScript — no SQL needed
const user = await User.findOne({ where: { email: "nishant@example.com" } });
```

## ORM — Object Relational Mapper

Used with **relational databases** that store data in **tables with rows and columns**.

### Popular ORMs for Node.js:
| ORM | Databases |
|-----|---------|
| **Sequelize** | MySQL, PostgreSQL, SQLite, MSSQL |
| **Prisma** | MySQL, PostgreSQL, SQLite, MongoDB |
| **TypeORM** | MySQL, PostgreSQL, SQLite, and more |
| **Knex.js** | Query builder (not full ORM) |

### Sequelize Example
```js
const { Sequelize, DataTypes } = require("sequelize");

const sequelize = new Sequelize("sqlite::memory:");

// Define a model (maps to a "users" table)
const User = sequelize.define("User", {
  name: DataTypes.STRING,
  email: { type: DataTypes.STRING, unique: true },
  age: DataTypes.INTEGER,
});

// Create, Read, Update, Delete
async function run() {
  await sequelize.sync();

  // CREATE
  const user = await User.create({ name: "Nishant", email: "n@n.com", age: 22 });

  // READ
  const found = await User.findOne({ where: { email: "n@n.com" } });
  console.log(found.name);

  // UPDATE
  await User.update({ age: 23 }, { where: { email: "n@n.com" } });

  // DELETE
  await User.destroy({ where: { email: "n@n.com" } });
}

run();
```

## ODM — Object Document Mapper

Used with **document databases** that store data as **JSON-like documents** (no fixed schema).

### Popular ODMs for Node.js:
| ODM | Database |
|-----|---------|
| **Mongoose** | MongoDB |
| **Typegoose** | MongoDB (with TypeScript) |

### Mongoose Example
```js
const mongoose = require("mongoose");

mongoose.connect("mongodb://localhost:27017/myapp");

// Define a Schema (structure of each document)
const userSchema = new mongoose.Schema({
  name: { type: String, required: true },
  email: { type: String, unique: true },
  age: Number,
  createdAt: { type: Date, default: Date.now },
});

// Create a Model from the schema
const User = mongoose.model("User", userSchema);

async function run() {
  // CREATE
  const user = await User.create({ name: "Nishant", email: "n@n.com", age: 22 });

  // READ
  const found = await User.findOne({ email: "n@n.com" });

  // UPDATE
  await User.updateOne({ email: "n@n.com" }, { age: 23 });

  // DELETE
  await User.deleteOne({ email: "n@n.com" });
}

run();
```

## Comparison Table

| Feature | ORM | ODM |
|---------|-----|-----|
| Database type | Relational (SQL) | Document (NoSQL) |
| Data format | Tables, rows, columns | JSON-like documents |
| Fixed schema | ✅ Yes (strict) | 🟡 Optional (flexible) |
| Relationships | Foreign keys, JOINs | References / Embedding |
| Popular tools | Sequelize, Prisma, TypeORM | Mongoose |
| Query style | SQL-like | JSON-like |

## Interview Questions

- **Q: What is an ORM?**
  A: Object Relational Mapper — a library that maps database tables to JavaScript objects, allowing CRUD operations without writing raw SQL.

- **Q: What is an ODM?**
  A: Object Document Mapper — same concept as ORM but for document databases like MongoDB. Mongoose is the most popular ODM for Node.js.

- **Q: What is Mongoose?**
  A: A Node.js ODM library for MongoDB. It provides schema definition, validation, relationships, and a clean API for CRUD operations.

- **Q: What is the difference between SQL and NoSQL?**
  A: SQL databases store data in structured tables with fixed schemas. NoSQL (like MongoDB) stores data as flexible JSON-like documents.

- **Q: What is a Schema in Mongoose?**
  A: A schema defines the structure of documents in a MongoDB collection — field names, types, defaults, and validation rules.

- **Q: What is the difference between `save()` and `create()` in Mongoose?**
  A: `create()` is a shorthand — it creates a new document instance and immediately saves it. `save()` saves an already-created document instance.

- **Q: What is Prisma and why is it gaining popularity?**
  A: Prisma is a modern ORM with a type-safe query builder, auto-generated client, and schema migrations. It works with SQL and MongoDB and integrates well with TypeScript.
