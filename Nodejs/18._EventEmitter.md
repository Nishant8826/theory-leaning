# EventEmitter

**EventEmitter** is Node.js's built-in mechanism for **event-driven programming** — you can create custom events, listen to them, and trigger them.

Many core Node.js modules use EventEmitter internally: `http`, `fs` streams, `process`, `socket.io`.

---

## Real-World Example
When a user places an order in an e-commerce app:
- Emit `order:placed` event
- Listener 1: Send confirmation email
- Listener 2: Notify the warehouse
- Listener 3: Update analytics

All three happen independently — clean, decoupled code.

---

## Code Example

### Basic EventEmitter
```js
const EventEmitter = require("events");

const emitter = new EventEmitter();

// Register a listener (subscribe to an event)
emitter.on("greet", (name) => {
  console.log(`Hello, ${name}!`);
});

// Emit (trigger) the event
emitter.emit("greet", "Nishant"); // Hello, Nishant!
emitter.emit("greet", "Raj");     // Hello, Raj!
```

---

### Real-World: Order System
```js
const EventEmitter = require("events");

class OrderSystem extends EventEmitter {
  placeOrder(order) {
    console.log(`Order placed: ${order.id}`);
    this.emit("order:placed", order); // trigger event with data
  }
}

const orderSystem = new OrderSystem();

// listener 1 — send email
orderSystem.on("order:placed", (order) => {
  console.log(`📧 Confirmation email sent for order ${order.id}`);
});

// listener 2 — notify warehouse
orderSystem.on("order:placed", (order) => {
  console.log(`🏭 Warehouse notified for ${order.product}`);
});

// listener 3 — analytics
orderSystem.on("order:placed", (order) => {
  console.log(`📊 Analytics updated`);
});

// place an order — all 3 listeners react
orderSystem.placeOrder({ id: "ORD-001", product: "Laptop", amount: 50000 });
```

### Important EventEmitter Methods

```js
const emitter = new EventEmitter();

// on() — listen indefinitely
emitter.on("data", (chunk) => console.log("Data:", chunk));

// once() — listen only ONE time, then auto-remove
emitter.once("connected", () => console.log("Connected!"));

// off() — remove a specific listener
function handler(data) { console.log(data); }
emitter.on("event", handler);
emitter.off("event", handler); // remove it

// emit() — trigger an event with data
emitter.emit("data", { userId: 1 });

// listenerCount() — how many listeners for an event?
console.log(emitter.listenerCount("data")); // 1

// removeAllListeners() — remove all listeners for an event
emitter.removeAllListeners("data");

// eventNames() — list all registered event names
console.log(emitter.eventNames()); // ["data"]
```

---

### Error Event (Special Convention)
```js
const emitter = new EventEmitter();

// Always listen for 'error' — otherwise it throws
emitter.on("error", (err) => {
  console.error("Error handled:", err.message);
});

emitter.emit("error", new Error("Something went wrong!"));
// Without 'error' listener → Node.js crashes!
```

---

### Extending EventEmitter (OOP Pattern)
```js
const EventEmitter = require("events");

class Logger extends EventEmitter {
  log(message) {
    const logEntry = { time: new Date().toISOString(), message };
    this.emit("log", logEntry);  // emit on every log call
  }
}

const logger = new Logger();

// Listen for log events
logger.on("log", (entry) => {
  console.log(`[${entry.time}] ${entry.message}`);
});

// Write to file
logger.on("log", async (entry) => {
  const fs = require("fs/promises");
  await fs.appendFile("app.log", JSON.stringify(entry) + "\n");
});

logger.log("Server started");
logger.log("User logged in");
```

## EventEmitter Methods Summary

| Method | What it does |
|--------|------------|
| `.on(event, fn)` | Subscribe to an event (multiple calls allowed) |
| `.once(event, fn)` | Subscribe but auto-remove after first trigger |
| `.emit(event, ...args)` | Trigger an event, passing data to listeners |
| `.off(event, fn)` | Remove a specific listener |
| `.removeAllListeners(event)` | Remove all listeners for an event |
| `.listenerCount(event)` | Count how many listeners are registered |
| `.eventNames()` | Get all registered event names |
| `.setMaxListeners(n)` | Set max listeners (default: 10) |

## Interview Questions

- **Q: What is EventEmitter in Node.js?**
  A: A built-in class that implements the observer/pub-sub pattern. Objects can emit named events and register multiple listeners that react to those events.

- **Q: What is the difference between `.on()` and `.once()`?**
  A: `.on()` fires every time the event is emitted. `.once()` fires only once and then automatically removes itself.

- **Q: How do you remove a listener?**
  A: Call `.off(event, listenerFn)` or `.removeAllListeners(event)`. The listener function must be a named reference — anonymous functions can't be removed.

- **Q: What is the `error` event convention?**
  A: If an `error` event is emitted and no listener is registered for it, Node.js throws an unhandled error and crashes. Always add `emitter.on("error", handler)`.

- **Q: What is the default max listeners limit?**
  A: 10. Node.js warns if you add more (to detect memory leaks). Change with `emitter.setMaxListeners(n)`.

- **Q: Where is EventEmitter used in Node.js core?**
  A: Streams (`Readable`, `Writable`), HTTP server (connection events), `process` (exit, signal events), and child processes — all extend EventEmitter.
