# Authentication — JWT & Sessions

**Authentication** = Verifying who a user is ("Are you who you say you are?")
**Authorization** = Deciding what they're allowed to do ("Are you allowed to do this?")

Two main approaches in Node.js backends:
1. **Session-based** = Server stores your session, gives you a cookie.
2. **JWT-based** = Server gives you a signed token, verifies it on each request.

---

## Real-World Example
User logs into a food delivery app:
- Submits email + password.
- Server checks DB, verifies password with bcrypt.
- Server generates a JWT token and sends it to the mobile app.
- Every subsequent API call includes the token in the `Authorization` header.
- Server verifies the token — no DB lookup needed.

## Complete JWT Authentication Flow

### 1. Password Hashing with bcrypt

Never store plain-text passwords. Always hash them.

```js
const bcrypt = require("bcryptjs");

// REGISTER — hash before saving
async function hashPassword(plainPassword) {
  const salt = await bcrypt.genSalt(10); // cost factor
  const hash = await bcrypt.hash(plainPassword, salt);
  return hash;
}

// LOGIN — compare plain text with hash
async function verifyPassword(plainPassword, hashedPassword) {
  const isMatch = await bcrypt.compare(plainPassword, hashedPassword);
  return isMatch; // true or false
}
```

### 2. User Registration Route
```js
const express = require("express");
const bcrypt = require("bcryptjs");
const User = require("./models/User");

const router = express.Router();

router.post("/register", async (req, res) => {
  try {
    const { name, email, password } = req.body;

    // Check if user already exists
    const existingUser = await User.findOne({ email });
    if (existingUser) {
      return res.status(409).json({ error: "Email already in use" });
    }

    // Hash password
    const hashedPassword = await bcrypt.hash(password, 10);

    // Save user
    const user = await User.create({ name, email, password: hashedPassword });

    res.status(201).json({
      message: "User registered successfully",
      user: { id: user._id, name: user.name, email: user.email },
    });
  } catch (err) {
    res.status(500).json({ error: "Registration failed" });
  }
});
```

### 3. Login Route — Generate JWT
```js
const jwt = require("jsonwebtoken");
const { JWT_SECRET, JWT_EXPIRY } = require("../config");

router.post("/login", async (req, res) => {
  try {
    const { email, password } = req.body;

    // Find user
    const user = await User.findOne({ email });
    if (!user) {
      return res.status(401).json({ error: "Invalid credentials" });
    }

    // Verify password
    const isMatch = await bcrypt.compare(password, user.password);
    if (!isMatch) {
      return res.status(401).json({ error: "Invalid credentials" });
    }

    // Generate JWT
    const token = jwt.sign(
      { userId: user._id, email: user.email, role: user.role },
      JWT_SECRET,
      { expiresIn: "7d" }
    );

    res.json({
      message: "Login successful",
      token,
      user: { id: user._id, name: user.name, email: user.email },
    });
  } catch (err) {
    res.status(500).json({ error: "Login failed" });
  }
});
```

### 4. Auth Middleware — Protect Routes
```js
const jwt = require("jsonwebtoken");
const { JWT_SECRET } = require("../config");

function authenticate(req, res, next) {
  const authHeader = req.headers.authorization;

  if (!authHeader || !authHeader.startsWith("Bearer ")) {
    return res.status(401).json({ error: "No token provided" });
  }

  const token = authHeader.split(" ")[1]; // "Bearer <token>"

  try {
    const decoded = jwt.verify(token, JWT_SECRET);
    req.user = decoded; // attach user data to request
    next();
  } catch (err) {
    if (err.name === "TokenExpiredError") {
      return res.status(401).json({ error: "Token expired, please log in again" });
    }
    return res.status(401).json({ error: "Invalid token" });
  }
}

module.exports = authenticate;
```

---

### 5. Protecting Routes
```js
const authenticate = require("../middleware/authenticate");

// Public route — no auth needed
router.get("/public", (req, res) => res.json({ message: "Public data" }));

// Protected route — must have valid JWT
router.get("/profile", authenticate, async (req, res) => {
  const user = await User.findById(req.user.userId).select("-password");
  res.json(user);
});

// Role-based protection
function authorize(...roles) {
  return (req, res, next) => {
    if (!roles.includes(req.user.role)) {
      return res.status(403).json({ error: "Access denied" });
    }
    next();
  };
}

router.delete("/users/:id", authenticate, authorize("admin"), async (req, res) => {
  await User.findByIdAndDelete(req.params.id);
  res.json({ message: "User deleted" });
});
```

## JWT Structure

A JWT looks like: `xxxxx.yyyyy.zzzzz`

| Part | Content |
|------|---------|
| Header (`xxxxx`) | Algorithm type (HS256) |
| Payload (`yyyyy`) | Data (userId, role, expiry) — **base64, NOT encrypted!** |
| Signature (`zzzzz`) | HMAC hash — verifies authenticity |

> ⚠️ The payload is public — anyone can decode it. Never store passwords or secrets in JWT payload.

## Interview Questions

- **Q: What is JWT?**
  A: JSON Web Token — a stateless, signed token containing a payload. Used for authentication without server-side session storage.

- **Q: Why do we hash passwords with bcrypt?**
  A: Hashing is one-way — even if the DB is stolen, plain passwords can't be recovered. bcrypt adds a salt to prevent rainbow table attacks.

- **Q: What does `jwt.verify()` do?**
  A: Verifies the token's signature using the secret key. Returns the decoded payload if valid, or throws if invalid or expired.

- **Q: What is the difference between authentication and authorization?**
  A: Authentication = verifying identity (who are you?). Authorization = verifying permissions (what can you do?).

- **Q: How do you protect a route in Express with JWT?**
  A: Use middleware that extracts the `Authorization` header, calls `jwt.verify()`, and attaches the decoded user to `req.user` before calling `next()`.

- **Q: What is the biggest security concern with JWT?**
  A: JWTs can't be invalidated before expiry. Use short expiry times (15 min), refresh tokens, and a token blacklist if immediate logout is needed.

- **Q: What is `bcrypt.genSalt()` and why use it?**
  A: Generates a random salt — unique data added to each password before hashing. Ensures identical passwords produce different hashes.
