# Node.js Basics

**Node.js** is a runtime environment that lets you run **JavaScript outside the browser** — on a server or your own computer.

Before Node.js, JavaScript could only run inside a web browser. Node.js changed that by using Chrome's **V8 engine** (written in C++) to execute JS on a server.

---

## Key Characteristics of Node.js

| Feature | Description |
|---------|-------------|
| **Single-threaded** | Uses one main thread (but handles concurrency via Event Loop) |
| **Non-blocking I/O** | Doesn't wait for tasks (file read, DB query) to finish before doing next task |
| **Event-driven** | Uses events and callbacks to handle async operations |
| **V8 Engine** | Uses Google Chrome's V8 JavaScript engine (written in C++) |
| **npm ecosystem** | Access to 2M+ reusable packages via npm |

## Code Example

### Your First Node.js Program
```js
// hello.js
console.log("Hello from Node.js!");
console.log("Node version:", process.version);
console.log("Platform:", process.platform);
```

Run it:
```bash
node hello.js
```

### A Basic HTTP Server
```js
// server.js
const http = require("http");

const server = http.createServer((req, res) => {
  res.writeHead(200, { "Content-Type": "text/plain" });
  res.end("Hello World from Node.js!\n");
});

server.listen(3000, () => {
  console.log("Server running at http://localhost:3000");
});
```

Run it:
```bash
node server.js
```

## Why Node.js for Backends?

### Traditional server (blocking)
```
Request 1 → Reads file → WAIT → Responds
Request 2 → WAITS for Request 1 to finish ← Problem!
```

### Node.js server (non-blocking)
```
Request 1 → Reads file → (doesn't wait) → handles next request
When file is ready → callback fires → responds to Request 1
```

This is why Node.js can handle **thousands of concurrent requests** without slowing down.

## Node.js Architecture Overview

```
Your Code (JavaScript)
        ↓
    V8 Engine (compiles JS → machine code)
        ↓
    Node.js APIs (fs, http, path, crypto...)
        ↓
    libuv (C++ library — handles async I/O, Thread Pool, Event Loop)
        ↓
    Operating System
```

## npm — Node Package Manager

npm comes bundled with Node.js. It's used to install, share, and manage packages (libraries).

```bash
npm init -y          # create package.json
npm install express  # install a package
npm install          # install all packages from package.json
npm run start        # run a script defined in package.json
```

**`package.json`** = the recipe card for your project — lists all dependencies and scripts.

## Interview Questions

- **Q: What is Node.js?**
  A: A JavaScript runtime built on Chrome's V8 engine that allows JavaScript to run on the server side, outside the browser.

- **Q: Why is Node.js called non-blocking?**
  A: Node.js doesn't wait for slow operations (file I/O, DB queries, network). It delegates them and continues processing other requests. When done, a callback fires.

- **Q: What is the V8 engine?**
  A: Google's open-source JavaScript engine (written in C++) used in Chrome and Node.js. It compiles JS to native machine code for fast execution.

- **Q: What is libuv?**
  A: A C++ library that powers Node.js's event loop, async I/O, and thread pool. It handles OS-level operations like file system and networking.

- **Q: What is npm?**
  A: Node Package Manager — a CLI tool for installing and managing third-party packages. It comes bundled with Node.js.

- **Q: Is Node.js single-threaded?**
  A: Yes, the main execution is single-threaded. However, libuv provides a thread pool for heavy tasks (file I/O, crypto). The event loop manages async results.

- **Q: What kind of applications is Node.js best suited for?**
  A: Real-time apps (chat, gaming), REST APIs, microservices, streaming apps, and I/O-heavy apps. It's not ideal for CPU-intensive tasks.
