# HTTP Module

## Simple Explanation
The **`http` module** is a built-in Node.js module that lets you create a basic **web server** that can receive HTTP requests and send back responses — without installing any third-party library.

Think of it as the foundation. Express.js is built on top of the `http` module — it's just a nicer, more powerful wrapper.

---

## Real-World Example
Your browser goes to `http://localhost:3000` → it sends an HTTP request → your Node.js server receives it → processes it → sends back HTML or JSON.

---

## Code Examples

### Creating a Basic Server
```js
const http = require("http");

const server = http.createServer((req, res) => {
  // req = incoming request object
  // res = outgoing response object

  res.writeHead(200, { "Content-Type": "text/plain" });
  res.end("Hello World!\n");
});

server.listen(3000, () => {
  console.log("Server running at http://localhost:3000");
});
```

---

### Routing by URL and Method
```js
const http = require("http");

const server = http.createServer((req, res) => {
  const { url, method } = req;

  // GET /
  if (method === "GET" && url === "/") {
    res.writeHead(200, { "Content-Type": "text/html" });
    res.end("<h1>Welcome Home!</h1>");
  }
  // GET /about
  else if (method === "GET" && url === "/about") {
    res.writeHead(200, { "Content-Type": "application/json" });
    res.end(JSON.stringify({ page: "About", version: "1.0" }));
  }
  // POST /data
  else if (method === "POST" && url === "/data") {
    let body = "";

    req.on("data", (chunk) => {
      body += chunk.toString(); // collect incoming data chunks
    });

    req.on("end", () => {
      const parsed = JSON.parse(body);
      console.log("Received:", parsed);
      res.writeHead(201, { "Content-Type": "application/json" });
      res.end(JSON.stringify({ received: true, data: parsed }));
    });
  }
  // 404 Not Found
  else {
    res.writeHead(404, { "Content-Type": "application/json" });
    res.end(JSON.stringify({ error: "Route not found" }));
  }
});

server.listen(3000, () => console.log("Server on port 3000"));
```

---

### Reading Request Properties
```js
const server = http.createServer((req, res) => {
  console.log("Method:", req.method);           // GET, POST, PUT, DELETE
  console.log("URL:", req.url);                 // /api/users
  console.log("Headers:", req.headers);          // { host, content-type, ... }
  console.log("Auth:", req.headers.authorization);

  res.end("OK");
});
```

---

### Making an HTTP Request from Node.js
```js
const https = require("https");

https.get("https://jsonplaceholder.typicode.com/posts/1", (res) => {
  let data = "";

  res.on("data", (chunk) => {
    data += chunk;
  });

  res.on("end", () => {
    const post = JSON.parse(data);
    console.log("Title:", post.title);
  });
}).on("error", (err) => {
  console.error("Error:", err.message);
});
```

> 💡 In modern Node.js (v18+), you can use the built-in `fetch()` API instead of `https.get()`.

## Why Use Express Instead of Pure `http`?

| Feature | `http` module | Express.js |
|---------|-------------|-----------|
| Routing | Manual `if/else` per URL | Clean `app.get()`, `app.post()` etc. |
| Body parsing | Manual chunk collection | `express.json()` middleware |
| Middleware | None built-in | Full middleware support |
| Error handling | Manual | Built-in + custom handlers |
| Code verbosity | High | Low |

The `http` module is great for learning. In production, use **Express.js**.

## Interview Questions

- **Q: What is the `http` module in Node.js?**
  A: A built-in module for creating HTTP servers and making HTTP requests. It's the low-level foundation Express.js is built upon.

- **Q: What is `http.createServer()`?**
  A: A method that creates an HTTP server. It takes a callback with `(req, res)` — `req` is the incoming request, `res` is used to send a response.

- **Q: What is the difference between `res.write()` and `res.end()`?**
  A: `res.write()` sends a chunk of the response body but keeps the connection open. `res.end()` sends the final chunk and closes the connection.

- **Q: How do you parse a POST body in the native `http` module?**
  A: Listen to `req.on("data", chunk => ...)` to collect chunks and `req.on("end", ...)` to parse the full body. Express's `express.json()` automates this.

- **Q: Why do developers prefer Express over the raw `http` module?**
  A: Express provides clean routing, built-in middleware support, and a much simpler API — reducing boilerplate significantly.

- **Q: What is `https` vs `http` in Node.js?**
  A: `https` is the secure version — requires a TLS/SSL certificate. In production, always use HTTPS. Locally, HTTP is fine.
