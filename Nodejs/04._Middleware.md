# Middleware

## Simple Explanation
A **middleware** is a function that runs **between** a request arriving and the final response being sent. It has access to the `request` object, `response` object, and a `next()` function.

Think of middleware like security checkpoints at an airport:
- You (request) enter the terminal.
- You pass through **check-in** → **security scan** → **passport control** → **boarding gate** (final handler).
- Each checkpoint is a middleware. If one stops you (no `next()`), you don't reach the gate.

---

## Real-World Example
A protected API endpoint:
```
Request → Check if JSON → Log request → Verify JWT token → Handle route
```
Each step is a middleware that either continues (calls `next()`) or stops (sends a response).

---

## How Middleware Works

```
request → middleware1 → middleware2 → middleware3 → final route handler
```

- Each middleware can **read/modify** `req` and `res`.
- It must either call `next()` to pass control forward, or send a response to end the cycle.
- ⚠️ If a middleware calls neither `next()` nor `res.send()`, the request **hangs forever**.

## Code Example

### Custom Middleware from Scratch
```js
const express = require("express");
const app = express();

// Custom logger middleware
function logger(req, res, next) {
  console.log(`[${new Date().toISOString()}] ${req.method} ${req.url}`);
  next(); // pass control to the next middleware
}

// Custom auth middleware
function checkAuth(req, res, next) {
  const token = req.headers.authorization;
  if (!token) {
    return res.status(401).json({ error: "Unauthorized" });
    // no next() — stops here and sends response
  }
  next(); // token exists → continue
}

app.use(logger); // runs on every request

app.get("/public", (req, res) => {
  res.json({ message: "Public route — no auth needed" });
});

app.get("/protected", checkAuth, (req, res) => {
  res.json({ message: "Private route — auth passed!" });
});

app.listen(3000, () => console.log("Server on port 3000"));
```

## Types of Middleware

### 1. Application-level Middleware
Runs for every request. Mounted with `app.use()`.

```js
app.use(express.json());                   // parse JSON bodies
app.use(express.urlencoded({ extended: false })); // parse form data

// Custom logger for all routes
app.use((req, res, next) => {
  console.log(`${req.method} ${req.url}`);
  next();
});
```

### 2. Router-level Middleware
Runs only for specific routes or groups. Mounted with `router.use()`.

```js
const router = express.Router();

// Only runs for /api/users routes
router.use((req, res, next) => {
  console.log("User route accessed");
  next();
});

router.get("/", (req, res) => res.json({ users: [] }));
app.use("/api/users", router);
```

### 3. Built-in Middleware (Express)
Functions already included in Express.

```js
app.use(express.static("public"));              // serve static files
app.use(express.json());                        // parse JSON request bodies
app.use(express.urlencoded({ extended: false })); // parse URL-encoded form data
```

### 4. Third-party Middleware
Installed via npm.

```js
const morgan = require("morgan");  // HTTP request logger
const cors = require("cors");      // Cross-Origin Resource Sharing
const helmet = require("helmet");  // Security headers

app.use(morgan("dev"));
app.use(cors());
app.use(helmet());
```

### 5. Error-handling Middleware
Has **4 parameters** `(err, req, res, next)`. Must be placed **after all routes**.

```js
// ✅ All routes above
app.get("/", (req, res) => {
  throw new Error("Something went wrong!");
});

// Error handler — placed LAST
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).json({
    error: "Internal Server Error",
    message: err.message,
  });
});
```

## Middleware Execution Order Matters!

```js
app.use(express.json());    // ✅ Parse body first
app.use(logger);             // ✅ Then log
app.use(checkAuth);          // ✅ Then authenticate
app.get("/data", handler);   // ✅ Then handle route
app.use(errorHandler);       // ✅ Error handler ALWAYS last
```

## Interview Questions

- **Q: What is middleware in Express?**
  A: A function with access to `req`, `res`, and `next`. It can modify request/response, end the cycle, or call `next()` to pass to the next handler.

- **Q: What happens if middleware doesn't call `next()`?**
  A: The request will hang indefinitely — the response is never sent. Always either call `next()` or send a response.

- **Q: What are the types of middleware in Express?**
  A: Application-level, Router-level, Built-in, Third-party, and Error-handling middleware.

- **Q: How is error-handling middleware different from regular middleware?**
  A: It has 4 parameters `(err, req, res, next)` instead of 3. It must be defined after all routes and other middleware.

- **Q: What is `app.use()` vs `app.get()`?**
  A: `app.use()` applies middleware to ALL request methods and paths (unless specified). `app.get()` handles only GET requests on a specific path.

- **Q: What is CORS middleware used for?**
  A: It adds headers that allow or restrict browsers from making cross-origin requests (e.g., your React frontend calling a different backend domain).

- **Q: Can middleware be applied to specific routes only?**
  A: Yes. Pass the middleware as the second argument: `app.get('/protected', checkAuth, handler)`.
