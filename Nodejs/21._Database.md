# Database Connection (MongoDB with Mongoose)

## Simple Explanation
A **database** is where your application permanently stores data — user accounts, orders, posts, etc.

**Mongoose** is an ODM (Object Document Mapper) that provides a clean API to connect to **MongoDB** and work with data using JavaScript objects.

Think of Mongoose as a **form with fields**:
- The form template = Schema (defines what data looks like)
- A filled-out form = Document (one record in the database)
- A folder of forms = Collection (all records of that type)

---

## Real-World Example
A user registration system:
- When a user registers → save their name, email, password to MongoDB via Mongoose
- When they log in → query MongoDB to find their record

## Code Examples

### Connecting to MongoDB
```js
// db.js
const mongoose = require("mongoose");

async function connectDB() {
  try {
    await mongoose.connect(process.env.MONGODB_URI, {
      // options not needed in Mongoose 7+, but here for reference
    });
    console.log("✅ MongoDB connected successfully");
  } catch (err) {
    console.error("❌ MongoDB connection failed:", err.message);
    process.exit(1); // kill the server if DB fails to connect
  }
}

module.exports = connectDB;
```

```js
// index.js
require("dotenv").config();
const connectDB = require("./db");
const app = require("./app");

connectDB().then(() => {
  app.listen(process.env.PORT || 3000, () => {
    console.log("Server running");
  });
});
```

### Defining a Mongoose Schema & Model
```js
// models/User.js
const mongoose = require("mongoose");

const userSchema = new mongoose.Schema({
  name: {
    type: String,
    required: [true, "Name is required"],
    trim: true,
    maxLength: [50, "Name cannot exceed 50 characters"],
  },
  email: {
    type: String,
    required: [true, "Email is required"],
    unique: true,
    lowercase: true,
    match: [/\S+@\S+\.\S+/, "Invalid email format"],
  },
  password: {
    type: String,
    required: true,
    minLength: [6, "Password must be at least 6 characters"],
    select: false, // don't return password in queries by default
  },
  role: {
    type: String,
    enum: ["user", "admin"],
    default: "user",
  },
  isActive: {
    type: Boolean,
    default: true,
  },
  createdAt: {
    type: Date,
    default: Date.now,
  },
});

// Export the model
const User = mongoose.model("User", userSchema);
module.exports = User;
```

### CRUD Operations with Mongoose

```js
const User = require("./models/User");

// CREATE
async function createUser() {
  const user = await User.create({
    name: "Nishant",
    email: "nishant@example.com",
    password: "hashedPassword123",
  });
  console.log("Created:", user._id);
}

// READ — find all
const allUsers = await User.find();
const activeUsers = await User.find({ isActive: true });

// READ — find one
const user = await User.findById("64abc123...");
const userByEmail = await User.findOne({ email: "nishant@example.com" });

// SELECT specific fields
const names = await User.find().select("name email -_id");

// SORT, LIMIT, SKIP (pagination)
const page = 1, limit = 10;
const users = await User.find()
  .sort({ createdAt: -1 })       // most recent first
  .limit(limit)
  .skip((page - 1) * limit);

// UPDATE
await User.findByIdAndUpdate(
  "64abc123...",
  { name: "Nishant Updated" },
  { new: true } // return updated document
);

// DELETE
await User.findByIdAndDelete("64abc123...");
await User.deleteMany({ isActive: false }); // delete multiple
```

### Mongoose Relationships (Population)
```js
// Post model references User
const postSchema = new mongoose.Schema({
  title: String,
  content: String,
  author: {
    type: mongoose.Schema.Types.ObjectId,
    ref: "User",  // reference to the User model
    required: true,
  },
  createdAt: { type: Date, default: Date.now },
});

const Post = mongoose.model("Post", postSchema);

// Fetch posts and populate author details (JOIN equivalent)
const posts = await Post.find()
  .populate("author", "name email")  // only get name and email from User
  .sort({ createdAt: -1 });

console.log(posts[0].author.name); // "Nishant"
```

### Mongoose Virtuals & Hooks (Middleware)
```js
// Virtual — computed field not stored in DB
userSchema.virtual("fullName").get(function () {
  return `${this.firstName} ${this.lastName}`;
});

// Pre-save hook — auto-hash password before saving
const bcrypt = require("bcryptjs");

userSchema.pre("save", async function (next) {
  if (!this.isModified("password")) return next(); // skip if not changed
  this.password = await bcrypt.hash(this.password, 10);
  next();
});
```

---

### Connection Events
```js
mongoose.connection.on("connected", () => console.log("DB connected"));
mongoose.connection.on("error", (err) => console.error("DB error:", err));
mongoose.connection.on("disconnected", () => console.log("DB disconnected"));

// Graceful shutdown
process.on("SIGINT", async () => {
  await mongoose.connection.close();
  process.exit(0);
});
```

## Interview Questions

- **Q: What is Mongoose?**
  A: A Node.js ODM for MongoDB. It provides schema definition, validation, model creation, and a chainable query API.

- **Q: What is a Mongoose Schema?**
  A: A blueprint defining the structure, types, defaults, and validation rules for documents in a MongoDB collection.

- **Q: What is `populate()` in Mongoose?**
  A: It replaces ObjectId references with actual document data — like a JOIN in SQL. Used for relationships between collections.

- **Q: What is the difference between `save()` and `create()` in Mongoose?**
  A: `create()` is a shorthand — it constructs and immediately saves. `save()` is called on an existing document instance.

- **Q: What are Mongoose hooks (middleware)?**
  A: Functions that run before (`pre`) or after (`post`) operations like `save`, `findOne`, `remove`. Used for tasks like auto-hashing passwords.

- **Q: What is `select: false` on a schema field?**
  A: It excludes that field from query results by default. Used to protect sensitive fields like passwords. Restore with `.select("+password")`.

- **Q: What happens if you don't handle a MongoDB connection failure?**
  A: The server starts but any DB operation will fail silently or throw. Always call `process.exit(1)` if the initial connection fails.
