# Routing in Express

**Routing** = Defining how your application responds to different URLs and HTTP methods.

---

## Real-World Example
A blog app might have routes like:
- `GET /posts` — get all posts
- `GET /posts/:id` — get one post
- `POST /posts` — create a new post
- `PUT /posts/:id` — update a post
- `DELETE /posts/:id` — delete a post

---

## Basic Routing

```js
const express = require("express");
const app = express();
app.use(express.json());

// GET
app.get("/", (req, res) => res.send("Home page"));

// POST
app.post("/login", (req, res) => {
  const { email, password } = req.body;
  res.json({ message: `Login attempted for ${email}` });
});

// PUT
app.put("/users/:id", (req, res) => {
  res.json({ message: `User ${req.params.id} updated` });
});

// PATCH (partial update)
app.patch("/users/:id", (req, res) => {
  res.json({ updated: req.body });
});

// DELETE
app.delete("/users/:id", (req, res) => {
  res.json({ message: `User ${req.params.id} deleted` });
});

app.listen(3000);
```

## Express Router — Organizing Routes into Files

As your app grows, keeping all routes in `index.js` becomes messy. Use **Express Router** to split routes into separate files.

### File Structure:
```
project/
├── routes/
│   ├── userRoutes.js
│   ├── productRoutes.js
│   └── authRoutes.js
├── index.js
└── package.json
```

### `routes/userRoutes.js`
```js
const express = require("express");
const router = express.Router();

router.get("/", (req, res) => {
  res.json({ users: ["Nishant", "Raj"] });
});

router.get("/:id", (req, res) => {
  res.json({ userId: req.params.id });
});

router.post("/", (req, res) => {
  const { name, email } = req.body;
  res.status(201).json({ message: "User created", user: { name, email } });
});

module.exports = router;
```

### `index.js` — Mounting routers
```js
const express = require("express");
const app = express();
app.use(express.json());

const userRoutes = require("./routes/userRoutes");
const productRoutes = require("./routes/productRoutes");

// Mount with a prefix
app.use("/api/users", userRoutes);     // /api/users, /api/users/:id
app.use("/api/products", productRoutes); // /api/products, etc.

// 404 handler
app.use((req, res) => {
  res.status(404).json({ error: "Route not found" });
});

app.listen(3000, () => console.log("Server on 3000"));
```

## Route Parameters vs Query Strings

```js
// Route parameter — part of the URL path
// GET /products/42
app.get("/products/:id", (req, res) => {
  console.log(req.params.id); // "42"
  res.json({ productId: req.params.id });
});

// Multiple parameters
// GET /users/5/orders/10
app.get("/users/:userId/orders/:orderId", (req, res) => {
  console.log(req.params); // { userId: '5', orderId: '10' }
});

// Query string — key=value after ?
// GET /search?term=laptop&limit=10&page=2
app.get("/search", (req, res) => {
  const { term, limit = 10, page = 1 } = req.query;
  res.json({ term, limit: Number(limit), page: Number(page) });
});
```

## Route-level Middleware

```js
function isAuthenticated(req, res, next) {
  const token = req.headers.authorization;
  if (!token) return res.status(401).json({ error: "Unauthorized" });
  // Verify token...
  next();
}

// Apply only to these routes
router.get("/profile", isAuthenticated, (req, res) => {
  res.json({ profile: "User data here" });
});

router.delete("/:id", isAuthenticated, (req, res) => {
  res.json({ message: "Deleted" });
});
```

---

## `app.route()` — Cleaner Route Chaining
```js
app.route("/users/:id")
  .get((req, res) => res.json({ userId: req.params.id }))
  .put((req, res) => res.json({ updated: req.body }))
  .delete((req, res) => res.json({ deleted: req.params.id }));
```

## Interview Questions

- **Q: What is Express Router?**
  A: A mini-application with routing capabilities. It allows you to group routes by feature (users, products) into separate files and mount them with a prefix.

- **Q: What is the difference between `req.params` and `req.query`?**
  A: `req.params` = values from URL path (`:id`). `req.query` = values from query string (`?key=value`).

- **Q: How do you structure routes in a large Express app?**
  A: Use `express.Router()` to create separate route files per feature. Mount them in `index.js` with `app.use("/api/resource", router)`.

- **Q: How do you handle route-specific middleware?**
  A: Pass the middleware function as an argument before the route handler: `router.get("/profile", authMiddleware, handler)`.

- **Q: What is `app.route()`?**
  A: A method to chain multiple HTTP methods for the same path — cleaner than repeating the path in `app.get()`, `app.put()`, `app.delete()`.

- **Q: What order should routes be defined in Express?**
  A: Specific routes before wildcard routes. Error-handling middleware last. The 404 handler must be last.
