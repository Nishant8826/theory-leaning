# Express.js Basics

## Simple Explanation
**Express.js** is the most popular Node.js web framework. It makes building REST APIs and web servers fast and simple by providing clean routing, middleware support, and a minimal but powerful API.

Think of Express like a **well-organized kitchen**:
- Node.js `http` module = raw ingredients (low-level, lots of manual work)
- Express.js = a fully equipped kitchen where you can cook faster (routing, middleware, error handling — all built-in)

---

## Real-World Example
Almost every Node.js backend you've seen (LinkedIn, Uber, WhatsApp Web's API) uses Express or a framework built on it. Your typical flow:

```
Browser/Mobile → HTTP Request → Express Route → Controller → DB → Response
```

## Code Examples

### Minimal Express Server
```js
const express = require("express");
const app = express();
const PORT = 3000;

app.use(express.json()); // parse JSON request bodies

app.get("/", (req, res) => {
  res.json({ message: "Hello from Express!" });
});

app.listen(PORT, () => {
  console.log(`Server running at http://localhost:${PORT}`);
});
```

---

### Route Parameters & Query Strings
```js
// Route parameter (:id)
app.get("/users/:id", (req, res) => {
  const { id } = req.params;       // /users/42 → id = "42"
  res.json({ userId: id });
});

// Multiple params
app.get("/posts/:postId/comments/:commentId", (req, res) => {
  console.log(req.params); // { postId: '1', commentId: '5' }
  res.json(req.params);
});

// Query strings
app.get("/search", (req, res) => {
  const { q, limit = 10, page = 1 } = req.query;
  // /search?q=nodejs&limit=5&page=2
  res.json({ query: q, limit, page });
});
```

---

### Handling Request Body
```js
app.use(express.json());                        // for JSON bodies
app.use(express.urlencoded({ extended: false })); // for form data

app.post("/users", (req, res) => {
  const { name, email, password } = req.body;

  if (!name || !email) {
    return res.status(400).json({ error: "Name and email are required" });
  }

  // Save to DB...
  res.status(201).json({ message: "User created", user: { name, email } });
});
```

### Full CRUD API Example
```js
const express = require("express");
const app = express();
app.use(express.json());

let users = [
  { id: 1, name: "Nishant", email: "n@n.com" },
  { id: 2, name: "Raj", email: "r@r.com" },
];

// GET all users
app.get("/users", (req, res) => {
  res.json(users);
});

// GET single user
app.get("/users/:id", (req, res) => {
  const user = users.find((u) => u.id === parseInt(req.params.id));
  if (!user) return res.status(404).json({ error: "User not found" });
  res.json(user);
});

// POST create user
app.post("/users", (req, res) => {
  const newUser = { id: users.length + 1, ...req.body };
  users.push(newUser);
  res.status(201).json(newUser);
});

// PUT update user
app.put("/users/:id", (req, res) => {
  const index = users.findIndex((u) => u.id === parseInt(req.params.id));
  if (index === -1) return res.status(404).json({ error: "User not found" });

  users[index] = { ...users[index], ...req.body };
  res.json(users[index]);
});

// DELETE user
app.delete("/users/:id", (req, res) => {
  users = users.filter((u) => u.id !== parseInt(req.params.id));
  res.json({ message: "User deleted" });
});

app.listen(3000, () => console.log("CRUD API on port 3000"));
```

## `req` and `res` Cheat Sheet

| Property / Method | Description |
|-------------------|-------------|
| `req.params` | URL route parameters (`/users/:id`) |
| `req.query` | Query string (`/search?q=node`) |
| `req.body` | Request body (requires `express.json()`) |
| `req.headers` | HTTP headers |
| `req.method` | HTTP method (GET, POST, etc.) |
| `req.url` | The request URL |
| `res.json(data)` | Send JSON response |
| `res.send(text)` | Send text/HTML response |
| `res.status(code)` | Set HTTP status code |
| `res.status(404).json({})` | Status + JSON together |
| `res.redirect(url)` | Redirect to another URL |
| `res.sendFile(path)` | Send a static file |

## Interview Questions

- **Q: What is Express.js?**
  A: A minimal, fast, and flexible Node.js web framework for building REST APIs and web apps. It simplifies routing, middleware, and HTTP handling.

- **Q: What is `app.use()` vs `app.get()`?**
  A: `app.use()` applies middleware to all HTTP methods and routes (unless scoped). `app.get()` handles only GET requests for a specific path.

- **Q: What is `req.params` vs `req.query`?**
  A: `req.params` extracts values from URL path segments (`:id`). `req.query` extracts values from the query string (`?name=value`).

- **Q: Why do you need `express.json()` middleware?**
  A: Express doesn't parse JSON request bodies by default. `express.json()` middleware reads the body and populates `req.body`.

- **Q: What does `res.status(201).json({})` do?**
  A: Sets the HTTP status code to 201 (Created) and sends a JSON response body. Chaining makes it cleaner.

- **Q: How do you handle 404 errors in Express?**
  A: Add a wildcard route at the end: `app.use((req, res) => res.status(404).json({ error: "Not found" }))`.
