# File System (fs Module)

The **`fs` (File System) module** is a built-in Node.js module that lets you **read, write, update, delete, and manage files** on your computer — all from JavaScript.

---

## Real-World Example
- Reading a JSON config file when your server starts.
- Writing logs to a `.log` file.
- Uploading and saving user-uploaded images to a folder.
- Reading CSV data to process records.

---

## Setup
```js
const fs = require("fs");           // Callback-based (older)
const fs = require("fs/promises");  // Promise-based (modern, recommended)
```

---

## Code Examples

### Reading a File (Async)
```js
const fs = require("fs");

// ✅ Non-blocking (async) — recommended
fs.readFile("data.txt", "utf8", (err, data) => {
  if (err) {
    console.error("Error:", err.message);
    return;
  }
  console.log("File content:", data);
});
console.log("This runs BEFORE file is read!"); // prints first

// ❌ Blocking (sync) — avoid in servers
const data = fs.readFileSync("data.txt", "utf8");
console.log(data);
```

### Reading with Promises (Modern — Recommended)
```js
const fs = require("fs/promises");

async function readConfig() {
  try {
    const data = await fs.readFile("config.json", "utf8");
    const config = JSON.parse(data);
    console.log("Config:", config);
  } catch (err) {
    console.error("Could not read config:", err.message);
  }
}

readConfig();
```

---

### Writing a File
```js
const fs = require("fs/promises");

async function writeLogs() {
  const log = `[${new Date().toISOString()}] User logged in\n`;

  // Overwrite file (creates if doesn't exist)
  await fs.writeFile("app.log", log);

  // Append to file (won't overwrite existing content)
  await fs.appendFile("app.log", `[${new Date().toISOString()}] User logged out\n`);

  console.log("Logs written!");
}

writeLogs();
```

---

### Checking if File Exists
```js
const fs = require("fs/promises");

async function checkFile(filePath) {
  try {
    await fs.access(filePath);
    console.log("File exists!");
  } catch {
    console.log("File does NOT exist");
  }
}

checkFile("data.txt");
```

---

### Deleting a File
```js
const fs = require("fs/promises");

async function deleteFile(path) {
  try {
    await fs.unlink(path);
    console.log("File deleted!");
  } catch (err) {
    console.error("Delete failed:", err.message);
  }
}
```

---

### Working with Directories
```js
const fs = require("fs/promises");

async function manageDirectories() {
  // Create a directory
  await fs.mkdir("uploads", { recursive: true }); // recursive: won't fail if exists

  // List contents of a directory
  const files = await fs.readdir(".");
  console.log("Files:", files);

  // Remove a directory
  await fs.rmdir("uploads"); // must be empty
  // OR remove with contents:
  await fs.rm("uploads", { recursive: true });
}
```

---

### Rename / Move a File
```js
await fs.rename("old-name.txt", "new-name.txt");
```

---

### Getting File Info
```js
const stats = await fs.stat("data.txt");

console.log("File size:", stats.size, "bytes");
console.log("Is file:", stats.isFile());
console.log("Is directory:", stats.isDirectory());
console.log("Modified:", stats.mtime);
```

## Sync vs Async Methods

| | Async (recommended) | Sync (blocking) |
|--|--------|--------|
| Example | `fs.readFile()` | `fs.readFileSync()` |
| Blocks server | ❌ No | ✅ YES — bad in servers |
| Use in | Servers, APIs | CLI scripts only |

## Interview Questions

- **Q: What is the `fs` module in Node.js?**
  A: A built-in module for interacting with the file system — reading, writing, deleting files, and managing directories.

- **Q: What is the difference between `readFile` and `readFileSync`?**
  A: `readFile` is asynchronous (non-blocking) and takes a callback. `readFileSync` is synchronous (blocking) — it freezes the event loop until the file is read.

- **Q: When should you use `fs/promises`?**
  A: When you want to use `async/await` with file operations instead of callbacks. It's the modern, recommended approach.

- **Q: What does `fs.appendFile` do?**
  A: It adds content to the end of a file without overwriting existing content. Creates the file if it doesn't exist.

- **Q: What is `fs.stat()` used for?**
  A: It retrieves information about a file — size, creation time, modification time, whether it's a file or directory.

- **Q: How do you create nested directories safely?**
  A: Use `fs.mkdir(path, { recursive: true })` — it creates all parent directories if they don't exist and won't error if the directory already exists.
