# Streams & Buffers

**Streams** = A way to handle data in **small chunks** instead of loading everything into memory at once.

**Buffers** = Temporary memory storage for raw binary data while it's being transferred.

---

## Real-World Example
- **Uploading a 1GB file** → Stream it, don't load it all into memory.
- **Serving video** → Stream video chunks to the browser progressively.
- **Reading large log files** → Read and process line-by-line with a readable stream.

---

## What is a Buffer?

A **Buffer** is a fixed-size chunk of memory outside the V8 heap, used to store raw binary data temporarily while it's being transferred.

```js
// Creating a buffer
const buf = Buffer.from("Hello, Node.js!");

console.log(buf);             // <Buffer 48 65 6c 6c 6f ...> (raw bytes)
console.log(buf.toString());  // "Hello, Node.js!"
console.log(buf.length);      // 15 (bytes)

// Allocate an empty buffer of size 10
const empty = Buffer.alloc(10);
console.log(empty); // <Buffer 00 00 00 00 00 00 00 00 00 00>
```

## 4 Types of Streams in Node.js

### 1. Readable Stream — Read data from a source
```js
const fs = require("fs");

const readable = fs.createReadStream("bigfile.txt", { encoding: "utf8" });

readable.on("data", (chunk) => {
  console.log("Got chunk:", chunk.length, "bytes");
});

readable.on("end", () => {
  console.log("File reading done!");
});

readable.on("error", (err) => {
  console.error("Error:", err.message);
});
```

### 2. Writable Stream — Write data to a destination
```js
const writable = fs.createWriteStream("output.txt");

writable.write("Hello, ");
writable.write("World!");
writable.end(); // signal we're done writing

writable.on("finish", () => {
  console.log("All data written to file!");
});
```

### 3. Duplex Stream — Both readable and writable
Used in TCP sockets, WebSocket connections.
```js
const net = require("net");

const server = net.createServer((socket) => {
  // socket is a Duplex stream
  socket.on("data", (data) => {
    console.log("Received:", data.toString());
    socket.write("Echo: " + data); // write back
  });
});

server.listen(8080, () => console.log("TCP server on 8080"));
```

### 4. Transform Stream — Reads, transforms, then writes
```js
const { Transform } = require("stream");

const upperCaseTransform = new Transform({
  transform(chunk, encoding, callback) {
    this.push(chunk.toString().toUpperCase());
    callback();
  },
});

// Pipe: read file → transform (uppercase) → write to output
fs.createReadStream("input.txt")
  .pipe(upperCaseTransform)
  .pipe(fs.createWriteStream("output.txt"));
```

## The `pipe()` Method — The Star of Streams

`pipe()` connects a readable stream to a writable stream. It handles backpressure automatically.

```js
// ✅ Efficient file copy using streams
const fs = require("fs");

const readable = fs.createReadStream("source.txt");
const writable = fs.createWriteStream("destination.txt");

readable.pipe(writable);           // connect them
writable.on("finish", () => {
  console.log("File copied successfully!");
});

// ✅ Serve a large file via HTTP using streams (memory-efficient)
const http = require("http");

http.createServer((req, res) => {
  const fileStream = fs.createReadStream("./large-video.mp4");
  fileStream.pipe(res); // stream file directly to HTTP response
}).listen(3000);
```

## Why Streams are Better than Reading All at Once

```js
// ❌ BAD — loads entire file into memory (can crash server for big files)
const data = fs.readFileSync("huge-file.txt", "utf8");
console.log(data);

// ✅ GOOD — streams chunked data, uses constant small amount of memory
const stream = fs.createReadStream("huge-file.txt");
stream.on("data", (chunk) => process(chunk));
```

## Interview Questions

- **Q: What are Streams in Node.js?**
  A: Streams allow processing data in chunks rather than all at once. There are 4 types: Readable, Writable, Duplex, and Transform.

- **Q: What is a Buffer?**
  A: A temporary storage for raw binary data. Buffers exist outside V8 heap and are used when transferring data between streams, networks, and file systems.

- **Q: What is `pipe()` in Node.js?**
  A: A method that connects a Readable stream to a Writable stream. It automatically handles data flow and backpressure.

- **Q: What is backpressure in streams?**
  A: When a writable stream is slower than the readable stream, data piles up in memory. Node.js streams handle this via backpressure — pausing the reader until the writer catches up.

- **Q: Why use streams instead of reading files directly?**
  A: Streams are memory-efficient — they process data in chunks instead of loading the entire file into memory, preventing crashes with large files.

- **Q: What is a Transform stream?**
  A: A Duplex stream that can modify or transform data as it passes through. Example: compression, encryption, uppercasing data.

- **Q: What is the difference between `readFile` and `createReadStream`?**
  A: `readFile` loads the entire file into memory at once. `createReadStream` reads the file in chunks, making it suitable for large files.
