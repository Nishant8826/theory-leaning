# Socket.io & WebSockets

**HTTP** is like sending letters — you send a request, the server replies, then the connection closes.

**WebSockets** are like a phone call — once connected, both sides can talk back and forth at any time without hanging up.

**Socket.io** is a library that makes WebSockets easy to use in Node.js — with fallbacks for older environments.

---

## Real-World Example
- A **chat app** (WhatsApp Web) — messages appear instantly without refreshing.
- A **live score update** — sports scores update without reloading the page.
- A **collaborative tool** — multiple users editing a document simultaneously.

---

## The Problem with Regular HTTP

```
HTTP (Unidirectional, Short-lived):
Client: "Hey, any new messages?"  → Server: "No."  → Connection closed.
Client: "Hey, any new messages?"  → Server: "Yes!"  → Connection closed.
(Repeated forever — wasteful polling!)
```

---

## How WebSockets Work

```
1. Client → Server: HTTP Upgrade request ("switch to WebSocket protocol")
2. Server: "OK, switching!" → Connection stays OPEN
3. Client ↔ Server: Send messages at any time, in both directions
4. Either side calls close() → connection ends
```

**Protocols:**
- `ws://` = insecure WebSocket
- `wss://` = secure WebSocket (used over HTTPS)

---

## Polling vs WebSockets

### Polling (Old approach)
```js
// Client asks for new data every 3 seconds
setInterval(async () => {
  const res = await fetch("/api/messages");
  const data = await res.json();
  displayMessages(data);
}, 3000);
```
❌ Wasteful — server responds even if no new data.

### WebSocket (Real-time approach)
- Server **pushes** data to client when it's ready.
- ✅ Instant, efficient, no wasted requests.

## Code Example

### Using the `ws` Library (Native WebSocket)

**Server (`server.js`)**
```js
const WebSocket = require("ws");

const wss = new WebSocket.Server({ port: 8080 });

wss.on("connection", (ws) => {
  console.log("Client connected!");

  ws.on("message", (message) => {
    console.log("Received:", message.toString());
    // Broadcast to ALL connected clients
    wss.clients.forEach((client) => {
      if (client.readyState === WebSocket.OPEN) {
        client.send(`Broadcast: ${message}`);
      }
    });
  });

  ws.on("close", () => {
    console.log("Client disconnected");
  });
});
```

**Client**
```js
const socket = new WebSocket("ws://localhost:8080");

socket.onopen = () => {
  socket.send("Hello Server!");
};

socket.onmessage = (event) => {
  console.log("Message from server:", event.data);
};
```

### Using Socket.io (Easier, More Features)

**Server**
```js
const express = require("express");
const http = require("http");
const { Server } = require("socket.io");

const app = express();
const server = http.createServer(app);
const io = new Server(server, {
  cors: { origin: "*" }, // allow all origins
});

io.on("connection", (socket) => {
  console.log(`User connected: ${socket.id}`);

  // Listen for "chat-message" event from client
  socket.on("chat-message", (msg) => {
    console.log("Message:", msg);
    // Broadcast to ALL other clients
    socket.broadcast.emit("chat-message", msg);
    // OR send to everyone including sender:
    // io.emit("chat-message", msg);
  });

  socket.on("disconnect", () => {
    console.log(`User disconnected: ${socket.id}`);
  });
});

server.listen(9000, () => console.log("Server on port 9000"));
```

**Client (HTML)**
```html
<script src="/socket.io/socket.io.js"></script>
<script>
  const socket = io("http://localhost:9000");

  // Send a message
  socket.emit("chat-message", "Hello World!");

  // Receive a message
  socket.on("chat-message", (msg) => {
    console.log("Received:", msg);
  });
</script>
```

## Socket.io Key Methods

| Method | What it does |
|--------|------------|
| `io.emit(event, data)` | Send to **ALL** connected clients |
| `socket.emit(event, data)` | Send to **this specific client** |
| `socket.broadcast.emit(event, data)` | Send to **everyone EXCEPT** this client |
| `socket.join(room)` | Join a room |
| `io.to(room).emit(event, data)` | Send to everyone in a **room** |

---

## Socket.io Rooms (Group Chats)
```js
io.on("connection", (socket) => {
  // Join a room (e.g., a specific chat group)
  socket.on("join-room", (roomId) => {
    socket.join(roomId);
    socket.to(roomId).emit("user-joined", `User ${socket.id} joined!`);
  });

  // Send message to a specific room
  socket.on("room-message", ({ roomId, message }) => {
    io.to(roomId).emit("room-message", message);
  });
});
```

## Interview Questions

- **Q: What is a WebSocket?**
  A: A protocol enabling a persistent, full-duplex (bi-directional) connection between client and server over TCP. Unlike HTTP, the connection stays open.

- **Q: How does a WebSocket connection start?**
  A: It starts as an HTTP request with an `Upgrade: websocket` header. If the server accepts, the protocol switches from HTTP to WebSocket.

- **Q: What is the difference between HTTP and WebSocket?**
  A: HTTP is stateless and unidirectional (request-response, connection closes). WebSocket is stateful and bidirectional (persistent connection, both sides can send anytime).

- **Q: What is Socket.io?**
  A: A Node.js library that wraps WebSocket and provides additional features: rooms, namespaces, auto-reconnection, and HTTP polling fallback.

- **Q: What is the difference between `socket.emit` and `io.emit`?**
  A: `socket.emit` sends to the specific connected client. `io.emit` sends to all connected clients.

- **Q: What are Socket.io rooms?**
  A: Logical groups of sockets. You can join a room and send messages only to members — great for group chats or game lobbies.

- **Q: What is long polling and how does it differ from WebSockets?**
  A: Long polling keeps an HTTP request open until data is available, then closes and reopens. WebSockets maintain one persistent connection — more efficient.
