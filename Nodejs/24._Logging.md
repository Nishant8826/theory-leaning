# Logging & Debugging

## Simple Explanation
**Logging** = Recording what happens in your app (requests, errors, events) so you can understand what's going on in production.

**Debugging** = Finding and fixing bugs in your code.

Think of logging like a **CCTV camera** in a store:
- It records everything that happens.
- When something goes wrong, you review the footage to understand what happened.

---

## Real-World Example
Your app crashes at 3am in production. Without logs, you have no idea why. With proper logging, you can see:
```
[2026-03-24T03:15:22Z] ERROR: Cannot read property 'id' of undefined
Stack: at UserController.getProfile (controllers/user.js:42)
Request: GET /api/users/profile — IP: 192.168.1.1
```

## `console.log` — Simple but Insufficient

```js
// Basic — avoid in production
console.log("Server started");
console.error("Something went wrong:", err.message);
console.warn("Deprecated function used");
```

❌ Problems: no timestamps, no log levels, no file output, all goes to stdout.

## Morgan — HTTP Request Logger

```bash
npm install morgan
```

```js
const morgan = require("morgan");

// 'dev' format — colorized, good for development
app.use(morgan("dev"));
// Output: GET /api/users 200 12.345 ms - 203

// 'combined' format — detailed, good for production
app.use(morgan("combined"));
// Output: ::1 - - [24/Mar/2026:03:15:22 +0000] "GET /api/users HTTP/1.1" 200 203

// Custom format
app.use(morgan(":method :url :status :response-time ms"));

// Log to a file
const fs = require("fs");
const path = require("path");
const accessLog = fs.createWriteStream(path.join(__dirname, "access.log"), { flags: "a" });
app.use(morgan("combined", { stream: accessLog }));
```

## Winston — Professional Logger

```bash
npm install winston
```

```js
// utils/logger.js
const winston = require("winston");

const logger = winston.createLogger({
  level: process.env.LOG_LEVEL || "info", // debug, info, warn, error
  format: winston.format.combine(
    winston.format.timestamp(),
    winston.format.colorize(),
    winston.format.printf(({ timestamp, level, message, ...meta }) => {
      return `[${timestamp}] ${level}: ${message} ${
        Object.keys(meta).length ? JSON.stringify(meta) : ""
      }`;
    })
  ),
  transports: [
    new winston.transports.Console(),     // log to terminal
    new winston.transports.File({          // log errors to file
      filename: "logs/error.log",
      level: "error",
    }),
    new winston.transports.File({          // log everything to file
      filename: "logs/combined.log",
    }),
  ],
});

// In production, disable colorize for cleaner file logs
if (process.env.NODE_ENV === "production") {
  logger.format = winston.format.combine(
    winston.format.timestamp(),
    winston.format.json()
  );
}

module.exports = logger;
```

```js
// Usage anywhere in your app
const logger = require("./utils/logger");

logger.info("Server started on port 3000");
logger.warn("Deprecated endpoint called");
logger.error("DB connection failed", { retries: 3, dbUrl: "..." });
logger.debug("User object:", { userId: 42, role: "admin" });
```

## Debugging with Node.js Built-in Tools

### Using `--inspect` flag
```bash
node --inspect index.js
```
Open Chrome → `chrome://inspect` → Click "Inspect" to open DevTools debugger.

### Using `debugger` keyword
```js
async function getUser(id) {
  debugger; // execution pauses here in debugger
  const user = await User.findById(id);
  return user;
}
```

### VS Code Debugger (`launch.json`)
```json
{
  "version": "0.2.0",
  "configurations": [
    {
      "type": "node",
      "request": "launch",
      "name": "Debug Node App",
      "program": "${workspaceFolder}/index.js",
      "envFile": "${workspaceFolder}/.env"
    }
  ]
}
```

## Debug Package (Conditional Logging)

```bash
npm install debug
```

```js
// Enable with: DEBUG=app:* node index.js
const debug = require("debug");

const appLog = debug("app:server");
const dbLog = debug("app:db");

appLog("Server starting...");
dbLog("Connecting to MongoDB...");

// Only shows logs for that namespace when DEBUG env var matches
```

## Logging Best Practices

```js
// ✅ DO — include context with every log
logger.info("User logged in", { userId: user._id, email: user.email });
logger.error("Failed to create order", { orderId, error: err.message });

// ❌ DON'T — log sensitive data
logger.info("Login attempt", { password: req.body.password }); // NEVER!

// ✅ Log request ID for tracing
const { v4: uuid } = require("uuid");
app.use((req, res, next) => {
  req.requestId = uuid(); // unique ID per request
  next();
});
logger.info("Request received", { requestId: req.requestId, url: req.url });
```

## Log Levels (in order of severity)
```
debug  → Detailed debugging info (dev only)
info   → Normal events (server start, user login)
warn   → Something unusual but recoverable
error  → Something failed (DB error, unhandled rejection)
```

## Practice Tasks
1. Add `morgan("dev")` to an Express app and verify requests are logged in the terminal.
2. Set up Winston to log `info` and above to the console, and `error` only to `logs/error.log`.
3. Add a request ID (`uuid`) to every log entry so you can trace the full lifecycle of a request.
4. Enable the VS Code debugger and set a breakpoint inside a route handler. Inspect `req.body` and `req.params`.
5. Create a log rotation setup where each day's logs go to a new file (research `winston-daily-rotate-file`).

## Interview Questions

- **Q: Why is `console.log` not enough for production logging?**
  A: No timestamps, no log levels, no file output, no structure. Hard to search through or set up alerting on.

- **Q: What is Morgan?**
  A: An HTTP request logger middleware for Express — logs method, URL, status code, and response time for every request.

- **Q: What is Winston?**
  A: A popular Node.js logging library that supports multiple log levels (`debug`, `info`, `warn`, `error`), multiple transports (console, file, external services).

- **Q: What are log levels?**
  A: Severity categories for log messages. Common levels: `debug < info < warn < error`. Set a threshold to filter which logs appear.

- **Q: What sensitive data should you never log?**
  A: Passwords, tokens, credit card numbers, personal identifiable information (PII). Always sanitize logs.

- **Q: How can you debug a Node.js app without `console.log`?**
  A: Use `--inspect` flag + Chrome DevTools, VS Code debugger with `launch.json`, or the `debug` npm package for conditional logging.
