# 🏛️ The 4 Pillars of OOPS

## Simple Explanation
Imagine you are building a **House**. 
1.  **Encapsulation:** You put all the electrical wires inside the walls so nobody gets a shock. (Data Safety).
2.  **Abstraction:** You only show the "Switch" to the user, not the 10 wires behind it. (Simplified View).
3.  **Inheritance:** You have a "Basic House" plan and use it to build a "Luxury House" with a pool. (Reuse the plan).
4.  **Polymorphism:** The "Key" for the door and the "Key" for the car look different but they both handle the "Unlock" job. (Same job, different way).

---

## What are the Pillars?
They are the 4 fundamental rules that make OOPS so powerful.

### 1. 🛡️ Encapsulation (Grouping & Privacy)
*   **What:** Bundling the data (properties) and the code that uses it (methods) into one single file (the Class).
*   **Why:** To keep the data safe from other parts of the app.
*   **Real-World:** A "Capsule" (like a pill). The medicine is inside the shell so you don't taste it directly.

### 2. 📺 Abstraction (Hiding Complexity)
*   **What:** Showing only the important details to the user and hiding the "how" it works.
*   **Why:** To make the code easier to use. You don't need to know how an Engine works to drive a car.
*   **Real-World:** A "Car Dashboard." You see the speed, but you don't see the 1,000 pipes under the hood.

### 3. 👶 Inheritance (Reusability)
*   **What:** Creating a new class from an existing class.
*   **Why:** To save time! You don't have to rewrite everything.
*   **Real-World:** You inherit your eye color from your parents. You are a "new version" of them.

### 4. 🎭 Polymorphism (Many Forms)
*   **What:** When one name (like `makeSound`) does different things based on the object.
*   **Why:** So you can treat different objects the same way.
*   **Real-World:** A "Dog" barks, a "Cat" meows. Both use the `makeSound` action, but the result is different.

---

## 🌍 Real-World Example: Banking App
*   **Encapsulation:** Your `bank_balance` is private. Only the `deposit` function can change it.
*   **Abstraction:** You click "Transfer," you don't see the 100 lines of encryption code running.
*   **Inheritance:** A "SavingsAccount" inherits from a "GeneralAccount" but adds an `interestRate`.
*   **Polymorphism:** The `calculateInterest()` function works differently for Loans vs Savings.

## Interview Questions
**1. Name the 4 pillars of OOPS?**
*   **A:** Encapsulation, Abstraction, Inheritance, and Polymorphism.

**2. Difference between Abstraction and Encapsulation?**
*   **A:** Abstraction is about **Hiding Complexity** (at the interface level), while Encapsulation is about **Hiding Data** (at the security level).

**3. What is the benefit of Inheritance?**
*   **A:** Code Reusability. It allows us to use existing logic and only add new features.

**4. Why is Polymorphism important?**
*   **A:** It allows us to write one piece of code that can work with many different types of objects smoothly.

**5. Which pillar helps in Data Hiding?**
*   **A:** Encapsulation.
