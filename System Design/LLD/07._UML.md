# 🗺️ UML Diagrams (The Movie Script)

## Simple Explanation
Imagine you want to make a **Movie**. 
1.  **Characters List:** Who is in the movie? (Class Diagram).
2.  **The Script:** Who talks to whom and when? (Sequence Diagram).
3.  **The Storyboard:** What happens if the hero chooses the left door vs the right door? (Activity Diagram).
**UML (Unified Modeling Language)** is the "Movie Script" for your software. It helps you draw the plan before you start writing code.

---

## What
UML is a set of standard diagrams used to **visualize** the design of a system. Instead of reading 1,000 lines of code, you look at one picture.

---

## Why
1.  **Clear Communication:** It’s easier to show a diagram to your boss than to explain your code for 1 hour.
2.  **Find Mistakes Early:** It’s cheaper to fix a "Box on a Paper" than to fix "Code on a Server."
3.  **Big Picture:** It helps you see how the Frontend, Backend, and Database all fit together.

---

## The 3 Most Important Diagrams

### 1. Class Diagram (The Structure)
*   **Shows:** What classes exist and how they are related.
*   **Analogy:** A Map of a city showing where the houses are.
*   **Example:** A `User` class connected to an `Order` class.

### 2. Sequence Diagram (The Interaction)
*   **Shows:** The step-by-step "Conversation" between objects over time.
*   **Analogy:** A WhatsApp chat history.
*   **Example:** User clicks "Buy" → UI talks to API → API talks to Database → Success message.

### 3. Activity Diagram (The Logic)
*   **Shows:** The flow of actions, including "If-Else" decisions.
*   **Analogy:** A Flowchart.
*   **Example:** If User is logged in → Show Dashboard. Else → Show Login Page.

---

## How it works (Simple Symbols)
*   **Square Box:** A Class or a Service.
*   **Arrow (--->):** One thing "Uses" another.
*   **Solid Diamond (◆--):** One thing "Owns" another (Composition).
*   **Empty Triangle (--|>):** One thing "Inherits" from another.

---

## Real-World Example: Login System
*   **Class Diagram:** We have a `User` class (email, password) and an `AuthService` class.
*   **Sequence Diagram:** 
    1. User enters email/pass.
    2. `AuthService` checks database.
    3. `AuthService` returns "Token."
*   **Activity Diagram:** 
    * Is email valid? No → Error. 
    * Is password correct? No → Error. 
    * Yes → Welcome!

## Interview Questions
**1. What is UML?**
*   **A:** It is a standardized visual language used to design and document software architecture.

**2. Difference between Class Diagram and Sequence Diagram?**
*   **A:** **Class Diagram** shows the **Static Structure** (what exists), while **Sequence Diagram** shows the **Dynamic Behavior** (how they talk over time).

**3. What is an "Association" in UML?**
*   **A:** It is a simple relationship where one class "uses" or "knows about" another class.

**4. When should a junior developer use UML?**
*   **A:** Before starting a complex task, to clarify the logic, or when trying to understand a new codebase.

**5. What does a "Diamond" symbol mean in a class diagram?**
*   **A:** It represents **Composition** or **Aggregation** (one object "contains" another, like a House contains Rooms).
