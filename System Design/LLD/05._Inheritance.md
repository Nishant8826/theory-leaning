# 👶 Inheritance (Like Parent, Like Child)

## Simple Explanation
Imagine you have a **Basic Phone** that can make calls and send SMS. 
Now you want to build a **SmartPhone**. Instead of starting from scratch, you just take the **Basic Phone** design and add a "Camera" and "Internet" to it. 
Your SmartPhone **Inherits** the ability to call and SMS from the Basic Phone.

---

## What
Inheritance is a way to create a new class (the **Child**) by using the features of an existing class (the **Parent**).

---

## Why
1.  **To Reuse Code:** Instead of writing the `login()` function 100 times for 100 different types of users, you write it once in the `User` class.
2.  **To Build Relationships:** It helps you say "A Dog **is an** Animal" or "A Manager **is an** Employee."
3.  **To Save Time:** You only need to write the *different* parts in the child class.

---

## How it works (extends & super)
In Javascript, we use the word `extends` to inherit. If the child needs to talk to the parent, it uses the word `super()`.

```javascript
// The Parent (Super Class)
class Animal {
    constructor(name) {
        this.name = name;
    }
    eat() { console.log(`${this.name} is eating...`); }
}

// The Child (Sub Class)
class Dog extends Animal {
    constructor(name, breed) {
        super(name); // Call the Parent's constructor
        this.breed = breed;
    }
    bark() { console.log("Woof! Woof!"); }
}

const myDog = new Dog("Buddy", "Golden Retriever");
myDog.eat(); // ✅ Works! (Inherited from Animal)
myDog.bark(); // ✅ Works! (His own feature)
```

---

## Key Terms
*   **Method Overriding:** When the child decides to do things **differently** than the parent (e.g., An Animal `makesSound()`, but a Dog `makesSound()` by barking).
*   **"Is-A" Relationship:** A `Dog` is an `Animal`. If this doesn't make sense, you shouldn't use inheritance.

---

## Real-World Example
*   **Users in an App:** You have a `User` class (can login). Then you have `Admin` (can delete users) and `Guest` (can only view). Both are children of `User`.
*   **E-commerce Items:** An `ElectronicItem` inherits from `Product`. It adds properties like `warrantyPeriod` and `voltage`.

---

## Interview Questions
**1. What is Inheritance?**
*   **A:** It is a mechanism where one class acquires the properties and methods of another class to promote code reusability.

**2. What is the "super" keyword?**
*   **A:** It is used to call the constructor or methods of the parent class from inside the child class.

**3. What is "Method Overriding"?**
*   **A:** It’s when a child class provides a new implementation for a method that is already defined in its parent class.

**4. Can a child class have its own properties?**
*   **A:** Yes! A child class can have its own unique properties and methods in addition to what it inherits.

**5. What is the "Is-A" relationship?**
*   **A:** It describes the relationship between a subclass and a superclass (e.g., A SavingsAccount **is-a** BankAccount).
