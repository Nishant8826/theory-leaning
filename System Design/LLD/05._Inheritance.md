# 🧬 Inheritance

## Simple Explanation
Suppose you are a **Father**. You have a **"Blue Eye"** trait and a **"House"** that you own. 

Your **Son** will "automatically" have Blue Eyes and can also live in your House. He doesn't have to "buy" the house again. In code, the **Child Class** "inherits" everything from the **Parent Class**.

---

## 🛠️ What is Inheritance?
Inheritance is the ability of a **new class** (Child) to reuse the properties and methods of an **existing class** (Parent).

## ❓ Why is it needed?
1.  **To Save Time:** You only write common code once and then reuse it in different variations.
2.  **To Keep Code Clean:** You don't have to repeat the same code for every "Car," every "Truck," and every "Bike."
3.  **To Model Relationships:** It represents a "IS-A" relationship. (A Car IS-A Vehicle).

---

## 🏗️ How does it work? (The Flow)
Parent Class (The Basics) → Child Class (The Special Features).

---

## 🚀 Deep Dive: How we do it in Code
In **Node.js/JavaScript**, we use the `extends` keyword.

### 🏢 Real-World Example: An Animal Kingdom
*   **Parent Class:** `Animal` (Everything has a name and breaths).
*   **Child Class 1:** `Dog` (It inherits `bark()` plus name and breathing).
*   **Child Class 2:** `Bird` (It inherits `fly()` plus name and breathing).

Neither the Dog nor the Bird has to write the "Breathing" code again.

---

## 💻 Code Example: Inheritance in Node.js
```javascript
// The Parent (Super Class)
class User {
    constructor(name) {
        this.username = name;
    }

    login() {
        console.log(`${this.username} is now logged in!`);
    }
}

// The Child (Sub Class)
class Admin extends User {
    deletePost() {
        console.log("Admin can delete this post!");
    }
}

const myUser = new Admin("nishant");
myUser.login(); // Inherited from User
myUser.deletePost(); // Special feature for Admin only
```

## 🎤 Interview Questions
**Q1: What is a Parent vs Child Class?**
*   **A:** The Parent (Super class) has the common features, and the Child (Sub class) inherits those features and adds its own special ones.

**Q2: What is the benefit of using `extends`?**
*   **A:** It promotes **Code Reusability**. You write the main logic once and "extend" it into many different specialized classes.

**Q3: Can a Child have two Parents in JavaScript?**
*   **A:** No. JavaScript (and Java) does NOT support "Multiple Inheritance" (one child, two parents). This is to keep the code simple and avoid errors. Use "Composition" instead!
