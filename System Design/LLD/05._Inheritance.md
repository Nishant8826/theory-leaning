## 3. Inheritance

**Inheritance** means **one class can reuse the properties and methods of another class**.

In simple words:  
👉 **Child class inherits features from Parent class**  
👉 **Avoids rewriting the same code again and again**

---

## Core Idea (Very Simple)
Inheritance represents an **"is-a" relationship**.

Example:
- `Car` **is a** `Vehicle`
- `AdminUser` **is a** `User`

---

## JavaScript Example (Simple)

### Parent Class
```js
class User {
  login() {
    console.log("User logged in");
  }
}
```

### Child Class
```js
class Admin extends User {
  deleteUser() {
    console.log("User deleted");
  }
}

const admin = new Admin();
admin.login();      // inherited from User
admin.deleteUser(); // own method
```

## Key Features of Inheritance
1. Code Reusability

Common code is written once and reused.
```js
class Vehicle {
  start() {
    console.log("Vehicle started");
  }
}

class Car extends Vehicle {}
class Bike extends Vehicle {}
```
2. Method Overriding

Child class can change parent behavior.
```js
class Employee {
  getRole() {
    return "Employee";
  }
}

class Manager extends Employee {
  getRole() {
    return "Manager";
  }
}
```
3. super Keyword

Used to access parent class methods or constructor.
```js
class Person {
  constructor(name) {
    this.name = name;
  }
}

class Student extends Person {
  constructor(name, grade) {
    super(name);
    this.grade = grade;
  }
}
```

## Types of Inheritance (Conceptual)

1. Single Inheritance → One parent, one child
2. Multilevel Inheritance → Parent → Child → Grandchild
3. Hierarchical Inheritance → One parent, many children

 JavaScript does not support multiple inheritance using classes.


## Real-World Analogy
- Employee → Manager
- Vehicle → Car
- User → Admin
