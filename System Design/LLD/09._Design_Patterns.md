# 🏗️ Design Patterns (The Recipe Book)

## Simple Explanation
Imagine you are building a **House**. 
1.  **The Master Key:** One key that opens every door. (Like a Singleton).
2.  **The Factory:** You tell a factory you want a "Chair," and it decides if it should be "Wooden" or "Plastic" based on what you need. (Factory Pattern).
3.  **The Alarm System:** When a window breaks, the **Alarm** notifies the **Police**, the **Owner**, and the **Security Guard**. (Observer Pattern).
**Design Patterns** are just "Famous Recipes" that developers use to solve common, everyday software problems.

---

## What
Design Patterns are proven solutions to common software design problems. They are like "Blueprints" that you can customize to fit your specific needs.

---

## Why
1.  **Speed Up Development:** You don't have to "invent the wheel." These recipes have been tested for 30 years!
2.  **Standard Language:** Instead of explaining for 1 hour, you can just say "I'm using the Singleton pattern," and every developer will understand.
3.  **Cleaner Code:** They help you organize your classes so they are easy to change later.

---

## The 3 Most Popular Patterns

### 1. Singleton (The One and Only)
*   **What:** Ensuring a class has only **ONE** instance (one object) in the whole app.
*   **Why:** You don't need 100 separate "Logins" or 100 "Database Connections." One is enough.
*   **How:** You check if an instance already exists; if yes, you return that same one.

### 2. Factory Pattern (The Creator)
*   **What:** You create an object without telling the user "how" it’s created or which specific class is used.
*   **Why:** So you can change how objects are made without breaking the rest of the app.
*   **How:** You have a `createProduct()` method that takes a "Type" and returns the correct product.

### 3. Observer Pattern (The News Channel)
*   **What:** When one object (The Subject) changes, all its "Subscribers" (The Observers) are notified automatically.
*   **Why:** Perfect for "Notifications" or "Events."
*   **How:** The Subject keeps a list of Observers and calls a function on them when something happens.

---

## Real-World Example
*   **Singleton:** The **Database Connection** in your app. It’s one big pipe that everyone uses.
*   **Factory:** A **Logger**. You can ask for a "File Logger" or a "Console Logger," and the Factory will give you the right one.
*   **Observer:** **Youtube Subscriptions**. When a YouTuber uploads (Subject), every subscriber (Observer) gets a notification.

## Interview Questions
**1. What is a Design Pattern?**
*   **A:** It is a general, reusable solution to a commonly occurring problem in software design.

**2. Difference between "Singleton" and "Global Variable"?**
*   **A:** A **Singleton** provides controlled access (like a private constructor) and ensures only one instance exists. A **Global Variable** can be easily changed by anyone and doesn't guarantee a single instance.

**3. When would you use the "Factory Pattern"?**
*   **A:** When you don't know the exact class or type of the object you need until the app is running (Dynamic creation).

**4. What is the "Observer Pattern" useful for?**
*   **A:** It’s great for creating systems that need to "react" to changes in another object, like a UI updating when a user clicks a button.

**5. Name a design pattern you have used?**
*   **A:** Suggestion: "I have used the **Singleton** pattern for managing my application's database connection to ensure resources are reused."
