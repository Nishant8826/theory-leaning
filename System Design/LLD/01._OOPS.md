## Simple Explanation
Imagine you are building a **Lego Castle**. You have different types of blocks: some are windows, some are doors, and some are simple bricks. Each block has its own size, color, and job. You don't just have one giant melted piece of plastic; you have separate "Objects" that you put together to build the castle. 
**OOPS (Object-Oriented Programming)** is exactly that! Instead of writing one long list of instructions, we create "Objects" that represent real things.

---

## What
OOPS is a way of writing code where everything is treated as an **Object**. 
An Object is a "thing" that has:
1.  **Properties (Data):** What it is (e.g., A car is "Red").
2.  **Actions (Methods):** What it can do (e.g., A car can "Drive").

---

## Why
1.  **To Solve Real-World Problems:** It‚Äôs easier to write code for a "Bank Account" if we think of it as an object with a `balance` and a `withdraw` button.
2.  **To Reuse Code:** Instead of writing the code for a "Button" 100 times, we create one `Button` object and use it everywhere.
3.  **To Keep Data Safe:** We can lock the data inside the object so other parts of the program can't accidentally break it.
4.  **To Manage Large Projects:** It‚Äôs much easier for 10 developers to work on 10 different objects than to work on one giant list of code.

---

## How it works (Class vs Object)
Think of a **Class** as a **Map** or a **Blueprint** of a house. It‚Äôs just paper!
Think of an **Object** as the **Actual House** built using that map. You can build 50 houses from 1 map!

```javascript
// This is the CLASS (The Map)
class Car {
  constructor(color, brand) {
    this.color = color;  // Property
    this.brand = brand;  // Property
  }

  drive() { console.log("The car is moving!"); } // Action
}

// These are the OBJECTS (The Houses)
const myCar = new Car("Red", "Tesla");
const yourCar = new Car("Blue", "BMW");
```

## Real-World Example
*   **Video Games:** Every character in a game (Mario, Luigi, Toad) is an **Object**. They all have a `health` property and a `jump` method.
*   **Banking:** Your `SavingsAccount` is an object. It has properties like `accountNumber` and actions like `transferFunds`.

## Interview Questions
**1. What is OOPS in one sentence?**
*   **A:** OOPS is a programming pattern that uses "Objects" (data + behavior) instead of just "functions" to build applications.

**2. Difference between a Class and an Object?**
*   **A:** A Class is a blueprint (the plan), while an Object is the instance (the actual thing) created from that blueprint.

**3. Why do we say OOPS is better for maintenance?**
*   **A:** Because if you need to change how a "Car" drives, you only change it in the `Car` class, and every car in your app gets updated automatically.

**4. Name the 4 main pillars of OOPS?**
*   **A:** Encapsulation, Abstraction, Inheritance, and Polymorphism.

**5. Can you have an Object without a Class?**
*   **A:** In some languages like Javascript, yes (using object literals), but in most OOP languages like Java or C++, you always need a Class first.
