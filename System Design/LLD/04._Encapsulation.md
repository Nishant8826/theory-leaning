# 💊 Encapsulation (Protect Your Data)

## Simple Explanation
Imagine you have a **Safe** (like a locker). You don't want anyone to just walk in and take your money. 
You put a **Lock** on the safe. To get the money out, you must use a **Key**.
**Encapsulation** is the "Lock" for your data. You hide your sensitive data inside a class and only allow access through "Safe Methods" (the Key).

---

## What
Encapsulation is the process of bundling data (properties) and methods (actions) into a single unit (the Class) and **restricting access** to the inner data.

---

## Why
1.  **Data Security:** Nobody can change your variables to a wrong value by mistake (e.g., setting a user's age to `-500`).
2.  **Control:** You have full control over how your data is read and modified.
3.  **Hiding Complexity:** The user doesn't need to know how the data is stored; they only need to know how to ask for it.

---

## How it works (Private Variables)
In modern Javascript, we use the `#` symbol to make a property **private**. This means nobody can touch it from outside the class.

```javascript
class BankAccount {
    #balance = 0; // PRIVATE - Nobody can touch this directly!

    constructor(initialAmount) {
        this.#balance = initialAmount;
    }

    // This is the "Key" to get data (Getter)
    getBalance() {
        return this.#balance;
    }

    // This is the "Key" to change data (Setter)
    deposit(amount) {
        if (amount > 0) {
            this.#balance += amount;
            console.log(`Deposited ${amount}. New balance is ${this.#balance}`);
        } else {
            console.log("Hey! You can't deposit a negative amount.");
        }
    }
}

const myAccount = new BankAccount(100);
console.log(myAccount.#balance); // ❌ ERROR! You can't touch it.
myAccount.deposit(50); // ✅ SUCCESS! Safe and controlled.
```

## Real-World Example
*   **The Pharmacy / Medical Capsule:** The medicine is inside a plastic shell. You take the shell as a whole. You don't open it and touch the powder yourself.
*   **Your Mobile Phone:** You have a "Settings" menu to change your brightness. You can't just open the phone and touch the wires to make it brighter.

## Interview Questions
**1. What is Encapsulation?**
*   **A:** It is the bundling of data and methods into a single unit (class) and hiding the internal state of the object.

**2. Difference between Abstraction and Encapsulation?**
*   **A:** **Abstraction** hides **Complexity** (at the design level), while **Encapsulation** hides **Data** (at the security level).

**3. What is a "Getter" and a "Setter"?**
*   **A:** A **Getter** is a method used to "read" a private property. A **Setter** is a method used to "update" a private property safely (with rules).

**4. Why is Encapsulation good for security?**
*   **A:** Because it prevents external code from making illegal changes to your data, like setting the price of a product to `$0.00` by mistake.

**5. How do you make a variable private in Javascript?**
*   **A:** By using the `#` symbol before the variable name inside the class.
