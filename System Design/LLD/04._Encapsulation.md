# 🔐 Encapsulation

## Simple Explanation
Imagine you have a **Capsule (a Medicine)**. 
*   All the bitter-tasting powder is **Hidden** and **Bound Together** inside one shell. 
*   You only take the capsule once. 

In code, we do the same "Bonding" of **Data (like Age)** and **Methods (like changeAge)** into one single **"Capsule"** (the Class).

---

## 🛠️ What is Encapsulation?
Encapsulation is the practice of **wrapping data and the code that handles it** together into a single unit (Class). It also means we **hide the data** from direct access by making it **private**.

## ❓ Why is it needed?
1.  **Security:** It prevents others from changing your data directly. (You don't want a "User" to change their own "Balance" back to $1,000,000).
2.  **Validation:** It allows us to check if the new value is correct before saving it!
3.  **Flexibility:** We can change the internal names without breaking the rest of the code.

---

## 🏗️ How does it work? (The Flow)
User asks for a change → Calls a **Getter/Setter** method → Method checks if they are allowed → Method updates the **Private Data**.

---

## 🚀 Deep Dive: How we do it in Code
In **Node.js/JavaScript**, we use the `#` prefix to make a variable private.

### 🏢 Real-World Example: A Bank Account
*   **Encapsulation:** The `balance` variable is private. (You cannot see it directly).
*   **Validation Handling:** You can only withdraw money if you have it! You must use the `withdraw()` method, which checks your balance for you!

---

## 💻 Code Example: Encapsulation in Node.js
```javascript
class BankAccount {
    #balance = 0; // The '#' makes it "Private" (Hidden/Secure)

    // A "Setter" method to safely put money in
    deposit(amount) {
        if (amount > 0) {
            this.#balance += amount;
            console.log(`Deposited $${amount}. New balance: $${this.#balance}`);
        } else {
            console.log("Error: Deposit amount must be positive!");
        }
    }

    // A "Getter" method to see the money safely
    getBalance() {
        return this.#balance;
    }
}

const myAccount = new BankAccount();
myAccount.deposit(100); 
// myAccount.#balance = 500000; // This will throw an error now! (Securely Encapsulated)
```

## 🎤 Interview Questions
**Q1: What is the main goal of Encapsulation?**
*   **A:** To bundle data and methods together and **secure** the data from accidental or global change.

**Q2: What are Getters and Setters?**
*   **A:** They are public methods that allow us to **Get** (read) or **Set** (write) private data safely and with validation.

**Q3: Encapsulation vs Abstraction?**
*   **A:** Abstraction hides **Complexity (the "How")**, while Encapsulation hides **Data (the "What happens inside")** for security and grouping.
