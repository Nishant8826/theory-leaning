# 🎭 Polymorphism (One Name, Many Forms)

## Simple Explanation
Imagine you have a **"Start"** button on different machines. 
*   On a **Car**, "Start" turns on the engine.
*   On a **Laptop**, "Start" turns on the screen.
*   On a **Microwave**, "Start" begins cooking.
Even though it’s the same **name** (Start), the action is **different** for each machine. 
**Polymorphism** is exactly that! It allows different objects to use the same "action name" but do it in their own way.

---

## What
Polymorphism comes from the Greek words **Poly** (Many) and **Morph** (Forms). It means a single action can behave differently based on the object that is using it.

---

## Why
1.  **To Simplify Code:** You don't have to remember 100 different names for the same action. (e.g., `carStart()`, `laptopStart()`, `ovenStart()`).
2.  **To Build Flexible Systems:** You can write one piece of code that tells any machine to `start()` and it will work!
3.  **To Make Code Clean:** It allows you to treat different objects in a single common way.

---

## How it works (Method Overriding)
We create a common "Parent" and let the "Child" change the behavior.

```javascript
class Animal {
    makeSound() { console.log("Some sound..."); }
}

class Dog extends Animal {
    makeSound() { console.log("Woof! Woof!"); } // Change it!
}

class Cat extends Animal {
    makeSound() { console.log("Meow meow!"); } // Change it!
}

// THE POWER of Polymorphism
const animals = [new Dog(), new Cat(), new Dog()];

// We don't care if it's a Dog or a Cat, just tell them to makeSound()!
animals.forEach(animal => animal.makeSound());
```

---

## Key Terms
*   **Compile-time Polymorphism (Overloading):** Having the same function name but different "inputs." (Javascript doesn't directly support this, but we simulate it).
*   **Runtime Polymorphism (Overriding):** Changing how a function works in a child class.

---

## Real-World Example
*   **A Professional Actor:** An actor can be a **Doctor** in one movie, a **Cop** in another, and a **Teacher** in the third. It’s the same "Actor," but they take on different **Forms**.
*   **Smartphone Connectors:** You have a `charge()` port. Whether you plug in a "Wall Charger" or a "Car Charger," your phone uses the same port to `charge()`.

## Interview Questions
**1. What is Polymorphism?**
*   **A:** It is the ability of an object to take on many forms. Most commonly, it allows a child class to change the way a parent's method works.

**2. Difference between Overloading and Overriding?**
*   **A:** **Overloading** is having methods with the same name but different "parameters" (same class). **Overriding** is changing a method's behavior in a "child class."

**3. Why is it used in System Design?**
*   **A:** It makes the system "Plug-and-Play." You can add a new type of `Notification` (like SMS) without changing the main code that sends notifications.

**4. How does Javascript handle Overloading?**
*   **A:** Since Javascript doesn't have native overloading, we check the number and type of arguments inside a single function using `if-else`.

**5. What is the benefit of using an Interface for Polymorphism?**
*   **A:** It ensures that every object (like a `PaymentMethod`) has the exact same name for its main action (like `pay()`), making it easy to swap them.
