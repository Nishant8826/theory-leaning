# 🎭 Polymorphism

## Simple Explanation
Imagine you have a **"Select"** button on your phone. 
*   If you "Select" a Photo, it opens. 
*   If you "Select" a Song, it plays. 
*   If you "Select" a Contact, it calls them. 

The **Action** is the same ("Select"), but the **Behavior** changes based on what you clicked. One name, many forms!

---

## 🛠️ What is Polymorphism?
Polymorphism is the ability of an **Object** to take **many forms**. It allows one interface or function to work differently depending on the object type.

## ❓ Why is it needed?
1.  **Flexibility:** You don't have to write 50 different function names like `playSong()`, `openPhoto()`, `callContact()`. You just use one name: `action()`.
2.  **Scalability:** If you add a new "Video" object later, you just tell it what to do on "Select," and the rest of the app already knows how to use it.
3.  **Clean Code:** It reduces `if-else` blocks in your code.

---

## 🏗️ How does it work? (The Flow)
User calls `makeSound()` → Checks if the object is **Dog** (Woof!) or **Cat** (Meow!) → Action changes based on the object.

---

## 🚀 Deep Dive: Method Overriding
This is the most common way to do it. You have a **Parent** method and a **Child** that "Redefines" (Overrides) it.

### 🏢 Real-World Example: A Payment System
*   **Parent Class:** `Payment` (has a `process()` method).
*   **Child Class 1:** `CardPayment` (`process()` checks for CVV).
*   **Child Class 2:** `UPIPayment` (`process()` checks for PIN).

The Checkout page only calls `payment.process()`—it doesn't care if it’s a Card or UPI!

---

## 💻 Code Example: Polymorphism in Node.js
```javascript
// Generic Parent
class Animal {
    makeSound() {
        console.log("Some generic animal sound...");
    }
}

// Child 1
class Dog extends Animal {
    makeSound() { // Overriding the Parent method
        console.log("Woof! Woof!");
    }
}

// Child 2
class Cat extends Animal {
    makeSound() { // Overriding again
        console.log("Meow! Meow!");
    }
}

// A list of different animals
const myPets = [new Dog(), new Cat(), new Animal()];

// Poly-morphism in action: One name, different sounds!
myPets.forEach(pet => pet.makeSound());
```

## 🎤 Interview Questions
**Q1: What is Method Overriding?**
*   **A:** It’s when a Child class creates a custom version of a method that already exists in its Parent class.

**Q2: What is Method Overloading?**
*   **A:** (Extra Info) It’s having multiple methods with the same name but different "inputs." (Not built-in to JavaScript, but common in Java/C++).

**Q3: How does Polymorphism simplify code?**
*   **A:** It replaces long stacks of `switch` or `if/else` statements with one simple function call that "automatically" knows what to do based on the object's type.
