## 4. Polymorphism
`Polymorphism` in JavaScript is an object-oriented programming concept that allows different objects to share the same method name but execute different behaviors based on the object calling the method. It enhances code flexibility and reusability by allowing a single interface to work with different underlying data types or objects. 

JavaScript achieves polymorphism primarily through **method overriding** and dynamic typing, and simulates method overloading. 

### Method Overriding (Runtime Polymorphism)
Method overriding is the most common form of polymorphism in JavaScript, achieved through inheritance (either class-based with the **class** keyword or prototype-based). A subclass provides a specific implementation of a method that is already defined in its superclass. When the method is called, the JavaScript runtime determines which version to execute based on the actual object's type, not the variable's declared type. 

```js
class Animal {
  makeSound() {
    return 'Generic animal sound';
  }
}

class Dog extends Animal {
  makeSound() { // Overrides the parent method
    return 'Woof! Woof!';
  }
}

class Cat extends Animal {
  makeSound() { // Overrides the parent method
    return 'Meow meow';
  }
}

const animal = new Animal();
const dog = new Dog();
const cat = new Cat();

console.log(animal.makeSound()); // Output: Generic animal sound
console.log(dog.makeSound());   // Output: Woof! Woof!
console.log(cat.makeSound());   // Output: Meow meow
```

In this example, the makeSound() method behaves differently for each animal object, even though they all inherit from the Animal class. 

### Method Overloading (Ad-hoc Polymorphism Simulation)
JavaScript does not support method overloading natively (where multiple methods have the same name but different parameter counts or types). However, this behavior can be simulated using conditional logic inside a single function to check the number or type of arguments passed. 

```js
function add(a, b) {
  if (typeof a === 'number' && typeof b === 'number') {
    return a + b;
  } else if (typeof a === 'string' && typeof b === 'string') {
    return a + b; // String concatenation
  } else if (typeof a === 'number' && b === undefined) {
    return a * a; // Square the single number
  } else {
    return 'Invalid input';
  }
}

console.log(add(2, 3));      // Output: 5
console.log(add('Hello', ' World')); // Output: Hello World
console.log(add(5));         // Output: 25
```
Here, the single add function behaves differently based on the input it receives. 

### Key Benefits
- `Code Reusability`: Allows developers to write generic code that works with a common interface but leverages specific implementations.
- `Flexibility and Maintainability`: Makes the codebase easier to extend and manage, as new classes can be added without modifying existing logic, as long as they follow the shared interface.
- `Dynamic Behavior`: The actual behavior is determined at runtime, providing dynamic and adaptable code
