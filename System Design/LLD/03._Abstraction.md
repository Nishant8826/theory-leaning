# 1. Abstraction 
**Abstraction** means showing only what is necessary to the user and hiding how things work internally.  
In simple words: **you use a feature without worrying about its internal logic**.

You focus on **what it does**, not **how it is done**.

---

## Key Concepts
- **Show Only What’s Needed**: Users interact with simple methods, not complex logic.
- **Simplifies Code**: Complex systems become easier to understand and use.
- **Contract-Based Design**: Developers agree on *what functions exist*, not how they are implemented.


## Types of Abstraction
1. `Data Abstraction`: Hides the internal representation of data structures, exposing only relevant operations.
```js
class BankAccount {
  #balance = 0; // private data

  deposit(amount) {
    this.#balance += amount;
  }

  getBalance() {
    return this.#balance;
  }
}

const account = new BankAccount();
account.deposit(1000);
account.getBalance(); // You don't know how balance is stored
```
2. `Process (Control) Abstraction`: Hides the implementation of a complex process, providing a simplified interface for use (e.g., a single function call for a multi-step task). 
```js
function makePayment(amount) {
  validateUser();
  checkBalance();
  deductAmount(amount);
  sendReceipt();
}

// User only calls one function
makePayment(500);
```


## Implementation Methods
- `Abstract Classes`: Classes that cannot be instantiated and may contain both abstract methods (no body) and concrete methods (with logic).
- `Interfaces`: Blueprints that contain only method signatures, defining a strict contract for classes to implement.
- `Access Specifiers`: In languages like C++, abstraction is also achieved by marking internal members as private and exposing public interfaces. 

## Real-World Examples
1. `ATM Machine`: You interact with the screen and buttons (interface) to withdraw cash without knowing the background verification or mechanical dispensing processes.
2. `Coffee Machine`: You press a single button to get coffee; you don't need to understand how the machine grinds beans or heats water internally.
3. `Driving a Car`: A driver uses the steering wheel and pedals to control the vehicle without needing to understand the engine's internal mechanics.
