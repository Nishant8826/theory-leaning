# Pillars of OOPS in JavaScript (Simple for Junior SDEs)

## 1. Abstraction 
`Abstraction` is the process of hiding internal implementation details while exposing only the essential features to the user. It focuses on what an object does rather than how it does it.

### Real-World Examples
1. `ATM Machine`: You interact with the screen and buttons (interface) to withdraw cash without knowing the background verification or mechanical dispensing processes.
2. `Coffee Machine`: You press a single button to get coffee; you don't need to understand how the machine grinds beans or heats water internally.
3. `Driving a Car`: A driver uses the steering wheel and pedals to control the vehicle without needing to understand the engine's internal mechanics.

## 2. Encapsulation
`Encapsulation` is the practice of bundling data (attributes) and the methods that operate on that data into a single unit, typically a class. It acts as a protective shield, preventing direct access to an object's internal state from outside code.

### Comparison with Abstraction
While often confused, they have different focuses: 
- `Abstraction focuses on what an object does`, exposing only the essential features to the user (e.g., a "Start" button on a car).
- `Encapsulation focuses on how it works internally by hiding` implementation details and securing data (e.g., the engine's internal mechanics are hidden under the hood). 



## 3. Inheritance
`Inheritance` is a mechanism that allows a new class (the subclass or child) to acquire the properties and behaviors of an existing class (the superclass or parent). This establishes an "is-a" relationship (e.g., a Dog is an Animal) and is a primary tool for code reusability
