# 🧩 CAP Theorem (The Golden Rule)

## Simple Explanation
Imagine you have two friends, Rahul and Amit. You give them a secret. 
Now, Amit goes on vacation to a forest where there is **no phone network** (this is a **Partition**). 
If someone asks Rahul the secret, what should he do?
1.  **Be Consistent:** Tell the person the secret, but only if he can double-check with Amit (But Amit has no network, so Rahul must say "I don't know").
2.  **Be Available:** Tell the person the secret immediately, even if it might be an old version (since he can't talk to Amit).

**CAP Theorem** says you cannot have 100% of both in a system where the network fails!

---

## What
CAP Theorem stands for **C**onsistency, **A**vailability, and **P**artition Tolerance. It states that in a distributed system, you can only pick **two** out of three.

### 📝 Breakdown:
1.  **Consistency (C):** Every user sees exactly the same data at the exact same moment. No "old" data allowed.
2.  **Availability (A):** The system always responds to requests, even if the data might be slightly outdated.
3.  **Partition Tolerance (P):** The system keeps working even when parts of the network "break" or can't talk to each other.

---

## Why
In the real world, networks **always fail** occasionally. Therefore, every system *must* handle **Partition Tolerance (P)**. This means your real choice is between **C (Consistency)** or **A (Availability)**.

*   **CP Systems (Consistency + Partition Tolerance):** Better for bank apps. Data must be 100% right even if the app slows down.
*   **AP Systems (Availability + Partition Tolerance):** Better for Instagram/Facebook. It’s okay if a "Like" shows up 2 seconds late, as long as the page loads!

---

## How it works
Imagine a database with two servers: Server A and Server B.
1.  User updates their profile on **Server A**.
2.  Suddenly, the wire between A and B snaps (**Partition**).
3.  Server A has the new data; Server B has the old data.
4.  If a user asks **Server B** for data:
    *   **CP choice:** Server B says: *"I don't know the latest data, so I'll show an error"* (Choosing Consistency).
    *   **AP choice:** Server B says: *"I'll show you what I have (even if it's old)"* (Choosing Availability).

---

## Real-World Example
*   **ATM Machine (CP):** If the ATM loses connection to the main bank server, it stops giving money. It chooses **Consistency** over Availability because it doesn't want you to withdraw money twice.
*   **Facebook News Feed (AP):** If one server can't talk to another, it might show you an old post. It chooses **Availability** so you aren't stuck with an empty screen.

## Interview Questions
**1. What is the CAP Theorem?**
*   **A:** It says that in a distributed system, you can only guarantee two out of three: Consistency, Availability, and Partition Tolerance.

**2. Can we have a system with CA (no Partition Tolerance)?**
*   **A:** No. In the real world, networks always fail. If you don't have Partition Tolerance, your whole app crashes when a single wire snaps.

**3. When would you choose an AP system?**
*   **A:** For social media feeds, search engines, or any app where "showing something" is better than "showing nothing/error."

**4. When would you choose a CP system?**
*   **A:** For financial transactions, medical records, or any system where accuracy is more important than speed.

**5. What happens during a network partition in a CP system?**
*   **A:** The system will refuse to answer requests if it cannot guarantee that the data is 100% up to date.
