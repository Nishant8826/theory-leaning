# ⚖️ Load Balancing

## Simple Explanation
Imagine a very popular restaurant. If there is only one waiter, they will get tired and customers will wait forever. 
A **Load Balancer** is like a "Manager" standing at the door. He sees which waiter is free and sends the next customer to them. This keeps everyone happy and ensures no waiter is over-worked!

---

## What
A Load Balancer is a tool (hardware or software) that sits in front of your servers and distributes user requests across them.

---

## Why
1.  **To handle more users:** One server has a limit. Multiple servers working together can handle millions of users.
2.  **To stay "Up" (Reliability):** If one server crashes, the Load Balancer stops sending requests there and shifts them to working servers.
3.  **Low Latency (Speed):** By splitting the work, every request is processed much faster.

---

## How it works
The Load Balancer uses "Algorithms" (rules) to decide where to send the user:
1.  **Round Robin:** Sends the 1st person to Server A, 2nd to Server B, 3rd to Server C, and then back to A. (Like dealing cards).
2.  **Least Connections:** Sends the user to the server which is currently the least busy.
3.  **IP Hash:** Uses the user's IP address to always send them to the **same** server. This is useful for "Sticky Sessions."
4.  **Weighted Round Robin:** If Server A is 2x faster than Server B, the Load Balancer sends 2x more requests to Server A.

---

## Real-World Example
*   **Google Search:** Millions of people search at once. Google doesn't have one giant computer; they have thousands of small ones. A Load Balancer connects you to the one that is closest and least busy.
*   **E-commerce (Amazon):** During a big sale, the Load Balancer ensures that even if 50 servers crash, 5,000 others keep taking orders.

## Practice Tasks
- **Task 1 (Basic):** You have two servers: a "Pro" server (very fast) and a "Lite" server (slow). Which algorithm would you use?
- **Task 2 (Thinking):** What happens if the **Load Balancer itself crashes**? (Hint: Think about "Redundancy").
- **Task 3 (Design):** You are building a Chat App. A user is connected to Server A. If they suddenly get sent to Server B by the Load Balancer, will their "Chat History" disappear?

## Interview Questions
**1. What is a Load Balancer in one sentence?**
*   **A:** It’s a traffic controller that distributes requests across multiple servers to ensure efficiency and uptime.

**2. Where can we put a Load Balancer?**
*   **A:** Anywhere! Between User and Web Server, between Web Server and Application Server, or between Application Server and Database.

**3. What is the "Round Robin" method?**
*   **A:** It’s a simple cycling method where requests are sent to servers one by one in a fixed order.

**4. What is "Sticky Sessions"?**
*   **A:** It’s a feature where a Load Balancer always sends a specific user to the same server for their entire session (usually done with IP Hashing).

**5. How does a Load Balancer know if a server is dead?**
*   **A:** It performs "Health Checks." It asks the server "Are you okay?" every few seconds. If the server doesn't respond, it stops sending requests.
