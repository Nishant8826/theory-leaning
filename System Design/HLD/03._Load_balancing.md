# 🚀 Load Balancer

## Simple Explanation
Imagine you have a very popular restaurant with only one waiter. That one waiter will get overwhelmed, and customers will have to wait for a long time. 

A **Load Balancer** is like a "Manager" at the entrance of the restaurant who sees which waiter is free and sends the customer to that waiter. In the tech world, it’s a device or software that sits in front of your servers and distributes "traffic" (user requests) across multiple servers.

---

## 🛠️ What is a Load Balancer?
It’s a "Traffic Controller." It sits between the user and the backend to distribute thousands of user clicks across several "healthy" servers.

## ❓ Why is it needed?
1.  **To handle more users:** One server has a limit. Multiple servers working together can handle millions of users.
2.  **Reliability:** If one server crashes, the Load Balancer sends users to the other working servers. Your site stays "Up."
3.  **Efficiency:** It prevents one server from being overloaded while others sit idle.

## 🏗️ How does it work? (The Flow)
User Click → Load Balancer → (Check if Server A is free) → Send request to Server A.

---

## 🚀 Deep Dive (Important Interviews Topic)

### 1. The Algorithms (How the Manager chooses)
*   **Round Robin:** The simplest way. Sends the 1st request to Server 1, the 2nd to Server 2, and so on. Like dealing cards in a game.
*   **Least Connections:** The Load Balancer checks who is doing the *least* work and sends the user there.
*   **IP Hashing:** It looks at the user's IP address (digital ID) and uses math to always send that specific user to the same server. (Good for "remembering" a user's session).

### 2. Health Checks (The "Are you alive?" Test)
Every few seconds, the Load Balancer asks each server: *"Are you okay?"* If a server doesn't answer (it crashed), the Load Balancer stops sending it users and puts it on a "Red List."

### 🏢 Real-World Example: YouTube or Netflix
Try to imagine **YouTube** without a load balancer. If all of us tried to watch a video from the same server in California, it would catch fire! Instead, a Load Balancer sees you are in London and sends you to a server in Europe that is currently free.

## 💻 Code Example: A Mock Load Balancer Logic
```javascript
// Simple Round Robin Logic
class LoadBalancer {
    constructor(servers) {
        this.servers = servers; // e.g., ['Server1', 'Server2', 'Server3']
        this.index = 0;
    }

    getNextServer() {
        // Pick the next server in a circle
        const server = this.servers[this.index];
        this.index = (this.index + 1) % this.servers.length;
        return server;
    }
}

const lb = new LoadBalancer(['192.168.1.1', '192.168.1.2', '192.168.1.3']);
console.log(`Sending traffic to: ${lb.getNextServer()}`);
console.log(`Sending traffic to: ${lb.getNextServer()}`);
```

## 🎤 Interview Questions
**Q1: What is a Layer 4 vs Layer 7 Load Balancer?**
*   **A:** Layer 4 looks at network data (IP/Port). Layer 7 looks at the actual content (URLs, headers, cookies). It’s "smarter" but also "slower."

**Q2: What is "Server Stickiness"?**
*   **A:** It’s a feature where a Load Balancer remembers which server a user was talking to and keeps sending that user to the same server for their entire session.

**Q3: Can a Load Balancer handle an attack like DDoS?**
*   **A:** Yes, it can help by filtering out "garbage" traffic and distributing the attack load so the whole system doesn't go down.
