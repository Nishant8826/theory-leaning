# 🎡 Consistent Hashing

## Simple Explanation
Imagine you have 100 students and 4 different teachers. You use a rule (like the last digit of their ID) to assign them to a teacher. 
But what if Teacher #2 leaves? Now your rule is broken, and you have to re-assign *every single student* to a new teacher. That’s a mess! 

**Consistent Hashing** is a smarter rule that says: if one teacher leaves, only *their* students have to move. Everyone else stays with their original teacher.

---

## What
Consistent Hashing is a technique used to distribute your data (like users) across multiple servers in a way that minimizes work when a server is added or removed.

---

## Why
1.  **To Handle Server Crashes:** If a server dies, we don't want to reshuffle the entire database.
2.  **To Scale Easily:** When we add a new server, only a small portion of the data needs to move to it.
3.  **To Minimize Re-Shuffling:** In simple hashing (using `%` or MOD), adding one server can force 99% of your data to move! Consistent Hashing reduces this to a tiny amount.

---

## How it works (The Magic Circle)
1.  Imagine all your potential "Hash Results" are placed on a giant **Circle (Ring)**.
2.  We place our **Servers** at different spots on this circle (e.g., Server A at 12 o'clock, Server B at 4 o'clock, Server C at 8 o'clock).
3.  When a **User Request** comes, we hash its ID and place it on the circle.
4.  The user is assigned to the **first server they find** while moving clockwise around the circle.
5.  **If Server B crashes:** The users who would have gone to B just "keep walking" until they find Server C. Only B's users move!

---

## Real-World Example
*   **Discord / Chat Apps:** If you have millions of chat rooms, you can't put them all on one server. Consistent Hashing decides which server "owns" which chat room. 
*   **Amazon (The DynamoDB):** This is how Amazon stores millions of shopping carts without losing them when one computer breaks down.

## Interview Questions
**1. What is the main problem with "Mod Hashing" (Standard Hashing)?**
*   **A:** It’s "not scalable." If you have `n` servers and you add one more, almost every piece of data moves to a new location.

**2. How does Consistent Hashing work in one sentence?**
*   **A:** It maps both servers and data onto a logical "ring" and assigns data to the nearest server on that ring.

**3. What is a "Virtual Node"?**
*   **A:** To keep the server load even, we place each server at *multiple* spots on the ring. These extra spots are called Virtual Nodes.

**4. Why is it important for a Cache system?**
*   **A:** If we use standard hashing and a cache server crashes, almost all "Cache Hits" become "Cache Misses," causing the database to crash under stress.

**5. Which tech uses it?**
*   **A:** Redis Cluster, Cassandra, and Amazon DynamoDB all use this technique!
