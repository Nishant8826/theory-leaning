# ES6+ Features

**ES6** (ECMAScript 2015) was a massive update to JavaScript that introduced many modern features. These features make code **shorter**, **cleaner**, and **easier to read**.

---

## 1. `let` and `const` (Block Scope)

```js
// Before ES6 — only var (function-scoped, messy)
var x = 10;

// ES6 — let (block-scoped, reassignable)
let score = 0;
score = 10; // ✅

// ES6 — const (block-scoped, constant reference)
const PI = 3.14;
// PI = 3; // ❌ Cannot reassign

const user = { name: "Nishant" };
user.name = "Raj"; // ✅ You can still mutate the object's contents!
```

---

## 2. Arrow Functions

Shorter syntax for functions. Also fixes `this` binding in callbacks.

```js
// Before
function add(a, b) { return a + b; }

// ES6
const add = (a, b) => a + b;
const square = n => n * n; // single param, no parens needed
const greet = () => "Hello!"; // no params
```

---

## 3. Template Literals (Template Strings)

Use backticks `` ` `` instead of quotes. Supports multiline strings and embedded expressions.

```js
const name = "Nishant";
const age = 22;

// Before
console.log("Hello, " + name + "! You are " + age + " years old.");

// ES6
console.log(`Hello, ${name}! You are ${age} years old.`);

// Multi-line string
const html = `
  <div>
    <h1>${name}</h1>
  </div>
`;
```

## 4. Destructuring

Extract values from arrays or properties from objects into variables — in one clean line.

### Object Destructuring
```js
const user = { name: "Nishant", age: 22, city: "Delhi" };

// Before
const name = user.name;
const age = user.age;

// ES6
const { name, age, city } = user;
console.log(name, age, city); // "Nishant" 22 "Delhi"

// Rename while destructuring
const { name: fullName, age: years } = user;
console.log(fullName, years); // "Nishant" 22

// Default values
const { country = "India" } = user;
console.log(country); // "India"
```

### Array Destructuring
```js
const colors = ["red", "green", "blue"];

const [first, second, third] = colors;
console.log(first); // "red"

// Skip elements
const [, , last] = colors;
console.log(last); // "blue"

// Swap variables
let a = 1, b = 2;
[a, b] = [b, a];
console.log(a, b); // 2 1
```

## 5. Spread Operator (`...`)

"Spread" an array or object into individual elements.

```js
// Combining arrays
const fruits = ["apple", "banana"];
const veggies = ["carrot", "potato"];
const food = [...fruits, ...veggies];
console.log(food); // ["apple", "banana", "carrot", "potato"]

// Copying an array (shallow)
const original = [1, 2, 3];
const copy = [...original];

// Combining objects
const defaults = { theme: "light", lang: "en" };
const userPrefs = { lang: "hi", fontSize: 16 };
const settings = { ...defaults, ...userPrefs }; // userPrefs overrides defaults
console.log(settings); // { theme: "light", lang: "hi", fontSize: 16 }

// Passing array as function arguments
function sum(a, b, c) { return a + b + c; }
const nums = [1, 2, 3];
console.log(sum(...nums)); // 6
```

---

## 6. Rest Parameters (`...`)

Same `...` syntax, but in **function parameters** — collects remaining arguments into an array.

```js
function buildTeam(leader, ...members) {
  console.log("Leader:", leader);
  console.log("Members:", members);
}

buildTeam("Nishant", "Raj", "Priya", "Sam");
// Leader: Nishant
// Members: ["Raj", "Priya", "Sam"]
```

---

## 7. Default Parameters

Set default values for function arguments.

```js
function greet(name = "Guest", greeting = "Hello") {
  return `${greeting}, ${name}!`;
}

console.log(greet());                 // "Hello, Guest!"
console.log(greet("Nishant"));        // "Hello, Nishant!"
console.log(greet("Nishant", "Hi")); // "Hi, Nishant!"
```

---

## 8. Shorthand Object Properties & Methods

```js
const name = "Nishant";
const age = 22;

// Before
const user = { name: name, age: age };

// ES6 shorthand
const user = { name, age };

// Method shorthand
const obj = {
  greet() { // no need for : function
    return "Hello!";
  }
};
```

---

## 9. Optional Chaining (`?.`)

Safely access deeply nested properties without errors.

```js
const user = { profile: { address: { city: "Delhi" } } };

// Before — unsafe, throws if profile is null
// console.log(user.profile.address.city);

// ES6+ — safe
console.log(user?.profile?.address?.city);   // "Delhi"
console.log(user?.settings?.theme);          // undefined (no error!)
```

---

## 10. Nullish Coalescing (`??`)

Returns the right side only when the left side is `null` or `undefined` (not 0 or "").

```js
const count = 0;
console.log(count || "No items");  // "No items" (wrong — 0 is falsy!)
console.log(count ?? "No items");  // 0          (correct — 0 is not null/undefined)

const username = null;
console.log(username ?? "Anonymous"); // "Anonymous"
```

## Interview Questions

- **Q: What is the difference between `let`, `const`, and `var`?**
  A: `var` is function-scoped and hoisted with `undefined`. `let`/`const` are block-scoped and in TDZ. `const` cannot be reassigned.

- **Q: What is the difference between spread and rest?**
  A: Spread (`...`) expands an array/object into individual values. Rest (`...`) collects multiple arguments into an array. Same syntax, opposite roles.

- **Q: What is destructuring?**
  A: A syntax to unpack values from arrays or objects into variables in one line.

- **Q: What does optional chaining (`?.`) do?**
  A: It safely accesses nested properties — returns `undefined` instead of throwing an error if a value in the chain is `null` or `undefined`.

- **Q: What is the difference between `||` and `??`?**
  A: `||` returns right side for any **falsy** value (0, "", false, null, undefined). `??` returns right side only for `null` or `undefined`.

- **Q: Can you mutate a `const` object?**
  A: Yes. `const` prevents reassignment of the variable, but the object's contents can still be changed.
