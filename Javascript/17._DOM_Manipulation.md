# DOM Manipulation

## Simple Explanation
**DOM** (Document Object Model) = A tree-like structure representing all HTML elements as JavaScript objects.

**DOM Manipulation** = Using JavaScript to **read, create, update, or delete** HTML elements â€” making pages dynamic.

Think of it like a family tree of HTML elements. JS lets you talk to any family member and change what they say or look like.

---

## Real-World Example
- A **live search** that filters results as you type.
- A **shopping cart** that updates the total live.
- A **dark mode toggle** that changes the page theme.

## Selecting Elements

```js
const title = document.getElementById("main-title");
const firstBtn = document.querySelector(".btn");       // first match
const allBtns = document.querySelectorAll(".btn");     // all matches
const paragraphs = document.getElementsByTagName("p"); // by tag
```

---

## Reading and Changing Content

```js
const heading = document.getElementById("heading");

console.log(heading.textContent); // read text
console.log(heading.innerHTML);   // read HTML

heading.textContent = "New Title";           // change text (safe)
heading.innerHTML = "New <b>Title</b>";      // change HTML (supports tags)
```

## Changing Styles & Classes

```js
const box = document.getElementById("box");

box.style.backgroundColor = "blue";
box.style.display = "none";

box.classList.add("active");
box.classList.remove("active");
box.classList.toggle("dark-mode");
box.classList.contains("active"); // returns true/false
```

## Changing Attributes

```js
const img = document.getElementById("avatar");
img.getAttribute("src");
img.setAttribute("src", "new-image.jpg");
img.removeAttribute("alt");
img.src = "profile.jpg"; // direct property
```

## Creating and Adding Elements

```js
const newItem = document.createElement("li");
newItem.textContent = "Buy groceries";
newItem.classList.add("todo-item");

const list = document.getElementById("todo-list");
list.appendChild(newItem);  // adds at end
list.prepend(newItem);       // adds at beginning
```

## Removing Elements

```js
document.getElementById("item-1").remove(); // modern
parent.removeChild(child);                  // older way
```

## Event Handling

```js
const button = document.getElementById("submit-btn");

button.addEventListener("click", () => {
  console.log("Clicked!");
});

const form = document.getElementById("login-form");
form.addEventListener("submit", function (event) {
  event.preventDefault(); // stop page refresh
  const email = document.getElementById("email").value;
  console.log("Email:", email);
});

const search = document.getElementById("search");
search.addEventListener("input", () => {
  console.log("Searching:", search.value);
});
```

## Mini Project: Todo List

```js
function addTodo() {
  const input = document.getElementById("todo-input");
  const text = input.value.trim();
  if (!text) return;

  const li = document.createElement("li");
  li.textContent = text;

  const deleteBtn = document.createElement("button");
  deleteBtn.textContent = "Delete";
  deleteBtn.onclick = () => li.remove();

  li.appendChild(deleteBtn);
  document.getElementById("todo-list").appendChild(li);
  input.value = "";
}
```

## Interview Questions

- **Q: What is the DOM?**
  A: A tree-like representation of all HTML elements that JavaScript can read and modify.

- **Q: `textContent` vs `innerHTML`?**
  A: `textContent` is safe (no HTML parsing). `innerHTML` parses tags but can be an XSS risk with user input.

- **Q: `getElementById` vs `querySelector`?**
  A: `getElementById` selects by ID (faster). `querySelector` uses any CSS selector (more flexible).

- **Q: What does `event.preventDefault()` do?**
  A: Stops browser default behavior â€” e.g., prevents form from refreshing the page.

- **Q: What is event bubbling?**
  A: An event on a child element propagates up to parent elements. Stop it with `event.stopPropagation()`.

- **Q: `appendChild` vs `prepend`?**
  A: `appendChild` adds as last child; `prepend` adds as first child.

- **Q: How do you remove an element?**
  A: `element.remove()` (modern) or `parent.removeChild(element)`.
