# Debouncing & Throttling

## Simple Explanation
Both are techniques to **control how often a function runs** — especially for expensive operations triggered by rapid user actions (typing, scrolling, resizing).

- **Debouncing** = Wait until the user **stops** doing the action, then fire once.
- **Throttling** = Fire at most once every X milliseconds, no matter how fast the user acts.

---

## Real-World Example
- **Debounce** → Search bar: Don't hit the API on every keystroke. Wait until the user stops typing for 300ms.
- **Throttle** → Scroll event: Don't run the scroll handler 60 times/second. Run it at most once every 200ms.

---

## Debouncing

```js
function debounce(fn, delay) {
  let timer;

  return function (...args) {
    clearTimeout(timer); // reset the timer every time the function is called
    timer = setTimeout(() => {
      fn(...args); // only fires after delay ms of inactivity
    }, delay);
  };
}

// Usage: Search input
const handleSearch = debounce((query) => {
  console.log("Searching for:", query);
  // fetch(`/api/search?q=${query}`)
}, 300);

document.getElementById("search-input").addEventListener("input", (e) => {
  handleSearch(e.target.value);
});
// Even if user types 10 characters fast, API is only called once (300ms after they stop)
```

---

## Throttling

```js
function throttle(fn, limit) {
  let lastCall = 0;

  return function (...args) {
    const now = Date.now();

    if (now - lastCall >= limit) {
      lastCall = now;
      fn(...args); // only fires if enough time has passed
    }
  };
}

// Usage: Scroll event
const handleScroll = throttle(() => {
  console.log("Scroll position:", window.scrollY);
}, 200);

window.addEventListener("scroll", handleScroll);
// Fires at most once every 200ms, regardless of scroll speed
```

---

## Key Difference

| | Debounce | Throttle |
|--|---------|---------|
| When it fires | After user **stops** for X ms | At most once every X ms |
| Use case | Search box, form validation | Scroll, resize, button spam |
| Fires how many times | Once (at end) | Many times (regularly spaced) |

## Interview Questions

- **Q: What is debouncing?**
  A: A technique that delays function execution until a user has stopped triggering an event for a set period. Only fires once at the end.

- **Q: What is throttling?**
  A: A technique that limits how often a function can run. It fires at most once per defined time interval, no matter how many events occur.

- **Q: When would you use debounce over throttle?**
  A: Debounce: when you only care about the final result (search box, auto-save). Throttle: when you need regular updates (scroll position, resize).

- **Q: How does debounce work internally?**
  A: It uses `clearTimeout` + `setTimeout`. Every new call resets the timer, so the function only runs after the timer completes without interruption.

- **Q: How does throttle work internally?**
  A: It stores the last execution time. On each call, if enough time has passed since the last run, it executes the function and updates the timestamp.

- **Q: What are performance benefits of debounce/throttle?**
  A: They prevent expensive operations (API calls, DOM updates) from running too frequently, reducing CPU usage and improving UX.
