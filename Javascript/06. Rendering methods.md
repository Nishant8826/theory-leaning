# SSR
### Server-Side Rendering
SSR (Server-Side Rendering) generates the HTML on the server for every request. The browser receives a fully rendered page.  
**Best for:** SEO-heavy pages, dynamic content that changes per request.

---

# CSR
### Client-Side Rendering
CSR (Client-Side Rendering) renders content in the browser using JavaScript. The server only sends a minimal HTML shell.  
**Best for:** Web apps with lots of user interactions (e.g., dashboards, SPAs).

---

# SSG
### Static-Site Generation
SSG (Static-Site Generation) generates HTML at build time. The same static page is served to all users.  
**Best for:** Blogs, documentation, marketing sites with content that rarely changes.

---

# ISR
### Incremental Static Regeneration
ISR (Incremental Static Regeneration) allows static pages to be regenerated in the background after a set time interval.  
**Best for:** SSG sites that need periodic updates (e.g., product listings, news).
