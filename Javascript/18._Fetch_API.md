# Fetch API / Making API Calls

## Simple Explanation
The **Fetch API** is a built-in browser function that lets you make **HTTP requests** — like asking a server for data (GET) or sending data to it (POST).

Think of it like ordering food through an app:
- You make a request (order food).
- The server processes it.
- You get back a response (your food or an error).

---

## Real-World Example
Loading a list of products from a backend, submitting a login form to a server, or fetching weather data from a public API.

---

## Basic GET Request

```js
fetch("https://jsonplaceholder.typicode.com/posts/1")
  .then((response) => response.json())
  .then((data) => console.log(data))
  .catch((error) => console.error("Error:", error));
```

---

## Using Async/Await (Recommended)

```js
async function getPost() {
  try {
    const response = await fetch("https://jsonplaceholder.typicode.com/posts/1");

    if (!response.ok) {
      throw new Error(`HTTP Error: ${response.status}`);
    }

    const post = await response.json();
    console.log("Title:", post.title);
  } catch (error) {
    console.error("Failed:", error.message);
  }
}

getPost();
```

## POST Request (Send Data)

```js
async function createPost() {
  const newPost = {
    title: "Learning Fetch API",
    body: "Fetch API is super useful",
    userId: 1,
  };

  try {
    const response = await fetch("https://jsonplaceholder.typicode.com/posts", {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
      },
      body: JSON.stringify(newPost), // convert object to JSON string
    });

    const data = await response.json();
    console.log("Created:", data);
  } catch (error) {
    console.error("Error:", error.message);
  }
}

createPost();
```

## HTTP Methods Summary

| Method | Purpose | Example |
|--------|---------|---------|
| `GET` | Read data | Fetch user profile |
| `POST` | Create data | Submit form, register |
| `PUT` | Update data (replace all) | Update entire user record |
| `PATCH` | Update data (partial) | Update just the email |
| `DELETE` | Delete data | Remove a post |

## Fetch with Auth Header (JWT)

```js
async function getProtectedData() {
  const token = localStorage.getItem("token");

  const response = await fetch("https://api.example.com/profile", {
    method: "GET",
    headers: {
      "Authorization": `Bearer ${token}`,
      "Content-Type": "application/json",
    },
  });

  const data = await response.json();
  console.log(data);
}
```

## Checking Response Status

```js
async function safeGet(url) {
  const response = await fetch(url);

  // response.ok is true for 200–299 status codes
  if (!response.ok) {
    throw new Error(`Error ${response.status}: ${response.statusText}`);
  }

  return response.json();
}
```

## Fetching from Multiple APIs at Once

```js
async function loadDashboard() {
  try {
    const [user, posts, notifications] = await Promise.all([
      fetch("/api/user").then((r) => r.json()),
      fetch("/api/posts").then((r) => r.json()),
      fetch("/api/notifications").then((r) => r.json()),
    ]);

    console.log(user, posts, notifications);
  } catch (error) {
    console.error("Dashboard load failed:", error.message);
  }
}
```

## Interview Questions

- **Q: What is the Fetch API?**
  A: A built-in browser function for making HTTP requests. It returns a Promise that resolves to the Response object.

- **Q: What does `response.ok` mean?**
  A: It's `true` for HTTP status codes 200–299 (success). Use it to check if the request was successful before processing data.

- **Q: Why do we need `response.json()`?**
  A: `fetch()` returns a raw Response. `.json()` parses the body as JSON and returns another Promise.

- **Q: What is the difference between `fetch` and `XMLHttpRequest`?**
  A: `fetch` is modern, cleaner, and Promise-based. `XMLHttpRequest` is older, verbose, and uses callbacks.

- **Q: How do you send data with Fetch?**
  A: Set `method: "POST"`, add `Content-Type: application/json` header, and pass `body: JSON.stringify(data)`.

- **Q: Does `fetch` throw an error on 404?**
  A: No! Fetch only rejects on network failure. A 404 still resolves — you must check `response.ok` manually.

- **Q: How do you cancel a fetch request?**
  A: Use `AbortController` — call `controller.abort()` and pass `signal: controller.signal` in the fetch options.
