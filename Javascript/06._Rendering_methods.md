# Rendering Methods (SSR, CSR, SSG, ISR)

When you visit a website, the HTML page you see has to come from somewhere. **Rendering methods** define **where** and **when** that HTML is generated.

---

## CSR — Client-Side Rendering

### What it is
The browser downloads a minimal HTML shell + JavaScript bundle. JavaScript runs in the browser and builds the page content.

**Best for:** Dashboards, SPAs (Single Page Apps), apps with lots of user interaction.

### How it works
```
Browser → Server: "Give me the homepage"
Server → Browser: Empty HTML + JS bundle
Browser: Runs JS → Builds and displays content
```

### Code Concept (React / Vite)
```html
<!-- Initial HTML from server (almost empty) -->
<div id="root"></div>
<script src="app.js"></script>
<!-- JavaScript fills the #root div with content -->
```

**Pros:**
- Fast after initial load (navigation is instant)
- Great for interactive UIs

**Cons:**
- Slow initial load (user sees blank screen first)
- Bad for SEO (crawlers may not execute JS)

---

## SSR — Server-Side Rendering

### What it is
The server generates a **full HTML page** for every request and sends it to the browser. The browser just displays it.

**Best for:** SEO-heavy pages, dynamic content that changes per user/request (e.g., product pages, news articles).

### How it works
```
Browser → Server: "Give me the homepage"
Server: Fetches data + renders HTML
Server → Browser: Fully built HTML page
Browser: Shows page immediately
```

**Pros:**
- Great SEO (crawlers see full content)
- Faster first page load

**Cons:**
- Every request hits the server (more server load)
- Slightly slower navigation from CSR

---

## SSG — Static Site Generation

### What it is
HTML is generated **at build time** (before anyone visits). The same pre-built HTML is served to every user.

**Best for:** Blogs, documentation, marketing landing pages — content that rarely changes.

### How it works
```
Build time: HTML files are generated once
Browser → Server: "Give me the homepage"
Server → Browser: Pre-built HTML (from CDN or file server)
Browser: Shows page instantly
```

**Pros:**
- Blazing fast (served from CDN)
- No server computation per request
- Great SEO

**Cons:**
- Stale data (you need to rebuild to update content)

---

## ISR — Incremental Static Regeneration

### What it is
Like SSG, but pages can be **regenerated in the background** after a set time interval — without a full rebuild.

**Best for:** SSG sites that need periodic updates (e.g., product listings, news, prices).

### How it works (Next.js example)
```js
export async function getStaticProps() {
  const data = await fetch("https://api.example.com/products");
  return {
    props: { data },
    revalidate: 60, // Regenerate page every 60 seconds
  };
}
```

**Pros:**
- Combines speed of SSG with freshness of SSR
- No full rebuild needed

**Cons:**
- First visit after revalidation period hits the old page (stale-while-revalidate)

---

## Quick Comparison Table

| Feature | CSR | SSR | SSG | ISR |
|---------|-----|-----|-----|-----|
| HTML Generated | In Browser | On Server (per request) | At Build Time | At Build Time + Revalidate |
| SEO | ❌ Poor | ✅ Great | ✅ Great | ✅ Great |
| Speed (First Load) | 🟡 OK | ✅ Fast | ✅ Fastest | ✅ Fast |
| Dynamic Data | ✅ Yes | ✅ Yes | ❌ No | 🟡 Periodic |
| Server Load | 🟡 Low | ❌ High | ✅ None | ✅ Low |

## Interview Questions

- **Q: What is CSR?**
  A: Client-Side Rendering — HTML is built by JavaScript in the browser. The server sends minimal HTML + a JS bundle.

- **Q: What is SSR?**
  A: Server-Side Rendering — the server generates a full HTML page on every request before sending it to the browser.

- **Q: What is SSG?**
  A: Static-Site Generation — HTML is built once at build time and served as static files to all users.

- **Q: What is ISR?**
  A: Incremental Static Regeneration — like SSG, but pages are automatically regenerated after a configured time interval.

- **Q: Which rendering method is best for SEO?**
  A: SSR, SSG, and ISR — all three provide pre-rendered HTML that search engine crawlers can read. CSR is poor for SEO.

- **Q: What is hydration?**
  A: After SSR/SSG sends pre-built HTML to the browser, React "hydrates" it — attaches event listeners and makes it interactive, without re-rendering the whole page.

- **Q: Which rendering method would you use for a dashboard?**
  A: CSR — dashboards are user-specific, highly interactive, and don't need SEO.
