# The `this` Keyword

## Simple Explanation
`this` refers to the **object that is currently executing the code**. It's like saying "me" — it always refers to whoever is speaking (calling the function).

The tricky part: `this` changes depending on **how** a function is called.

---

## Real-World Example
In a restaurant:
- A waiter says `this.table` — "this" refers to the table they're currently serving.
- The same waiter at a different table → `this.table` is different.

---

## `this` in Different Contexts

### 1. Global Context
In the browser's global scope, `this` refers to the `window` object.
```js
console.log(this); // Window object (in browser)
```

### 2. Inside a Regular Function
```js
function show() {
  console.log(this); // In browser: Window object
                     // In strict mode: undefined
}
show();
```

### 3. Inside an Object Method ✅ Most common use
```js
const user = {
  name: "Nishant",
  greet() {
    console.log(`Hi, I'm ${this.name}`); // this = user object
  },
};

user.greet(); // "Hi, I'm Nishant"
```

### 4. Arrow Function — `this` is NOT its own
Arrow functions **don't have their own `this`**. They borrow `this` from their surrounding scope.

```js
const user = {
  name: "Nishant",
  greet: () => {
    console.log(`Hi, I'm ${this.name}`); // this = Window (not user!)
  },
};

user.greet(); // "Hi, I'm undefined" ❌
```

### 5. Inside a Class
```js
class Car {
  constructor(brand) {
    this.brand = brand; // this = the new Car instance
  }

  info() {
    return `Car brand: ${this.brand}`;
  }
}

const myCar = new Car("Toyota");
console.log(myCar.info()); // "Car brand: Toyota"
```

## `this` Binding Methods

### `call()` — call with specific `this` and individual args
```js
function introduce(city, country) {
  console.log(`Hi, I'm ${this.name} from ${city}, ${country}`);
}

const person = { name: "Nishant" };
introduce.call(person, "Delhi", "India");
// "Hi, I'm Nishant from Delhi, India"
```

### `apply()` — same as `call()` but args as an array
```js
introduce.apply(person, ["Delhi", "India"]);
// "Hi, I'm Nishant from Delhi, India"
```

### `bind()` — create a new function with `this` permanently bound
```js
const boundIntroduce = introduce.bind(person, "Delhi", "India");
boundIntroduce(); // "Hi, I'm Nishant from Delhi, India"

// Useful for event handlers
const button = document.getElementById("btn");
button.addEventListener("click", myCar.info.bind(myCar));
```

## Common `this` Trap — Lost Context
```js
const user = {
  name: "Nishant",
  greet() {
    setTimeout(function () {
      console.log(this.name); // ❌ undefined — regular function loses this
    }, 1000);
  },
};

// ✅ Fix 1: Arrow function (inherits this from greet)
const user2 = {
  name: "Nishant",
  greet() {
    setTimeout(() => {
      console.log(this.name); // ✅ "Nishant"
    }, 1000);
  },
};

// ✅ Fix 2: Save reference
const user3 = {
  name: "Nishant",
  greet() {
    const self = this;
    setTimeout(function () {
      console.log(self.name); // ✅ "Nishant"
    }, 1000);
  },
};
```

## Quick Summary Table

| Context | `this` refers to |
|---------|-----------------|
| Global scope | `window` (browser) / `global` (Node) |
| Regular function (non-strict) | `window` |
| Regular function (strict mode) | `undefined` |
| Object method | The object |
| Arrow function | Enclosing scope's `this` |
| Class constructor | New instance |
| `call/apply/bind` | Whatever you pass in |

## Interview Questions

- **Q: What is `this` in JavaScript?**
  A: `this` refers to the object that is currently executing a function. Its value depends on how the function is called.

- **Q: What does `this` refer to in an arrow function?**
  A: Arrow functions don't have their own `this`. They inherit `this` from their lexical (enclosing) scope.

- **Q: What is the difference between `call`, `apply`, and `bind`?**
  A: All three let you set `this` manually. `call` passes args individually, `apply` passes as array, `bind` returns a new function with `this` permanently set.

- **Q: Why does `this` lose its context in `setTimeout`?**
  A: `setTimeout` calls the callback as a regular function, not as a method — so `this` defaults to `window`. Fix with arrow functions or `bind`.

- **Q: What is `this` in strict mode?**
  A: In strict mode, `this` inside a regular function that's not called as a method is `undefined` (not `window`).

- **Q: Can you change what `this` is inside an arrow function?**
  A: No. Arrow functions permanently inherit `this` from their enclosing scope — you cannot override it with `call`, `apply`, or `bind`.
