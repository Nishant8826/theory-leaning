# Currying

## Simple Explanation
**Currying** is a technique in functional programming where a function that takes **multiple arguments** is transformed into a sequence of functions that each take **one argument at a time**.

Instead of `add(2, 3)`, you do `add(2)(3)`.

Think of it like a checkout process:
- Step 1: Enter your email
- Step 2: Enter your address
- Step 3: Enter payment

Each step is separate, and you can pause between them.

---

## Real-World Example
Imagine a discount function:
- You want to apply a 10% discount to multiple products.
- Instead of passing the discount % every time, you "pre-configure" it once.

---

## Code Example

### Basic Currying
```js
// Normal function
const add = (a, b) => a + b;
console.log(add(2, 3)); // 5

// Curried version
const curriedAdd = (a) => (b) => a + b;

console.log(curriedAdd(2)(3)); // 5

// Reuse the first argument
const addFive = curriedAdd(5);
console.log(addFive(10)); // 15
console.log(addFive(20)); // 25
```

### Real-World: Discount Calculator
```js
const applyDiscount = (discountPercent) => (price) => {
  const discountAmount = (price * discountPercent) / 100;
  return price - discountAmount;
};

const tenPercentOff = applyDiscount(10);
const fiftyPercentOff = applyDiscount(50);

console.log(tenPercentOff(100));    // 90
console.log(tenPercentOff(200));    // 180
console.log(fiftyPercentOff(100));  // 50
```

### Currying with 3 arguments
```js
const greet = (greeting) => (time) => (name) => {
  return `${greeting}! Good ${time}, ${name}.`;
};

const hello = greet("Hello")("Morning");
console.log(hello("Nishant")); // "Hello! Good Morning, Nishant."
console.log(hello("Raj"));     // "Hello! Good Morning, Raj."
```

## Why Use Currying?
| Benefit | Example |
|---------|---------|
| **Reusability** | Pre-configure part of a function |
| **Cleaner code** | Avoid repeating the same arguments |
| **Function composition** | Build complex logic from simple functions |
| **Partial application** | Lock in some arguments, supply rest later |

---

## Generic Curry Function
```js
function curry(fn) {
  return function curried(...args) {
    if (args.length >= fn.length) {
      return fn(...args);
    }
    return (...moreArgs) => curried(...args, ...moreArgs);
  };
}

const sum = (a, b, c) => a + b + c;
const curriedSum = curry(sum);

console.log(curriedSum(1)(2)(3)); // 6
console.log(curriedSum(1, 2)(3)); // 6
console.log(curriedSum(1)(2, 3)); // 6
```

## Interview Questions

- **Q: What is currying?**
  A: Currying transforms a function with multiple arguments into a series of functions each accepting one argument. Example: `f(a, b, c)` becomes `f(a)(b)(c)`.

- **Q: What is the difference between currying and partial application?**
  A: Currying always returns functions that take **one argument at a time**. Partial application pre-fills **some** (not necessarily one) arguments, returning a function for the rest.

- **Q: Why is currying useful?**
  A: It enables function reuse, avoids repeated arguments, and is the foundation of functional programming patterns.

- **Q: Does currying change what a function does?**
  A: No. The result is the same — only the way arguments are passed changes.

- **Q: Is currying related to closures?**
  A: Yes! Curried functions use closures to remember the arguments passed in previous calls.

- **Q: When would you use currying in a real project?**
  A: Logging functions (pre-set the module name), discount calculation (pre-set the discount %), event handling (pre-set the event type), etc.
