# Shallow Copy vs Deep Copy

When you copy an object or array in JavaScript, there are two types of copies:

- **Shallow Copy** = Copy the surface only. Nested objects are still **share** same reference.
- **Deep Copy** = Copy everything, including nested objects — **fully independent**.

---

## Shallow Copy

A shallow copy creates a **new object/array** but only copies the **top-level** properties. Nested objects are **copied by reference**, not by value.

### Primitive values (always copied independently):
```js
let a = 5;
let b = a;
b += 5;

console.log(a, b); // 5  10 — completely separate!
```

### Objects (shallow copy — nested is still shared):
```js
let original = {
  name: "Nishant",
  address: { city: "Delhi" }, // nested object
};

let shallowCopy = { ...original }; // spread operator

// Change top-level property → does NOT affect original
shallowCopy.name = "Raj";
console.log(original.name); // "Nishant" ✅ safe

// Change nested property → AFFECTS original!
shallowCopy.address.city = "Mumbai";
console.log(original.address.city); // "Mumbai" ❌ changed!
```

### Ways to create shallow copies:
```js
const obj = { a: 1, nested: { b: 2 } };

// 1. Spread operator
const copy1 = { ...obj };

// 2. Object.assign()
const copy2 = Object.assign({}, obj);

// For arrays
const arr = [1, 2, [3, 4]];
const arrCopy1 = [...arr];
const arrCopy2 = arr.slice();
const arrCopy3 = Array.from(arr);
```

---

## Deep Copy

A deep copy creates **completely independent copies at every level** — no shared references.

### Ways to create deep copies:

**1. JSON.parse + JSON.stringify (most common, but limited)**
```js
let original = {
  name: "Nishant",
  address: { city: "Delhi" },
};

let deepCopy = JSON.parse(JSON.stringify(original));

deepCopy.address.city = "Mumbai";
console.log(original.address.city); // "Delhi" ✅ not affected!

// ⚠️ Limitation: Cannot copy functions, undefined, Date, Map, Set, etc.
```

**2. structuredClone() (modern, recommended)**
```js
let original = { name: "Nishant", scores: [10, 20, 30] };
let deepCopy = structuredClone(original);

deepCopy.scores.push(40);
console.log(original.scores); // [10, 20, 30] ✅ not affected!
```

**3. Lodash (popular library)**
```js
import _ from "lodash";
const deepCopy = _.cloneDeep(original);
```

---

## Comparison Table

| Feature | Shallow Copy | Deep Copy |
|---------|-------------|-----------|
| Nested objects | Shared reference ❌ | Fully copied ✅ |
| Top-level values | Independent ✅ | Independent ✅ |
| Performance | Fast ✅ | Slower ❌ |
| Methods to use | `{...obj}`, `Object.assign()`, `.slice()` | `JSON.parse/stringify`, `structuredClone()` |
| Handles functions | Yes | JSON method ❌, structuredClone ❌ |

## Interview Questions

- **Q: What is a shallow copy?**
  A: A copy where only the top-level properties are duplicated. Nested objects/arrays still share the same memory reference as the original.

- **Q: What is a deep copy?**
  A: A fully independent copy at all levels — changes to the copy do not affect the original, even for nested objects.

- **Q: What is the difference between `=` (assignment) and spread `{...obj}`?**
  A: `=` doesn't copy — it creates another reference to the same object. `{...obj}` creates a shallow copy (new object, but nested objects still shared).

- **Q: What are the limitations of `JSON.parse(JSON.stringify(obj))`?**
  A: It cannot handle functions, `undefined`, `Date` objects, `Map`, `Set`, circular references, or `Symbol` keys.

- **Q: What is `structuredClone()`?**
  A: A modern browser/Node API for deep cloning objects. It handles more types than JSON but doesn't support functions.

- **Q: When should you use deep copy over shallow copy?**
  A: Use deep copy when you have nested objects/arrays and need to ensure changes to the copy don't affect the original data.
