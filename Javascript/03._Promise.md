# Promises

## Simple Explanation
A **Promise** is like ordering food at a restaurant:
- You place the order (create a promise).
- The kitchen prepares it (async operation runs).
- Either you get your food ✅ (resolved) or they say "sorry, out of stock" ❌ (rejected).

A Promise represents a value that **will be available in the future** — it's a placeholder for the result of an async operation.

---

## Real-World Example
Fetching user data from a server:
- You ask the server for data (promise is **pending**).
- Server responds with data (**fulfilled**) or an error (**rejected**).

---

## States of a Promise
| State | Meaning |
|-------|---------|
| **Pending** | Operation is still running |
| **Fulfilled** | Operation completed successfully |
| **Rejected** | Operation failed |

> 📌 Once a promise is fulfilled or rejected, it **cannot change** state.

---

## Code Example

### Creating a Promise
```js
const orderFood = new Promise((resolve, reject) => {
  const foodAvailable = true;

  if (foodAvailable) {
    resolve("Here is your burger! 🍔");
  } else {
    reject("Sorry, we're out of stock.");
  }
});
```

### Consuming a Promise
```js
orderFood
  .then((message) => {
    console.log("Success:", message); // "Here is your burger! 🍔"
  })
  .catch((error) => {
    console.log("Error:", error);     // "Sorry, we're out of stock."
  })
  .finally(() => {
    console.log("Order process done."); // Always runs
  });
```

## Promise Chaining
When you need to do async tasks **one after another**, you chain `.then()` calls.

```js
fetch("https://jsonplaceholder.typicode.com/users/1")
  .then((response) => response.json())      // Step 1: parse JSON
  .then((user) => {
    console.log("User:", user.name);        // Step 2: use user data
    return user.id;
  })
  .then((id) => {
    console.log("User ID:", id);            // Step 3: use the ID
  })
  .catch((error) => {
    console.error("Something went wrong:", error);
  });
```

## Promise Methods

### `Promise.all()` — Wait for ALL to finish
```js
const p1 = Promise.resolve("Data 1");
const p2 = Promise.resolve("Data 2");
const p3 = Promise.resolve("Data 3");

Promise.all([p1, p2, p3]).then((results) => {
  console.log(results); // ["Data 1", "Data 2", "Data 3"]
});
// ❌ If ANY promise rejects, the whole thing fails
```

### `Promise.allSettled()` — Wait for ALL, regardless of result
```js
Promise.allSettled([p1, p2, Promise.reject("Error")]).then((results) => {
  results.forEach((r) => console.log(r.status)); // fulfilled / rejected
});
```

### `Promise.race()` — First one wins
```js
Promise.race([p1, p2, p3]).then((result) => {
  console.log(result); // Whichever resolves first
});
```

## Interview Questions

- **Q: What is a Promise?**
  A: A Promise is an object representing the eventual completion (or failure) of an async operation. It can be pending, fulfilled, or rejected.

- **Q: What are the 3 states of a Promise?**
  A: Pending (running), Fulfilled (success), Rejected (failure). Once settled, it can't change.

- **Q: What is the difference between `.then()` and `.catch()`?**
  A: `.then()` handles success, `.catch()` handles errors. `.then()` can also take a second argument for rejection handling.

- **Q: What is Promise chaining?**
  A: Calling multiple `.then()` methods in sequence, where each returns a new Promise, allowing sequential async operations.

- **Q: What is `Promise.all()`?**
  A: It takes an array of promises and resolves when **all** resolve. If any one rejects, it immediately rejects.

- **Q: What is the difference between `Promise.all()` and `Promise.allSettled()`?**
  A: `Promise.all()` fails fast if any promise rejects. `Promise.allSettled()` waits for all promises and gives results for each, regardless of success or failure.

- **Q: What does `.finally()` do?**
  A: It runs after the promise is settled (either fulfilled or rejected). Used for cleanup tasks like hiding a loading spinner.
