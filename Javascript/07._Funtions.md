# Functions

## Simple Explanation
A **function** is a reusable block of code that performs a specific task. Instead of writing the same code over and over, you write it once in a function and call it whenever you need it.

Think of a function as a **vending machine** — you put in input (money + choice), and it gives you output (the snack).

---

## Real-World Example
A login form:
- You need to validate the email format in multiple places.
- Instead of copy-pasting validation logic, write a `validateEmail(email)` function once and call it anywhere.

---

## Code Example

```js
function validateEmail(email) {
  return email.includes("@");
}

console.log(validateEmail("nishant@gmail.com")); // true
console.log(validateEmail("notanemail"));         // false
```

---

## Function Declaration

The most common and traditional way to define a function. Also called a **regular function**.

- ✅ **Hoisted** — can be called before it's defined in the code.
- `this` binding depends on how it's called.

```js
greet("Nishant"); // ✅ Works! — function is hoisted

function greet(name) {
  return `Hello, ${name}!`;
}
```

---

## Function Expression

A function assigned to a variable. Also called an **anonymous function**.

- ❌ **Not hoisted** — cannot be called before the line where it's defined.
- `this` binding depends on how it's called.

```js
// greet("Nishant"); // ❌ Error — cannot access before initialization

const greet = function (name) {
  return `Hello, ${name}!`;
};

greet("Nishant"); // ✅ Works after definition
```

---

## Arrow Function

Introduced in **ES6** — a shorter syntax for writing functions.

- ❌ **Not hoisted**.
- ❌ **No own `this`** — inherits `this` from the surrounding (parent) scope. This makes them great for callbacks.
- ❌ **Cannot be used as constructors**.

```js
// Concise (single expression — implicit return)
const add = (a, b) => a + b;
console.log(add(2, 3)); // 5

// With block body (explicit return needed)
const multiply = (a, b) => {
  const result = a * b;
  return result;
};
console.log(multiply(4, 5)); // 20

// Single parameter — no parentheses needed
const double = n => n * 2;
console.log(double(7)); // 14
```

---

## Comparison Table

| Feature | Declaration | Expression | Arrow |
|---------|------------|------------|-------|
| Hoisted | ✅ Yes | ❌ No | ❌ No |
| Own `this` | ✅ Yes | ✅ Yes | ❌ No (inherits) |
| Can be constructor | ✅ Yes | ✅ Yes | ❌ No |
| Syntax | `function foo(){}` | `const foo = function(){}` | `const foo = () => {}` |

## Default Parameters
```js
function greet(name = "Guest") {
  return `Hello, ${name}!`;
}

console.log(greet());          // "Hello, Guest!"
console.log(greet("Nishant")); // "Hello, Nishant!"
```

## Rest Parameters
```js
function sum(...numbers) {
  return numbers.reduce((total, n) => total + n, 0);
}

console.log(sum(1, 2, 3, 4, 5)); // 15
```

## IIFE — Immediately Invoked Function Expression
A function that runs **immediately** after being defined. Useful for creating private scope.

```js
(function () {
  const secret = "I run immediately!";
  console.log(secret);
})();
// "I run immediately!"
```

## Interview Questions

- **Q: What is the difference between function declaration and function expression?**
  A: Declarations are hoisted (can be called before definition). Expressions are not — they must be defined before calling.

- **Q: What is an arrow function and how does it differ from a regular function?**
  A: Arrow functions have a shorter syntax and don't have their own `this`, `arguments`, or `prototype`. They inherit `this` from their enclosing scope.

- **Q: What is hoisting in the context of functions?**
  A: Function declarations are fully hoisted — both name and body. Function expressions and arrow functions are not hoisted.

- **Q: What is an IIFE?**
  A: An Immediately Invoked Function Expression — a function that runs immediately after it's defined. Used to create isolated scope.

- **Q: What are rest parameters?**
  A: `...paramName` allows a function to accept any number of arguments as an array.

- **Q: Can arrow functions be used as constructors?**
  A: No. They don't have a `prototype` property and will throw an error if used with `new`.

- **Q: When should you use an arrow function vs a regular function?**
  A: Use arrow functions for callbacks, array methods, and when you want to inherit `this`. Use regular functions when you need their own `this` (e.g., object methods, constructors).
