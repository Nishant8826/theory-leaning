# Callbacks

## Simple Explanation
A **callback** is a function you pass as an **argument to another function**, and it gets called later — when the first function is done with its job.

Think of it like calling a plumber:
- You tell them "fix the pipe, and **call me when you're done**."
- That "call me when done" = callback.

---

## Real-World Example
When you click a button on a website, the click event fires and calls your **callback function** that handles it.

---

## Code Example

### Basic Callback
```js
function doWork(task, callback) {
  console.log(`Working on: ${task}`);
  callback(); // call the function passed in
}

function whenDone() {
  console.log("Task is complete!");
}

doWork("Building UI", whenDone);
// "Working on: Building UI"
// "Task is complete!"
```

### Callback with Data
```js
function fetchUser(id, onSuccess, onError) {
  if (id === 1) {
    onSuccess({ name: "Nishant", age: 22 });
  } else {
    onError("User not found");
  }
}

fetchUser(
  1,
  (user) => console.log("Got user:", user.name), // success callback
  (err) => console.error("Error:", err)           // error callback
);
// "Got user: Nishant"
```

### Common Built-in Callbacks
```js
// Array methods use callbacks
const numbers = [1, 2, 3, 4, 5];

const doubled = numbers.map((n) => n * 2);
console.log(doubled); // [2, 4, 6, 8, 10]

const evens = numbers.filter((n) => n % 2 === 0);
console.log(evens); // [2, 4]

// setTimeout — passes a callback to be called after a delay
setTimeout(() => {
  console.log("This runs after 2 seconds");
}, 2000);

// Event listener
document.getElementById("btn").addEventListener("click", () => {
  console.log("Button clicked!");
});
```

## Synchronous vs Asynchronous Callbacks

### Synchronous Callback — runs immediately
```js
const result = [5, 3, 8, 1].sort((a, b) => a - b);
console.log(result); // [1, 3, 5, 8]
```

### Asynchronous Callback — runs later
```js
console.log("Start");
setTimeout(() => {
  console.log("I run after 1 second"); // async callback
}, 1000);
console.log("End");
// Output: Start → End → I run after 1 second
```

## Callback Hell (The Problem)
When you nest many callbacks inside each other, you get deeply nested, hard-to-read code — called **"callback hell"** or **"pyramid of doom"**.

```js
// ❌ Callback hell — hard to read and maintain
login(user, function (token) {
  fetchProfile(token, function (profile) {
    fetchPosts(profile.id, function (posts) {
      renderPosts(posts, function () {
        console.log("Done!");
      });
    });
  });
});
```

### Solution: Use Promises or Async/Await
```js
// ✅ Clean version with Promises
login(user)
  .then(fetchProfile)
  .then((profile) => fetchPosts(profile.id))
  .then(renderPosts)
  .then(() => console.log("Done!"))
  .catch(console.error);
```

## Interview Questions

- **Q: What is a callback function?**
  A: A function passed as an argument to another function, to be executed later — either immediately (sync) or after an async operation completes.

- **Q: What is the difference between synchronous and asynchronous callbacks?**
  A: Synchronous callbacks run immediately (e.g., `array.map()`). Asynchronous callbacks run later, after some delay or event (e.g., `setTimeout`, `fetch`).

- **Q: What is callback hell?**
  A: When multiple callbacks are deeply nested inside each other, creating a "pyramid" of indented code that's hard to read and debug.

- **Q: How do you avoid callback hell?**
  A: Use Promises (`.then()/.catch()`) or `async/await`, which flatten the nested structure into readable sequential code.

- **Q: Are callbacks only for async operations?**
  A: No. Array methods like `.map()`, `.filter()`, `.sort()` use synchronous callbacks.

- **Q: What is a higher-order function?**
  A: A function that accepts another function as an argument (or returns one). Callbacks make this possible.
