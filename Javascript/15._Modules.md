# Modules (import / export)

**Modules** let you split your JavaScript code into multiple files and **share** code between them.

---

## Named Exports & Imports

### Exporting (named)
```js
// utils.js
export function formatPrice(price) {
  return `$${price.toFixed(2)}`;
}

export function capitalize(str) {
  return str.charAt(0).toUpperCase() + str.slice(1);
}

export const TAX_RATE = 0.1;
```

### Importing (named)
```js
// app.js
import { formatPrice, capitalize, TAX_RATE } from "./utils.js";

console.log(formatPrice(9.99));     // "$9.99"
console.log(capitalize("nishant")); // "Nishant"
console.log(TAX_RATE);              // 0.1
```

### Renaming on import
```js
import { formatPrice as fp } from "./utils.js";
console.log(fp(14.5)); // "$14.50"
```

## Default Exports & Imports

Each file can have **one** default export.

### Exporting (default)
```js
// logger.js
export default function log(message) {
  console.log(`[LOG]: ${message}`);
}
```

### Importing (default)
```js
// You can name it anything
import log from "./logger.js";
import myLogger from "./logger.js"; // also valid

log("User logged in");
```

## Named + Default Together
```js
// api.js
export const BASE_URL = "https://api.example.com";

export function getHeaders() {
  return { "Content-Type": "application/json" };
}

export default async function fetchData(endpoint) {
  const response = await fetch(`${BASE_URL}/${endpoint}`);
  return response.json();
}
```

```js
// app.js
import fetchData, { BASE_URL, getHeaders } from "./api.js";
```

---

## Import All (`* as`)
```js
import * as utils from "./utils.js";

console.log(utils.formatPrice(5)); // "$5.00"
console.log(utils.TAX_RATE);       // 0.1
```

## CommonJS vs ES Modules

JavaScript has two module systems:

| Feature | CommonJS (Node.js) | ES Modules (Modern) |
|---------|-------------------|---------------------|
| Syntax | `require()`, `module.exports` | `import`, `export` |
| Loading | Synchronous | Asynchronous |
| Used in | Node.js (older) | Browsers, modern Node |
| File ext | `.js` | `.mjs` or `.js` with `"type": "module"` |

**CommonJS (old style)**
```js
// math.js
module.exports = { add: (a, b) => a + b };

// app.js
const { add } = require("./math.js");
```

**ES Modules (modern)**
```js
// math.js
export const add = (a, b) => a + b;

// app.js
import { add } from "./math.js";
```

## Dynamic Imports (Lazy Loading)

Load modules **on demand** — useful for code splitting and performance.

```js
async function loadChart() {
  const { default: Chart } = await import("./chart.js"); // loaded only when called
  new Chart();
}

button.addEventListener("click", loadChart);
```

## Interview Questions

- **Q: What are ES Modules?**
  A: A modern JavaScript system to split code into reusable files using `import` and `export` syntax.

- **Q: What is the difference between named export and default export?**
  A: Named exports: multiple per file, imported using `{}` with exact names. Default export: one per file, imported with any name without `{}`.

- **Q: What is the difference between ES Modules and CommonJS?**
  A: CommonJS uses `require`/`module.exports` (synchronous, Node.js). ES Modules use `import`/`export` (asynchronous, works in browsers and modern Node).

- **Q: Can you have both named and default exports in one file?**
  A: Yes. A file can have one default export and any number of named exports.

- **Q: What is dynamic import?**
  A: `import()` as a function — loads a module asynchronously at runtime (lazy loading). Returns a Promise.

- **Q: Why use modules?**
  A: Modules improve code organization, reusability, maintainability, encapsulation, and prevent global namespace pollution.
