# Objects & Prototypes

## Simple Explanation
An **object** in JavaScript is like a container that holds related information (properties) and actions (methods) together using **key-value pairs**.

Think of it like a person profile — they have a name, age, and can do things like greet or walk.

In JavaScript, almost everything is an object (arrays, functions, dates, etc.).

---

## Real-World Example
A user profile on a website:
- Properties: `name`, `email`, `age`
- Methods: `login()`, `logout()`

---

## Code Example

```js
// Creating an object
const user = {
  name: "Nishant",
  age: 22,
  email: "nishant@example.com",
  greet: function () {
    return `Hello, I'm ${this.name}`;
  },
};

console.log(user.name);       // "Nishant"
console.log(user.greet());    // "Hello, I'm Nishant"

// Adding a new property
user.city = "Delhi";

// Deleting a property
delete user.age;
```

## Object Creation Ways

### 1. Object Literal (most common)
```js
const car = { brand: "Toyota", model: "Camry" };
```

### 2. Function Constructor (before ES6)
```js
function Person(name, age) {
  this.name = name;
  this.age = age;
}
const p = new Person("Nishant", 22);
```

### 3. ES6 Classes (modern way)
```js
class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }
  greet() {
    return `Hi, I'm ${this.name}`;
  }
}
const p = new Person("Nishant", 22);
```

### 4. Object.create()
```js
const proto = { greet() { return `Hello, ${this.name}`; } };
const user = Object.create(proto);
user.name = "Nishant";
console.log(user.greet()); // "Hello, Nishant"
```

## Prototype

### Simple Explanation
Every object in JavaScript has a hidden link (`[[Prototype]]`) that points to another object — its **prototype**. When you try to access a property, JS first looks in the object itself, then goes up the prototype chain.

Think of it as: "If I don't have money, I'll ask my parent. If they don't have it, I'll ask my grandparent." — That's the **prototype chain**.

### Prototypal Inheritance
```js
const animal = {
  breathe() { return "Breathing..."; }
};

const dog = Object.create(animal);
dog.bark = function () { return "Woof!"; };

console.log(dog.bark());    // "Woof!" — found in dog itself
console.log(dog.breathe()); // "Breathing..." — found in animal (prototype)
console.log(dog.__proto__ === animal); // true
```

The chain goes: `dog → animal → Object.prototype → null`

### Shadowing
If an object has the same property as its prototype, it **shadows** (overrides) the prototype's version.

```js
const animal = { sound: "..." };
const cat = Object.create(animal);
cat.sound = "Meow"; // This shadows animal.sound

console.log(cat.sound); // "Meow" — uses cat's own property
```

## Inheritance (using Classes)

### Simple Explanation
Inheritance = a child class gets all properties and methods from a parent class. The child can also add its own stuff.

```js
class Animal {
  constructor(name) {
    this.name = name;
  }
  speak() {
    return `${this.name} makes a sound.`;
  }
}

class Dog extends Animal {
  speak() {
    return `${this.name} barks!`;
  }
}

const d = new Dog("Rex");
console.log(d.speak()); // "Rex barks!"
```

## Super Keyword
`super` is used in a child (derived) class to:
1. Call the **parent class constructor**.
2. Access **parent class methods**.

```js
class Vehicle {
  constructor(brand) {
    this.brand = brand;
  }
  info() {
    return `Brand: ${this.brand}`;
  }
}

class Car extends Vehicle {
  constructor(brand, model) {
    super(brand); // calls Vehicle's constructor
    this.model = model;
  }
  info() {
    return `${super.info()}, Model: ${this.model}`;
  }
}

const c = new Car("Toyota", "Camry");
console.log(c.info()); // "Brand: Toyota, Model: Camry"
```

## Interview Questions

- **Q: What is an object in JavaScript?**
  A: An object is a collection of key-value pairs. Keys are strings (or Symbols) and values can be anything — strings, numbers, functions, other objects, etc.

- **Q: What is a prototype in JavaScript?**
  A: Every object has a `[[Prototype]]` link to another object. When a property is not found on the object itself, JS looks up the prototype chain.

- **Q: What is prototypal inheritance?**
  A: Objects can inherit properties/methods from other objects via the prototype chain. If a property isn't found on an object, JS looks in its prototype.

- **Q: What is the difference between `__proto__` and `prototype`?**
  A: `__proto__` is the actual prototype link on an object instance. `prototype` is a property on constructor functions used to set the prototype for new instances.

- **Q: What is shadowing in prototypes?**
  A: When an object has a property with the same name as one in its prototype, the object's own property "shadows" the inherited one.

- **Q: What does `super` do in a class?**
  A: `super()` calls the parent class constructor. `super.method()` calls a parent class method from inside a child class.

- **Q: What is the difference between `Object.create()` and `new`?**
  A: `Object.create(proto)` creates a new object with `proto` as its prototype. `new Constructor()` creates an instance using a constructor function/class.
