In n8n, nodes are categorized by their role in the workflow. Understanding these types helps you structure automations that are both functional and scalable. 
## 1. Trigger Nodes (Workflow Starters)
Every workflow must start with at least one Trigger Node. These "listen" for an event to happen before firing the automation. 
- `Webhook Trigger`: Fires when an external service sends an HTTP request to your n8n URL.
- `Schedule (Cron) Trigger`: Runs at specific intervals (e.g., "every Monday at 9 AM").
- `App Triggers`: Respond to specific app events, like a new email in Gmail or a new row in Airtable. 
## 2. Action Nodes (Tasks)
These nodes perform the actual work once the workflow is running. They are the most common type and include:
- `App Actions`: Direct integrations with 400+ services (e.g., Slack: Send Message, Google Sheets: Append Row).
- `HTTP Request Node`: The "universal connector" for any service that doesn't have a dedicated n8n node. 
## 3. Logic & Control Nodes (The Brain)
These "Core Nodes" don't connect to outside apps but instead manage how data flows through your workflow. 
- `IF / Switch Nodes`: Branch your workflow based on conditions (e.g., "If Price > 100").
- `Merge Node`: Combines data from two different branches back into one stream.
- `Wait Node`: Pauses the workflow for a set amount of time. 
## 4. Data Transformation Nodes
These nodes clean and reshape your data to make it usable for the next step. 
- `Edit Fields (Set) Node`: Used to create, rename, or delete specific data fields.
- `Code Node`: Allows you to write custom JavaScript or Python for complex logic. 
## 5. AI & LangChain Nodes (Agentic AI)
A newer category for building intelligent agents. 
- `AI Agent Node`: Uses LLMs (like OpenAI or Anthropic) to "reason" and use tools to complete tasks.
- `Vector Store Nodes`: Allow your AI to "remember" or search through uploaded documents.
