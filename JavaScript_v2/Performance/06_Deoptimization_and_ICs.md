# 📌 06 — Deoptimization & Inline Caches

## 🧠 Concept Explanation

(See also Advanced/03_Garbage_Collection.md and Performance/01_Code_Optimization.md)

Inline Caches (ICs) are V8's mechanism for caching the result of property lookups. They exist in the generated bytecode and are updated as V8 observes the types flowing through each operation.

**IC States:**
1. **Uninitialized (0)** — Never executed
2. **Premonomorphic (1)** — Seen once
3. **Monomorphic (1 type)** — Fast path: direct slot access
4. **Polymorphic (2-4 types)** — Linear search through cached handlers
5. **Megamorphic (5+ types)** — Slow: generic handler, no caching

## 🔍 Code Examples

### IC-Friendly Code Patterns

```javascript
// MEGAMORPHIC — BAD
function processItem(item) {
  return item.value  // Called with objects of 5+ different shapes
}

// Called with diverse shapes:
processItem({ value: 1 })
processItem({ x: 2, value: 3 })
processItem({ value: 4, y: 5 })
processItem({ a: 1, b: 2, value: 6 })
processItem({ value: 7, c: 3, d: 4 })
// IC: megamorphic → slow every call

// MONOMORPHIC — GOOD
function processItem(item) {
  return item.value
}
// Called ONLY with { value: N } shaped objects → monomorphic IC
```

### Tracking Deoptimizations

```bash
# Trace optimizations and deoptimizations
node --trace-opt --trace-deopt --allow-natives-syntax app.js 2>&1 | grep -E "(optimiz|deoptimiz)"

# Output:
# [optimizing: processItem (0x...)]
# [deoptimizing: processItem: wrong type for property access]
# [optimizing: processItem (0x...)] (after type observation)
```

### V8 Native Syntax for Testing

```javascript
// node --allow-natives-syntax

function add(a, b) { return a + b }

// Warmup
for (let i = 0; i < 10000; i++) add(1, 2)
%OptimizeFunctionOnNextCall(add)
add(1, 2)

console.log(%GetOptimizationStatus(add))
// 2 = optimized, 4 = never optimized, 8 = already deoptimized, 16 = optimized by maglev

// Trigger deoptimization by passing wrong type:
add("hello", 2)  // String + Number → type assumption violated
console.log(%GetOptimizationStatus(add))  // 8 = deoptimized!
```

## 🏢 Industry Best Practices

1. **Profile with --prof** — Generate V8 tick profiler output for production analysis.
2. **Use consistent argument types** — Never mix number/string/object at same callsite.
3. **Avoid `delete` on objects** — Forces dictionary mode (slow property access).
4. **Keep object shapes stable** — No property additions after construction.
5. **Use TypeScript** — Type system catches many shape inconsistency issues at compile time.
