# Ec2

## Why This Exists
Buying physical servers takes months of planning, ordering, and racking, plus huge upfront costs. What if you only need a server for 3 hours to run an experiment? EC2 (Elastic Compute Cloud) lets you rent virtual servers by the minute, giving you raw computing power instantly.

## Real World Analogy
**Renting a Car vs Buying a Car**. 
Buying a car means paying $30,000 upfront, doing your own maintenance, and being stuck with it if you suddenly need a truck instead of a sedan. 
EC2 is like renting a car. You pay by the hour. You can rent a tiny Smart car (t2.micro) for a quick trip, or a massive 18-wheeler (p3.16xlarge) to haul heavy cargo (AI training), and return it the moment you're done.

## Core Concepts
*   **Instance Types:** The hardware specs. `t2.micro` (small, cheap), `m5.large` (balanced), `c5.xlarge` (compute-heavy).
*   **AMI (Amazon Machine Image):** The operating system template (e.g., Ubuntu, Amazon Linux, Windows).
*   **EBS Volumes:** The virtual hard drives attached to your EC2 instance.
*   **Security Groups:** A virtual firewall that controls what network traffic is allowed in or out of the instance.

## Architecture / Flow
1. **Choose AMI:** Select Ubuntu 22.04.
2. **Choose Instance Type:** Select `t2.micro`.
3. **Configure Network:** Put it in a VPC and attach a Security Group allowing port 22 (SSH) and 80 (HTTP).
4. **Add Storage:** Attach a 20GB EBS volume.
5. **Launch:** AWS provisions the VM and gives you a Public IP address in seconds.

## Practical Commands
*   *(Usually done via AWS Console or Terraform, but CLI is possible)*
*   `ssh -i my-key.pem ubuntu@<public-ip>` - Connect to your Linux instance.
*   `aws ec2 stop-instances --instance-ids i-123456` - Turn it off to save money.

## Hands-On Exercise
Launch a Free Tier `t2.micro` Amazon Linux instance. Use SSH to connect to it. Run `sudo yum install httpd -y` and `sudo systemctl start httpd`. Open a web browser, paste the Public IP, and see the default Apache test page!

## Mini Project
**"Cloud Portfolio"**
Launch an EC2 instance. Install a web server (Nginx or Apache). Copy the HTML/CSS files of your personal portfolio website onto the instance. Ensure the Security Group allows HTTP traffic, and share the IP address with a friend.

## Real Production Usage
While containers (Docker) and Serverless are trendy, massive chunks of the internet still run directly on EC2. It is used for legacy monolithic applications, custom databases, or specialized high-performance computing tasks where you need absolute control over the OS kernel.

## Common Mistakes
*   **Leaving it running:** Forgetting to stop or terminate an instance after an experiment and getting a surprise $50 bill at the end of the month.
*   **Wide Open Security Groups:** Setting a rule allowing SSH (Port 22) from `0.0.0.0/0` (the entire internet). Bots will continuously try to hack your server. Only allow SSH from your specific home IP address.

## Debugging Guide
*   **Connection Refused on SSH?** 
    1. Did you assign a Public IP?
    2. Is Port 22 open in the Security Group?
    3. Are you using the correct `.pem` key file and the correct username (e.g., `ubuntu` vs `ec2-user`)?

## Best Practices
*   **Use IAM Roles:** Never hardcode AWS Access Keys inside your application code on an EC2 instance. Attach an IAM Role to the instance instead.
*   **Tagging:** Always add a "Name" and "Environment" tag to your instances. A cluster of 50 unnamed instances is a nightmare to manage.

## Interview Questions
*   **Q: What happens to the data on an EBS volume when you stop (power off) an EC2 instance?**
    *   *A: The data persists. You only stop paying for the compute (CPU/RAM), but you continue paying a small fee for the stored data on the EBS volume.*
*   **Q: What is a Security Group?**
    *   *A: It acts as a stateful virtual firewall at the instance level, controlling inbound and outbound traffic.*

## Summary
EC2 is the foundational building block of AWS. It gives you the ultimate flexibility of a raw Linux or Windows server, instantly available and scalable, forming the basis for almost all other AWS services.
