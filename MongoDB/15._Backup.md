# Backup & Restore in MongoDB

## Simple Explanation
**Backup** = making a copy of your database so you can restore it if something goes wrong (accidental delete, server crash, data corruption).

**Restore** = bringing your database back from a backup.

Think of it like a **save point in a video game**:
- **Backup** = saving the game before a hard level
- **Restore** = going back to that save if you lose

Without backups, a single mistake can wipe months of user data permanently.

---

## Real-World Example
A developer accidentally runs:
```bash
db.users.deleteMany({})  # deletes ALL users
```
Without a backup → data is permanently lost.
With a daily backup → restore to yesterday's data in minutes.

## Backup Methods

### Method 1: `mongodump` — Shell Export

`mongodump` exports your data as BSON files. Comes with MongoDB tools.

```bash
# Backup entire MongoDB server
mongodump --out /backups/all

# Backup a specific database
mongodump --db myapp --out /backups/myapp

# Backup a specific collection
mongodump --db myapp --collection users --out /backups/

# Backup with authentication
mongodump --uri "mongodb://username:password@localhost:27017/myapp" --out /backups/

# Backup from MongoDB Atlas
mongodump --uri "mongodb+srv://user:pass@cluster.mongodb.net/myapp" --out /backups/
```

Output structure:
```
backups/
  └── myapp/
        ├── users.bson      ← collection data
        ├── users.metadata.json
        ├── products.bson
        └── products.metadata.json
```

---

### Method 2: `mongorestore` — Restore from Backup

```bash
# Restore an entire database
mongorestore --db myapp /backups/myapp

# Restore with drop (replaces existing data)
mongorestore --db myapp --drop /backups/myapp

# Restore a specific collection
mongorestore --db myapp --collection users /backups/myapp/users.bson

# Restore to a different database name
mongorestore --nsFrom "myapp.*" --nsTo "myapp_restored.*" /backups/
```

---

### Method 3: `mongoexport` — Export as JSON/CSV

`mongoexport` is simpler than `mongodump` — exports readable JSON or CSV.

```bash
# Export a collection as JSON
mongoexport --db myapp --collection users --out users.json

# Export as CSV with specific fields
mongoexport --db myapp --collection products \
  --fields name,price,category \
  --type csv \
  --out products.csv

# Export with a query filter
mongoexport --db myapp --collection orders \
  --query '{ "status": "completed" }' \
  --out completed_orders.json
```

---

### Method 4: `mongoimport` — Import JSON/CSV

```bash
# Import JSON file
mongoimport --db myapp --collection users --file users.json

# Import JSON array file
mongoimport --db myapp --collection users --file users.json --jsonArray

# Import CSV file (with headers in first row)
mongoimport --db myapp --collection products \
  --type csv --headerline \
  --file products.csv

# Drop and reimport
mongoimport --db myapp --collection users --drop --file users.json
```

---

### Method 5: MongoDB Atlas Continuous Backups (Cloud)

For Atlas users, backup is managed automatically:
- **M10+ clusters** → Continuous Cloud Backups
- **Free tier** → No automated backup (manually export)

```
Atlas Dashboard → Backup → Create Snapshot → Download or Restore
```

## Automated Backup Script (Linux/Mac)
```bash
#!/bin/bash
# backup.sh — run daily via cron

DATE=$(date +%Y-%m-%d)
BACKUP_DIR="/backups/$DATE"

echo "Starting backup for $DATE..."
mongodump --uri "$MONGODB_URI" --out "$BACKUP_DIR"

# Compress the backup
tar -czf "$BACKUP_DIR.tar.gz" "$BACKUP_DIR"
rm -rf "$BACKUP_DIR"

echo "Backup complete: $BACKUP_DIR.tar.gz"

# Keep only last 7 days (delete older backups)
find /backups/ -name "*.tar.gz" -mtime +7 -delete
echo "Old backups cleaned up."
```

Set up a cron job:
```bash
# Run backup every day at 2am
0 2 * * * /path/to/backup.sh >> /var/log/mongo_backup.log 2>&1
```

## Backup Tools Comparison

| Tool | Format | Use Case |
|------|--------|---------|
| `mongodump` | BSON | Full DB/collection backup (recommended) |
| `mongorestore` | BSON | Restore from mongodump backup |
| `mongoexport` | JSON/CSV | Share data, import into other tools |
| `mongoimport` | JSON/CSV | Import data from files |
| Atlas Cloud Backup | Managed | Automated, point-in-time recovery |

## Interview Questions

- **Q: What is `mongodump`?**
  A: A CLI tool that exports MongoDB database or collection data as BSON files. The recommended way to create backups for self-hosted MongoDB.

- **Q: What is `mongorestore`?**
  A: A CLI tool that imports BSON data created by `mongodump` back into MongoDB. Used to restore from a backup.

- **Q: What is the difference between `mongodump` and `mongoexport`?**
  A: `mongodump` exports as BSON (binary, faster, for full backups). `mongoexport` exports as JSON or CSV (human-readable, for sharing data).

- **Q: What does `--drop` flag do in `mongorestore`?**
  A: Drops the existing collection before restoring — ensures a clean restore without merging old and new data.

- **Q: How does MongoDB Atlas handle backups?**
  A: M10+ clusters get Continuous Cloud Backups with point-in-time recovery. Snapshots can be restored to the cluster or downloaded. Free tier requires manual export.

- **Q: Why is backup critical for production databases?**
  A: Accidental deletes, data corruption, hardware failures, or cyberattacks can permanently destroy data. Backups are the last line of defense.

- **Q: What is a point-in-time restore?**
  A: The ability to restore a database to its exact state at a specific moment in the past — not just the last full backup, but any moment within a retention window.
