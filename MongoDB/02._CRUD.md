# CRUD Operations

## Simple Explanation
**CRUD** = **C**reate, **R**ead, **U**pdate, **D**elete — the four basic operations you can perform on data in any database.

Think of it like managing contacts on your phone:
- **Create** = Adding a new contact
- **Read** = Looking up a contact
- **Update** = Editing a contact's phone number
- **Delete** = Removing a contact

---

## Real-World Example
An e-commerce backend handles products:
- **Create** — add a new product when a seller lists it
- **Read** — fetch all products for the shop page
- **Update** — change the price when there's a sale
- **Delete** — remove a product that's discontinued

---

## Code Examples

### CREATE — Insert Documents

```sh
# Insert ONE document
db.products.insertOne({
  name: "Laptop",
  price: 55000,
  category: "electronics",
  inStock: true,
  tags: ["portable", "powerful"],
  createdAt: new Date()
})

# Insert MULTIPLE documents at once (batch insert)
db.products.insertMany([
  { name: "Phone", price: 20000, category: "electronics", inStock: true },
  { name: "Shirt", price: 500, category: "clothing", inStock: true },
  { name: "Headphones", price: 3000, category: "electronics", inStock: false }
])
```

---

### READ — Find Documents

```sh
# Find ALL documents in a collection
db.products.find()

# Find with a filter (condition)
db.products.find({ category: "electronics" })

# Find ONE matching document (returns first match)
db.products.findOne({ name: "Laptop" })

# Find by _id (most efficient read)
db.products.findById(ObjectId("64abc123..."))

# Projection — choose which fields to return
# 1 = include, 0 = exclude
db.products.find({}, { name: 1, price: 1, _id: 0 })
# Returns only name and price, hides _id

# Find with multiple conditions
db.products.find({ category: "electronics", inStock: true })

# Count matching documents
db.products.countDocuments({ category: "electronics" })
```

---

### UPDATE — Modify Documents

```sh
# Update ONE document (first match)
db.products.updateOne(
  { name: "Laptop" },     // filter — which document
  { $set: { price: 52000, onSale: true } }  // changes to apply
)

# Update MANY documents
db.products.updateMany(
  { category: "electronics" },
  { $set: { discount: 10 } }
)

# upsert: true — insert if no document matches
db.products.updateOne(
  { name: "Tablet" },
  { $set: { price: 30000, category: "electronics" } },
  { upsert: true }
)

# Replace an entire document (replaceOne)
db.products.replaceOne(
  { name: "Shirt" },
  { name: "T-Shirt", price: 600, category: "clothing", inStock: true }
)
```

### Common Update Operators
| Operator | What it does |
|----------|-------------|
| `$set` | Set a field value |
| `$unset` | Remove a field |
| `$inc` | Increment a number |
| `$push` | Add an item to an array |
| `$pull` | Remove a item from an array |
| `$addToSet` | Add to array only if unique |
| `$rename` | Rename a field |

```sh
# $inc — increment stock count
db.products.updateOne({ name: "Laptop" }, { $inc: { stock: -1 } })

# $push — add a tag to the tags array
db.products.updateOne({ name: "Laptop" }, { $push: { tags: "gaming" } })

# $unset — remove the discount field
db.products.updateOne({ name: "Laptop" }, { $unset: { discount: "" } })
```

---

### DELETE — Remove Documents

```sh
# Delete ONE document (first match)
db.products.deleteOne({ name: "Shirt" })

# Delete MANY documents
db.products.deleteMany({ inStock: false })

# Delete ALL documents in a collection
db.products.deleteMany({})
```

## CRUD Summary Table

| Operation | Method | Description |
|-----------|--------|-------------|
| Create | `insertOne()` | Insert one document |
| Create | `insertMany()` | Insert multiple documents |
| Read | `find()` | Get all / filtered documents |
| Read | `findOne()` | Get first matching document |
| Update | `updateOne()` | Update first matching document |
| Update | `updateMany()` | Update all matching documents |
| Update | `replaceOne()` | Replace entire document |
| Delete | `deleteOne()` | Delete first matching document |
| Delete | `deleteMany()` | Delete all matching documents |

## Interview Questions

- **Q: What is the difference between `insertOne` and `insertMany`?**
  A: `insertOne` inserts a single document. `insertMany` inserts an array of documents in one batch operation — faster than multiple `insertOne` calls.

- **Q: What does `$set` do in an update?**
  A: Sets (creates or updates) a specific field without touching other fields in the document.

- **Q: What is `upsert` in MongoDB?**
  A: An update option (`{ upsert: true }`) that inserts a new document if no matching document is found, instead of doing nothing.

- **Q: What is the difference between `updateOne` and `replaceOne`?**
  A: `updateOne` with `$set` only changes specified fields. `replaceOne` replaces the entire document content (except `_id`).

- **Q: Why is `findById` faster than `findOne`?**
  A: Because `_id` is indexed by default. MongoDB goes directly to the document without scanning the entire collection.

- **Q: What does `deleteMany({})` do?**
  A: Deletes ALL documents in the collection. The collection itself remains — only documents are removed.

- **Q: How do you increment a numeric field in MongoDB?**
  A: Use `$inc`: `db.col.updateOne({ _id: id }, { $inc: { stock: -1 } })`.
