# Data Types in MongoDB (BSON)

## Simple Explanation
MongoDB stores data in **BSON** (Binary JSON) format. BSON is like JSON but extended with additional types that JavaScript/JSON alone can't handle — like dates, binary data, and unique IDs.

Think of BSON as a **supercharged version of JSON** that supports more data types for real-world use cases.

---

## Real-World Example
When you store an order in MongoDB, you need:
- A `String` for the product name
- A `Number` for the price
- A `Date` for when it was placed
- An `ObjectId` to uniquely identify the order
- A `Boolean` to track if it's paid
- An `Array` for the list of items

---

## Code Example — All Data Types in One Document

```js
db.orders.insertOne({
  // 1. String
  customerName: "Nishant",
  status: "pending",

  // 2. Number (Int32, Int64, Double)
  price: 55000.50,        // Double (decimal)
  quantity: 3,            // Int32

  // 3. Boolean
  isPaid: false,
  isDelivered: true,

  // 4. ObjectId — 12-byte unique identifier (auto-generated primary key)
  userId: ObjectId("64abc1234def5678abcd1234"),

  // 5. Date / Timestamp
  createdAt: new Date(),           // ISODate
  updatedAt: ISODate("2026-03-24T17:00:00Z"),

  // 6. Array — list of values or objects
  items: ["Laptop", "Mouse", "Keyboard"],
  tags: [{ name: "urgent" }, { name: "premium" }],

  // 7. Embedded Document (Object / Nested object)
  address: {
    street: "123 MG Road",
    city: "Mumbai",
    zip: "400001",
    country: "India"
  },

  // 8. Null
  couponCode: null,

  // 9. Binary Data
  // profilePicture: BinData(0, "base64string=="),

  // 10. Regular Expression
  // namePattern: /^Nishant/i
})
```

## BSON Data Types Reference Table

| Type | Description | Example |
|------|-------------|---------|
| `String` | Text data (UTF-8) | `"Nishant"` |
| `Number` | Integer or decimal | `55000`, `3.14` |
| `Double` | 64-bit floating point | `3.14159` |
| `Int32` | 32-bit integer | `42` |
| `Int64` | 64-bit integer (Long) | Large numbers |
| `Boolean` | `true` or `false` | `isPaid: true` |
| `ObjectId` | 12-byte unique ID | Auto-generated `_id` |
| `Date` | Date and time | `new Date()`, `ISODate()` |
| `Array` | List of values | `["a", "b", "c"]` |
| `Object` | Embedded document (nested object) | `{ city: "Mumbai" }` |
| `Null` | Absence of value | `null` |
| `Binary` | Raw binary data | Images, files |
| `Regex` | Regular expression | `/^Nishant/i` |
| `Decimal128` | High-precision decimal (for money) | `NumberDecimal("55000.50")` |

## ObjectId — The Primary Key

Every document gets an `_id` field (auto-assigned as `ObjectId` if not provided).

An `ObjectId` is **12 bytes** consisting of:
```
| 4 bytes  | 3 bytes  | 2 bytes | 3 bytes        |
| timestamp| machine  | process | random counter |
```

```js
const id = ObjectId("64abc1234def5678abcd1234");

console.log(id.toString());          // "64abc1234def5678abcd1234"
console.log(id.getTimestamp());      // Date when it was created
console.log(id.equals(otherId));     // compare two ObjectIds
```

## Working with Dates

```js
// Insert with current timestamp
db.logs.insertOne({ event: "login", at: new Date() });

// Query by date range
db.logs.find({
  at: {
    $gte: new Date("2026-01-01"),
    $lt: new Date("2026-04-01")
  }
});

// ISODate format in queries
db.orders.find({ createdAt: { $gt: ISODate("2026-03-01T00:00:00Z") } });
```

## Interview Questions

- **Q: What is BSON?**
  A: Binary JSON — a binary-encoded format for storing MongoDB documents. It supports more data types than JSON (ObjectId, Date, Binary, etc.) and is faster to parse.

- **Q: What is an `ObjectId` in MongoDB?**
  A: A 12-byte unique identifier auto-generated for every document's `_id` field. It encodes a timestamp, machine ID, process ID, and a random counter.

- **Q: What is the difference between `Date` and `Timestamp` in BSON?**
  A: `Date` is a standard date-time stored as milliseconds since epoch. `Timestamp` is an internal MongoDB type used for replication and oplog — not typically used by developers.

- **Q: What is `Decimal128` and when should you use it?**
  A: A high-precision 128-bit decimal type. Use it for financial calculations where floating-point precision matters (e.g., currency, prices).

- **Q: Can you store an array inside a MongoDB document?**
  A: Yes. Arrays are first-class BSON types. You can store arrays of strings, numbers, objects, or even other arrays.

- **Q: Can two documents in the same collection have different fields?**
  A: Yes. MongoDB has a flexible schema. Each document can have completely different fields — one might have `address`, another might not.
