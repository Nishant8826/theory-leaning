# Cursor Methods & Projection

## Simple Explanation
When you run `db.collection.find()`, MongoDB doesn't return all documents at once — it returns a **cursor**, which is a **pointer to the result set**.

You can then chain methods on the cursor to:
- **Sort** results
- **Limit** how many you get
- **Skip** some (useful for pagination)
- **Count** results
- **Project** (choose which fields to show/hide)

Think of it like a **search result page**:
- You search (run `find()`) → get a cursor
- You pick how many results per page (`limit`)
- You skip to page 2 (`skip`)
- You sort by relevance (`sort`)

---

## Real-World Example
An e-commerce website listing products:
- Sort by price (low to high)
- Show only 12 per page
- Skip 24 products for page 3
- Return only name, price, and image URL (projection)

---

## Code Examples

### Sort
```js
// Sort by price ascending (low to high)
db.products.find().sort({ price: 1 })

// Sort by price descending (high to low)
db.products.find().sort({ price: -1 })

// Multiple sort criteria (sort by category, then by price)
db.products.find().sort({ category: 1, price: -1 })
```

---

### Limit — Restrict Number of Results
```js
// Get only the top 5 products
db.products.find().limit(5)

// Get 5 most expensive products
db.products.find().sort({ price: -1 }).limit(5)
```

---

### Skip — Skip First N Documents
```js
// Skip the first 10 documents
db.products.find().skip(10)

// Used for pagination: page 3 with 10 items per page
const page = 3;
const limit = 10;
db.products.find().sort({ createdAt: -1 }).skip((page - 1) * limit).limit(limit)
// skip(20) → shows results 21–30
```

---

### Count — Count Matching Documents
```js
// Count all documents in a collection
db.products.countDocuments()

// Count with a filter
db.products.countDocuments({ category: "electronics" })

// Count cursor results (deprecated — use countDocuments instead)
// db.products.find({ category: "electronics" }).count()
```

---

### Projection — Select Fields to Return

Projection is the 2nd argument in `find()` — it controls which fields appear in the result.

```js
// Include only specific fields (1 = include)
db.products.find({}, { name: 1, price: 1 })
// Returns: { _id: ..., name: "Laptop", price: 55000 }

// Exclude specific fields (0 = exclude)
db.products.find({}, { __v: 0, createdAt: 0 })
// Returns everything EXCEPT __v and createdAt

// Exclude _id
db.products.find({}, { name: 1, price: 1, _id: 0 })
// Returns: { name: "Laptop", price: 55000 }

// ⚠️ You can't mix 0 and 1 in the same projection
// EXCEPT: _id can be 0 even when others are 1
// This is INVALID (mix of include/exclude):
// db.products.find({}, { name: 1, price: 0 })  ❌

// Project nested field
db.users.find({}, { "address.city": 1, name: 1, _id: 0 })

// Project an array element (first element of reviews)
db.products.find({}, { reviews: { $slice: 1 } })
```

---

### Chaining Methods Together
```js
// Full pagination query with projection
db.products
  .find({ inStock: true }, { name: 1, price: 1, _id: 0 })
  .sort({ price: -1 })
  .skip(20)
  .limit(10)

// Most recent 5 users — only name and email
db.users
  .find({}, { name: 1, email: 1, _id: 0 })
  .sort({ createdAt: -1 })
  .limit(5)
```

## Pagination Pattern
```js
// Page-based pagination
function paginatedQuery(page, limit) {
  return db.products
    .find()
    .sort({ createdAt: -1 })
    .skip((page - 1) * limit)
    .limit(limit);
}

// Usage: page 1 = first 10, page 2 = next 10, etc.
```

---

## Cursor Methods Summary

| Method | What it does |
|--------|------------|
| `.sort({ field: 1/-1 })` | Sort ascending (1) or descending (-1) |
| `.limit(n)` | Return max n documents |
| `.skip(n)` | Skip first n documents |
| `.countDocuments()` | Count matching documents |
| `.find({}, { field: 1 })` | Projection — include specific fields |
| `.find({}, { field: 0 })` | Projection — exclude specific fields |

## Interview Questions

- **Q: What is a cursor in MongoDB?**
  A: A pointer to the result set returned by `find()`. Instead of loading all documents at once, MongoDB iterates through them in batches using the cursor.

- **Q: What is projection in MongoDB?**
  A: A way to control which fields are returned in query results. Use `1` to include a field, `0` to exclude. You can't mix both (except `_id`).

- **Q: How do you implement pagination in MongoDB?**
  A: Use `.skip((page - 1) * limit).limit(limit)`. For example, page 3 with limit 10: `.skip(20).limit(10)`.

- **Q: What is the difference between `.sort(1)` and `.sort(-1)`?**
  A: `1` = ascending (A-Z, 0-9). `-1` = descending (Z-A, 9-0).

- **Q: Why is `countDocuments()` preferred over `.count()`?**
  A: `.count()` is deprecated in newer MongoDB versions. `countDocuments()` is accurate with all query filters and respects index hints.

- **Q: Can you chain `.sort()`, `.skip()`, and `.limit()` together?**
  A: Yes. MongoDB always executes them in this internal order: `sort → skip → limit`, regardless of the order you chain them.
