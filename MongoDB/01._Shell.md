# MongoDB Shell (mongosh)

## Simple Explanation
The **MongoDB Shell** (`mongosh`) is a command-line interface that lets you interact directly with your MongoDB database — create databases, insert documents, run queries, all from a terminal window.

Think of it like a **conversation with your database**:
- You type a command → MongoDB does the work → shows you the result.

It's great for learning, debugging, and quick database operations.

---

## Real-World Example
You're building a backend app and want to quickly check if a user was inserted correctly, or test a query before writing it in your Node.js code — you use the shell.

---

## MongoDB Architecture

MongoDB stores data in:
- **Collections** = groups of documents (like folders)
- **Documents** = individual records stored as BSON (Binary JSON objects)

```
MongoDB Server
   └── myapp (database)
         ├── users (collection)
         │     ├── { _id: ..., name: "Nishant", age: 22 }
         │     └── { _id: ..., name: "Raj", age: 25 }
         └── products (collection)
               └── { _id: ..., name: "Laptop", price: 50000 }
```

## Code Example — Shell Commands

### Connecting to MongoDB
```sh
# Connect to local MongoDB
mongosh

# Connect to MongoDB Atlas (cloud)
mongosh "mongodb+srv://<username>:<password>@<cluster>.mongodb.net"
```

### Database Commands
```sh
# Show all databases on the server
show dbs

# Switch to (or create) a database
use myapp

# Show current database name
db

# Drop the current database
db.dropDatabase()
```

### Collection Commands
```sh
# Show all collections in current database
show collections

# Create a collection explicitly
db.createCollection("users")

# Drop a collection
db.users.drop()

# Check collection stats
db.users.stats()
```

### Quick Insert & Find (Test It Out)
```sh
# Insert a document
db.users.insertOne({ name: "Nishant", age: 22, city: "Mumbai" })

# Find all documents in a collection
db.users.find()

# Find with formatting (pretty print)
db.users.find().pretty()

# Count documents
db.users.countDocuments()

# Find one document
db.users.findOne({ name: "Nishant" })
```

### Other Useful Shell Commands
```sh
# List all commands
help

# Get help for a specific method
db.users.find.help()

# Check MongoDB server version
db.version()

# Check server status
db.serverStatus()

# Exit the shell
exit
```

---

## Important Notes

- `use <dbName>` switches to that database. If it doesn't exist, MongoDB creates it when you first insert data.
- Collections are also created automatically on first insert — `createCollection()` is optional.
- `db` refers to the currently selected database.
- `mongosh` replaced the old `mongo` shell (deprecated in MongoDB 5+).

## Interview Questions

- **Q: What is `mongosh`?**
  A: The MongoDB Shell — a command-line interface for interacting with MongoDB databases. Used to run queries, manage collections, and debug.

- **Q: What does `use <dbName>` do?**
  A: Switches to that database. If it doesn't exist, MongoDB creates it automatically when the first document is inserted.

- **Q: What is the difference between a database, collection, and document in MongoDB?**
  A: Database = container of collections. Collection = group of related documents (like a table). Document = a single BSON record (like a row).

- **Q: Do you need to create a collection before inserting data?**
  A: No. MongoDB creates a collection automatically when you first insert a document into it.

- **Q: What is `db.collection.drop()` vs `db.dropDatabase()`?**
  A: `drop()` removes a single collection. `dropDatabase()` deletes the entire database including all its collections.

- **Q: What replaced the old `mongo` shell?**
  A: `mongosh` (MongoDB Shell) — introduced in MongoDB 5. It has better syntax highlighting and modern JavaScript support.
