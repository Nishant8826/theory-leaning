# Aggregation Pipeline

## Simple Explanation
The **Aggregation Pipeline** is MongoDB's way to process and transform data through a sequence of steps called **stages**.

Think of it like a **factory assembly line**:
- Raw material enters (your collection)
- Each station (stage) does one job: filter, group, reshape, sort
- Final product comes out (transformed result)

Each stage takes documents as input and passes transformed documents to the next stage.

---

## Real-World Example
On an e-commerce site, to get a **sales report** showing total revenue per category:
1. `$match` — Filter only completed orders
2. `$group` — Group by category, sum the revenue
3. `$sort` — Sort by revenue descending
4. `$limit` — Return top 5 categories

---

## Code Examples

### Basic Syntax
```js
db.collection_name.aggregate([
  { $stage1: { /* config */ } },
  { $stage2: { /* config */ } },
  // ...more stages
])
```

---

### Stage 1: `$match` — Filter Documents
```js
// Only process electronics with price > 1000
db.products.aggregate([
  { $match: { category: "electronics", price: { $gt: 1000 } } }
])
// Works just like find() — but inside the pipeline
```

---

### Stage 2: `$group` — Group & Aggregate
```js
// Count products per category
db.products.aggregate([
  {
    $group: {
      _id: "$category",          // group by this field
      count: { $sum: 1 },        // count documents per group
      avgPrice: { $avg: "$price" },
      totalRevenue: { $sum: "$price" },
      minPrice: { $min: "$price" },
      maxPrice: { $max: "$price" }
    }
  }
])

// Result:
// { _id: "electronics", count: 15, avgPrice: 25000, totalRevenue: 375000 }
// { _id: "clothing", count: 30, avgPrice: 750, totalRevenue: 22500 }
```

---

### Stage 3: `$project` — Reshape Output
```js
db.users.aggregate([
  {
    $project: {
      _id: 0,
      fullName: { $concat: ["$firstName", " ", "$lastName"] },  // computed field
      email: 1,
      ageGroup: {
        $cond: {
          if: { $gte: ["$age", 18] },
          then: "adult",
          else: "minor"
        }
      }
    }
  }
])
```

---

### Stage 4: `$sort`, `$limit`, `$skip`
```js
db.products.aggregate([
  { $match: { inStock: true } },
  { $sort: { price: -1 } },   // sort expensive first
  { $skip: 10 },              // skip first 10
  { $limit: 5 }               // return next 5
])
```

---

### Stage 5: `$unwind` — Flatten Arrays
```js
// Each product has an array of tags
// $unwind creates one document per tag
db.products.aggregate([
  { $unwind: "$tags" }
])
// { name: "Laptop", tags: "gaming" }
// { name: "Laptop", tags: "portable" }
// { name: "Laptop", tags: "powerful" }
```

---

### Stage 6: `$lookup` — Join Collections
```js
// Join orders with users collection
db.orders.aggregate([
  {
    $lookup: {
      from: "users",            // target collection
      localField: "userId",     // field in orders
      foreignField: "_id",      // field in users
      as: "userDetails"         // name of the result array
    }
  }
])
// Each order now has a userDetails array with full user data
```

---

### Stage 7: `$out` — Write Results to a New Collection
```js
db.orders.aggregate([
  { $match: { status: "completed" } },
  { $group: { _id: "$category", total: { $sum: "$amount" } } },
  { $out: "revenue_summary" }    // saves result to new collection
])
```

---

### Full Real-World Pipeline Example
```js
// Monthly sales report: total revenue per category for 2026
db.orders.aggregate([
  // Step 1: Only completed orders in 2026
  {
    $match: {
      status: "completed",
      createdAt: { $gte: new Date("2026-01-01") }
    }
  },
  // Step 2: Group by category, calculate totals
  {
    $group: {
      _id: "$category",
      totalRevenue: { $sum: "$amount" },
      orderCount: { $sum: 1 },
      avgOrderValue: { $avg: "$amount" }
    }
  },
  // Step 3: Add a readable name field
  {
    $project: {
      category: "$_id",
      totalRevenue: 1,
      orderCount: 1,
      avgOrderValue: { $round: ["$avgOrderValue", 2] },
      _id: 0
    }
  },
  // Step 4: Sort by revenue descending
  { $sort: { totalRevenue: -1 } },
  // Step 5: Top 10 categories only
  { $limit: 10 }
])
```

## Aggregation Stages Summary

| Stage | What it does |
|-------|-------------|
| `$match` | Filter documents (like `find()`) |
| `$group` | Group by a field, calculate totals/averages |
| `$project` | Reshape documents — include/exclude/compute fields |
| `$sort` | Sort documents |
| `$limit` | Keep only first n documents |
| `$skip` | Skip first n documents |
| `$unwind` | Flatten array field — one doc per array element |
| `$lookup` | Join with another collection |
| `$out` | Write result to a new collection |
| `$addFields` | Add or overwrite fields without removing others |
| `$count` | Count documents passing through |

## Interview Questions

- **Q: What is the MongoDB Aggregation Pipeline?**
  A: A framework for processing documents through sequential stages. Each stage transforms the data and passes it to the next — like an assembly line.

- **Q: What is `$group` in aggregation?**
  A: Groups documents by a specified field and applies accumulator expressions (like `$sum`, `$avg`, `$count`) to each group.

- **Q: What does `$unwind` do?**
  A: Deconstructs an array field — creates one document per array element. Essential before grouping or filtering on array content.

- **Q: What is `$lookup`?**
  A: A left outer join stage that merges documents from another collection based on a matching field — like a SQL JOIN.

- **Q: What is the difference between `$project` in aggregation vs projection in `find()`?**
  A: `$project` in aggregation can also compute new fields, rename fields, and apply expressions — it's far more powerful than `find()` projection.

- **Q: When would you use `$out`?**
  A: When you want to save aggregation results to a new collection — useful for materialized views or pre-computed reports.
