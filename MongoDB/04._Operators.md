# Query Operators

## Simple Explanation
**Query operators** are special keywords (starting with `$`) used inside `find()` to **filter documents** based on conditions — comparisons, logical checks, array matching, and more.

Think of them like **filters in a search bar**:
- "Show me products with price > 500"
- "Find users from Mumbai OR Delhi"
- "Get all posts with the tag 'react'"

---

## Real-World Example
An e-commerce platform uses operators to:
- Show products in a price range (`$gte`, `$lte`)
- Filter multiple categories at once (`$in`)
- Find users who haven't set an address (`$exists: false`)
- Search products with a specific tag (`$all`)

---

## Code Examples

### 1. Comparison Operators

```js
// $eq — equal to (same as plain: { price: 500 })
db.products.find({ price: { $eq: 500 } })

// $ne — not equal to
db.products.find({ status: { $ne: "sold" } })

// $gt / $gte — greater than / greater than or equal
db.products.find({ price: { $gt: 1000 } })
db.products.find({ price: { $gte: 500 } })

// $lt / $lte — less than / less than or equal
db.products.find({ price: { $lt: 5000 } })
db.products.find({ stock: { $lte: 10 } })

// Range query — price between 1000 and 5000
db.products.find({ price: { $gte: 1000, $lte: 5000 } })

// $in — matches any value in the given array
db.products.find({ category: { $in: ["electronics", "clothing"] } })

// $nin — matches NONE of the values in the array
db.products.find({ category: { $nin: ["books", "toys"] } })
```

---

### 2. Logical Operators

```js
// $and — all conditions must match (default behavior — rarely written explicitly)
db.users.find({
  $and: [
    { age: { $gte: 18 } },
    { city: "Mumbai" }
  ]
})

// Shorthand (implicit $and)
db.users.find({ age: { $gte: 18 }, city: "Mumbai" })

// $or — at least one condition must match
db.users.find({
  $or: [
    { city: "Mumbai" },
    { city: "Delhi" }
  ]
})

// $nor — none of the conditions should match
db.users.find({
  $nor: [
    { city: "Mumbai" },
    { isBlocked: true }
  ]
})

// $not — negates a condition
db.users.find({ age: { $not: { $lt: 18 } } })
// Same as: age >= 18
```

---

### 3. Element Operators

```js
// $exists — check if a field exists in the document
db.users.find({ address: { $exists: true } })   // has address field
db.users.find({ phone: { $exists: false } })    // missing phone field

// $type — check field's BSON type
db.users.find({ age: { $type: "number" } })     // age is a number
db.users.find({ createdAt: { $type: "date" } }) // date type
```

---

### 4. Array Operators

```js
// $all — array must contain ALL of the specified values
db.products.find({ tags: { $all: ["laptop", "gaming"] } })
// returns products that have BOTH "laptop" AND "gaming" in tags

// $size — array has exactly this many elements
db.products.find({ tags: { $size: 3 } })

// $elemMatch — at least one array element matches ALL conditions
db.orders.find({
  items: {
    $elemMatch: { name: "Laptop", qty: { $gte: 2 } }
  }
})
```

---

### 5. Update Operators (used inside `updateOne`/`updateMany`)

```js
// $set — set a field value
db.products.updateOne({ name: "Laptop" }, { $set: { price: 52000 } })

// $unset — remove a field
db.products.updateOne({ name: "Laptop" }, { $unset: { discount: "" } })

// $inc — increment (or decrement) a numeric field
db.products.updateOne({ name: "Laptop" }, { $inc: { stock: -1 } })

// $push — add element to array
db.products.updateOne({ name: "Laptop" }, { $push: { tags: "portable" } })

// $pull — remove specific element from array
db.products.updateOne({ name: "Laptop" }, { $pull: { tags: "gaming" } })

// $addToSet — push to array only if not already present
db.products.updateOne({ name: "Laptop" }, { $addToSet: { tags: "portable" } })
```

## Operators Reference Table

| Operator | Category | Description |
|----------|----------|-------------|
| `$eq` | Comparison | Equal to value |
| `$ne` | Comparison | Not equal to value |
| `$gt` / `$gte` | Comparison | Greater than / or equal |
| `$lt` / `$lte` | Comparison | Less than / or equal |
| `$in` | Comparison | Matches any in array |
| `$nin` | Comparison | Matches none in array |
| `$and` | Logical | All conditions must match |
| `$or` | Logical | At least one condition matches |
| `$nor` | Logical | No condition should match |
| `$not` | Logical | Negates a condition |
| `$exists` | Element | Field exists or not |
| `$type` | Element | Field matches BSON type |
| `$all` | Array | Array contains all values |
| `$size` | Array | Array has exact length |
| `$elemMatch` | Array | Array element matches condition |

## Interview Questions

- **Q: What are query operators in MongoDB?**
  A: Special `$`-prefixed keywords used in `find()` to filter documents — comparisons, logical conditions, array checks, and element type checks.

- **Q: What is the difference between `$in` and `$all`?**
  A: `$in` matches documents where a field equals any value in the array. `$all` matches documents where an array field contains ALL specified values.

- **Q: What does `$exists` do?**
  A: Checks whether a field is present (`true`) or absent (`false`) in a document.

- **Q: What is the difference between `$and` and implicit AND?**
  A: Both produce the same result. `$and` is only required when multiple conditions apply to the same field. Otherwise, comma-separating conditions in the filter object achieves the same result.

- **Q: What does `$elemMatch` do?**
  A: Matches documents where at least one array element satisfies ALL the specified conditions — useful when array elements are embedded objects.

- **Q: When would you use `$nor`?**
  A: When you want to exclude documents matching ANY of multiple conditions — like: find users who are neither blocked nor inactive.
